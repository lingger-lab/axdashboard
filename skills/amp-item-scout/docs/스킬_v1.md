# AMP Item Scout — 아이템 발굴 + 시장성 진단 + 의뢰자 매칭

## 목적

AMP 코칭 의뢰자에게 "당신의 경력과 핵심역량에 맞는 아이템 + 시장 근거"를
제시하기 위한 진단DB를 구축·유지·활용하는 스킬.

이 스킬의 2가지 모드:
- **수집 모드:** 아이템을 발굴하고 시장성 근거를 붙여 Google Sheets에 저장
- **매칭 모드:** 의뢰자의 경력·경험·핵심역량을 받아 Google Sheets에서 매칭되는 아이템을 제시

## 모드 판별

- "아이템 발굴해줘", "이번 주 스카우트", "DB 업데이트" → **수집 모드**
- "내 경력에 맞는 아이템", "물류관리 20년인데 뭘 만들 수 있어?", "의뢰자 매칭" → **매칭 모드**

## 수집 모드

### PHASE 1: 아이템 발굴 (웹 검색)

아래 소스에서 AI 온라인 서비스 아이템을 웹 검색으로 수집한다.

**해외 소스 (아이템 트렌드 + 매출 검증):**
1. Product Hunt — 최근 7일 상위/트렌딩 서비스
2. There's An AI For That (theresanaiforthat.com) — AI 도구 신규 등록
3. Indie Hackers (indiehackers.com) — 실제 MRR 공개 제품 (시장성 근거의 핵심)
4. AppSumo (appsumo.com) — 실제 판매 가격/리뷰/거래량
5. MicroLaunch (microlaunch.net) — 인디해커 신규 런칭
6. SaaSHub (saashub.com) — 서비스 비교/리뷰

**한국 소스 — 유튜브 (아이템 소개 + 사례 분석):**
검색 쿼리 예시: "AI 1인 창업 아이템 2026", "AI 부업 추천", "1인 온라인 사업"
7. 유튜브 검색 — "AI 부업", "1인 창업 아이템", "AI 수익화" 키워드로 최근 영상 탐색
   주요 채널: 동기부여학과, 신사임당, 자청, 전준수, 드로우앤드류 등
   → 이 채널들이 소개하는 구체적 아이템과 수익 사례를 추출

**한국 소스 — 블로그/매거진 (트렌드 + 분석):**
8. 네이버 블로그 검색 — "AI 창업", "AI 부업 아이템", "바이브코딩 수익화" 키워드
9. 브런치 (brunch.co.kr) — 솔로비즈니스나이트, 1인 창업 관련 에세이
10. 아이보스 (i-boss.co.kr) — 마케팅/창업 트렌드 칼럼
11. 캐릿 (careet.net) — 트렌드/비즈니스 분석
12. 이오플라 (eopla.net) — 1인 창업 매거진

**한국 소스 — 커뮤니티/마켓플레이스 (실제 거래 + 수요):**
13. 크몽 (kmong.com) — AI 관련 서비스 거래량, 실제 가격대
14. 탈잉 (taling.me) — 1:1 교육/코칭 아이템, 인기 클래스
15. 숨고 (soomgo.com) — 프리랜서 서비스 수요/견적
16. 클래스101 (class101.net) — 온라인 교육 아이템 트렌드
17. 디스콰이엇 (disquiet.io) — 한국판 Product Hunt, 국내 MVP 런칭

**한국 소스 — 불만/수요 발굴:**
18. 네이버 카페 — 업종별 카페에서 "이런 서비스 있었으면" 검색
19. 블라인드 (blind.com) — 직장인 업무 고충, AI 도입 반응
20. GeekNews (news.hada.io) — 한국 개발자/스타트업 커뮤니티

**검색 방법:** 모든 한국 소스는 Claude 웹 검색으로 접근 가능.
"site:youtube.com AI 부업 아이템 2026" 형태로 검색하거나,
채널명/사이트명 + 키워드 조합으로 검색한다.

**수집 기준:**
- 1인 또는 소규모 팀(~5명)이 운영 가능
- AI가 80% 처리 + 사람이 20% 판단하는 구조가 가능
- B2C 또는 B2SMB(소기업 대상) 우선

**각 아이템에서 추출:**

| 필드 | 설명 |
|---|---|
| 아이템명 | 서비스 이름 |
| URL | 원본 링크 |
| 한 줄 설명 | 무엇을 하는 서비스인가 |
| 고객 | 누구의 문제인가 |
| 고통 | 고객이 실제로 하는 말 |
| AI 80% | AI가 처리하는 부분 |
| 사람 20% | 사람이 판단해야 하는 마지막 한 끗 |
| 필요 역량 키워드 | 이 아이템의 "사람 20%"를 수행하려면 어떤 경력·역량이 필요한가 (자유 형식) |
| 필요 경력 수준 | 예: "해당 분야 10년+", "경력 무관" |

**"필요 역량 키워드" 작성 규칙:**
- 고정된 카테고리를 쓰지 않는다 (10대 분야 같은 분류표 없음)
- 의뢰자가 자기 경력을 설명할 때 쓸 법한 실제 단어를 사용한다
- 예시: "세무신고, 재무제표, 원가분석" / "품질검사, 공정관리, ISO" / "고객응대, CS관리, 컴플레인처리" / "콘텐츠기획, SNS운영, 카피라이팅" / "물류, 재고관리, 입출고" / "영업, 거래처관리, 견적" / "강의, 커리큘럼설계, 학생관리"
- 한 아이템에 키워드 3~7개를 쉼표로 나열
- 이 키워드가 나중에 의뢰자 경력과 시맨틱 매칭되는 검색어가 된다

### PHASE 2: 시장성 근거 수집 (상위 3~5건)

발굴된 아이템 중 상위 3~5건에 대해 4가지 근거를 웹 검색:

1. **해외에서 팔리는 증거** — 유사 서비스명 + MRR 또는 이용자 수
2. **한국에서 팔리는 증거** — 유사 서비스 + 크몽 거래량 등
3. **사람들이 불만인 증거** — 네이버카페/블라인드 관련 불만
4. **킬 리스크** — 이 아이템이 안 될 이유 1가지

데이터 신뢰도 태그 필수:
- [실시간검색] — 이번 검색에서 확인
- [추정] — 근거 기반 추론, 교차 검증 필요

근거를 못 찾으면 "근거 없음"으로 표시. 절대로 만들어내지 않는다.

### PHASE 3: Google Sheets 저장

Google Drive MCP를 사용하여 "AMP_아이템DB" 시트에 저장한다.

**시트 컬럼 구조:**

| 열 | 필드명 | 예시 |
|---|---|---|
| A | item_id | IT001 |
| B | 수집일 | 2026-05-27 |
| C | 아이템명 | 창업기업 세무 안심 점검 |
| D | URL | https://... |
| E | 한줄설명 | AI가 거래 분류, 전문가가 월1회 점검 |
| F | 고객 | 창업 1~3년차 소기업 |
| G | 고통 | 부가세 신고 제대로 한 건지 불안 |
| H | AI_80% | 거래 자동 분류, 기한 알림, 이상치 탐지 |
| I | 사람_20% | 월 1회 원격 점검, 이상치 판단, 긴급 자문 |
| J | 필요역량키워드 | 세무신고, 재무제표, 부가세, 원가분석, 자금관리 |
| K | 필요경력 | 재무회계 10년+ |
| L | 해외증거 | Bench.co MRR $2M+ [실시간검색] |
| M | 한국증거 | 삼쩜삼 1,500만 이용자 [실시간검색] |
| N | 불만증거 | 네이버카페 "세무사 비용 부담" 월 50건+ |
| O | 킬리스크 | 세무사 저가 구독 서비스 등장 시 |
| P | 데이터신뢰도 | 실시간검색 |
| Q | 소싱출처 | Indie Hackers |

**저장 방법:**
1. Google Drive MCP로 "AMP_아이템DB" 스프레드시트 검색
2. 있으면 → 기존 시트 하단에 행 추가
3. 없으면 → 새 스프레드시트 생성 + 헤더 행 입력 + 데이터 입력
4. MCP 접근 불가 시 → CSV 파일로 생성하여 제공 + 수동 입력 안내

## 매칭 모드

의뢰자가 자신의 경력·경험·핵심역량을 제시하면,
Google Sheets의 "AMP_아이템DB"에서 매칭되는 아이템을 찾아 제시한다.

### 입력 받기

의뢰자에게 3가지를 물어본다 (이미 제공된 경우 생략):

```
1. "어떤 일을 몇 년 하셨나요?" (경력)
   예: "물류관리 20년", "중학교 수학교사 15년"
   
2. "가장 자신 있는 것은 무엇인가요?" (핵심역량)
   예: "재고 최적화", "학생 눈높이 설명", "거래처 설득"
   
3. "현장에서 나만 느끼는 감각이 있다면?" (마지막 한 끗)
   예: "재고 수치만 보면 어디서 누수 나는지 바로 안다"
```

### 매칭 실행

1. 의뢰자 입력에서 키워드를 추출한다
   - "물류관리 20년, 재고 최적화" → 키워드: 물류, 재고, 입출고, 관리, 최적화
   
2. Google Sheets "AMP_아이템DB"의 J열(필요역량키워드)에서 
   해당 키워드와 겹치는 아이템을 검색한다
   
3. 매칭되는 아이템을 상위 3건 추출한다
   - 키워드 겹침 수가 많은 순
   - 필요 경력 수준이 의뢰자 경력 이하인 것

4. 시트 접근이 안 되면, 이전 대화에서 수집한 아이템 목록을 
   기억에서 검색하여 매칭한다

## 사용 예시

**수집 모드:**
> "이번 주 아이템 스카우트 해줘"
> "Indie Hackers도 포함해서 아이템 발굴해줘"
> "교육 관련 아이템만 집중 소싱해줘"

**매칭 모드:**
> "물류관리 20년인데 뭘 만들 수 있어?"
> "중학교 수학교사 15년 경력에 맞는 아이템 찾아줘"
> "거래처 영업 25년, 견적서 작성이 특기인데 매칭해줘"
> "의뢰자가 세무사 사무실에서 10년 일했대. 매칭해줘"

**DB 현황:**
> "진단DB 현재 몇 건이야?"
> "시장성 검증 완료된 아이템만 보여줘"

## 주의사항

- "경남 10대 분야" 같은 고정 분류를 쓰지 않는다. 의뢰자의 실제 경력 키워드로 매칭한다.
- "해외에서 통한 것 = 한국에서도 통한다"고 가정하지 않는다. 한국 결제 인프라(네이버페이, 토스, 카카오페이)와 유통 구조에 맞게 재해석해야 한다.
- 시장성 근거를 못 찾으면 "근거 없음"으로 표시한다. 없는 데이터를 만들지 않는다.
- 킬 리스크는 반드시 포함한다. "이 아이템이 안 될 이유"를 항상 1개 이상 찾는다.
- 인디해커 벤치마크: 첫 매출까지 평균 3개월, $1K MRR까지 평균 6개월. 이보다 낙관적인 추정은 경계한다.
- Google Sheets 저장 실패 시 반드시 CSV 파일로 대안을 제공한다.
