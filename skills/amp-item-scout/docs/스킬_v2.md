# AMP Item Scout v3

## 2가지 모드

**수집 모드** — 아이템 발굴 + 시장성 근거 + DB 저장
- 트리거: "아이템 발굴", "스카우트", "DB 업데이트", "이번 주 아이템"

**매칭 모드** — 의뢰자 경력 → DB에서 맞는 아이템 제시
- 트리거: "내 경력에 맞는", "○○ 20년인데 뭘", "의뢰자 매칭"

---

## 수집 모드

### STEP 1: 아이템 발굴

**소싱 우선순위 (한국 → 해외 순서):**

1순위 — 한국 실거래 채널 (가장 현실적):
- 크몽(kmong.com): "AI" 카테고리 인기 서비스, 리뷰 수, 가격대
  → web_fetch로 개별 서비스 페이지의 리뷰 수·평점까지 수집
- 탈잉(taling.me): 인기 클래스, 수강생 수
- 숨고(soomgo.com): 프리랜서 서비스 수요, 견적 범위
- 클래스101(class101.net): 온라인 교육 트렌드

2순위 — 한국 콘텐츠 채널 (아이템 아이디어):
- 유튜브: "AI 부업 아이템 2026", "1인 창업 추천", "AI 수익화"
  주요 채널: 동기부여학과, 신사임당, 자청, 전준수, 드로우앤드류
- 네이버 블로그: "AI 1인 창업", "바이브코딩 수익화", "AI 부업"
- 브런치(brunch.co.kr): 솔로비즈니스나이트, 1인 창업 에세이

3순위 — 한국 매거진/커뮤니티 (트렌드 분석):
- 아이보스(i-boss.co.kr): 마케팅/창업 트렌드
- 캐릿(careet.net): 비즈니스 분석
- 이오플라(eopla.net): 1인 창업 매거진
- 디스콰이엇(disquiet.io): 한국판 Product Hunt

4순위 — 한국 불만/수요 발굴:
- 네이버 카페: 업종별 "이런 서비스 있었으면"
- 블라인드(blind.com): 직장인 업무 고충
- GeekNews(news.hada.io): 개발자/스타트업 커뮤니티

5순위 — 해외 소스 (글로벌 트렌드 + 매출 검증):
- Indie Hackers: 실제 MRR 공개 (시장성 근거의 핵심)
- Product Hunt: 신규 서비스 트렌드 (단, 1인 운영 가능한 것만 선별)
- TAAFT(theresanaiforthat.com): AI 도구 신규 등록
- AppSumo: 실제 판매 가격/리뷰
- MicroLaunch: 인디해커 런칭
- SaaSHub: 서비스 비교/리뷰

**소스 지정 가능:** 사용자가 "유튜브 위주로" 또는 "크몽만" 등 지정하면 해당 채널 집중.

**수집 기준:**
- 1인 또는 소규모 팀(~5명) 운영 가능
- AI 80% + 사람 20% 구조가 가능한 것
- 대형 플랫폼(Zapier, Notion 등) 자체가 아니라, 그것을 활용한 "서비스 아이템"을 찾는다

**추출 항목:**

| 필드 | 설명 |
|---|---|
| 아이템명 | 서비스 이름 |
| URL | 원본 링크 (유튜브 영상, 블로그 글, 서비스 URL) |
| 한 줄 설명 | 무엇을 하는 서비스인가 |
| 고객 | 누구의 문제인가 |
| 고통 | 고객이 실제로 하는 말 |
| AI 80% | AI가 처리하는 부분 |
| 사람 20% | 마지막 한 끗 (사람이 판단해야 하는 것) |
| 필요역량키워드 | 의뢰자 매칭용, 자유 형식, 3~7개 |
| 필요경력 | "해당 분야 10년+", "경력 무관" 등 |

**필요역량키워드 작성 규칙:**
- 고정 카테고리 없음. 의뢰자가 자기 경력을 설명할 때 쓸 실제 단어
- 예: "세무신고, 재무제표, 원가분석" / "물류, 재고관리, 입출고" / "강의, 커리큘럼설계"
- 이 키워드로 나중에 의뢰자 경력과 매칭됨

### STEP 2: 시장성 근거 수집 (상위 3~5건)

발굴된 아이템 중 상위 3~5건에 대해:

1. **해외 팔리는 증거** — 유사 서비스 + MRR/이용자 수
2. **한국 팔리는 증거** — 크몽 거래량, 유사 서비스
   → 가능하면 web_fetch로 크몽 개별 서비스 페이지 접근하여 리뷰 수 확인
3. **불만 증거** — 네이버카페/블라인드 관련 게시물
4. **킬 리스크** — 이 아이템이 안 될 이유 1가지

모든 근거에 [실시간검색] 또는 [추정] 태그 필수.
없는 데이터는 만들지 않는다. "근거 없음 — 현장 인터뷰 필요"로 표시.

### STEP 3: DB 저장

**우선순위:**
1. Notion MCP 연결됨 → "AMP_아이템DB" 데이터베이스에 직접 페이지 추가
2. Notion 미연결 → CSV 파일 생성하여 다운로드 제공
3. 어떤 경우든 화면에 정리된 결과를 출력

**Notion DB 속성 (연결 시):**
- item_id (Title)
- 수집일 (Date)
- 아이템명 (Text)
- URL (URL)
- 고객 (Text)
- 고통 (Text)
- AI_80% (Text)
- 사람_20% (Text)
- 필요역량키워드 (Multi-select 또는 Text)
- 필요경력 (Text)
- 해외증거 (Text)
- 한국증거 (Text)
- 불만증거 (Text)
- 킬리스크 (Text)
- 데이터신뢰도 (Select: 실시간검색/추정)
- 소싱출처 (Text)

### STEP 4: 결과 출력

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  AMP 아이템 스카우트 — [날짜]
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

📊 발굴: [N]건 | 시장성 검증: [N]건
📂 저장: [Notion 완료 / CSV 제공 / 화면 출력만]

━━ 시장성 검증 아이템 ━━

[1] 아이템명
    고객: ○○○ | 필요역량: ○○, ○○, ○○
    고통: "○○○"
    한 끗: ○○○
    ────
    해외: ○○○ [실시간검색]
    한국: ○○○ [실시간검색]
    불만: ○○○
    킬 리스크: ○○○

━━ 트렌드 요약 ━━
· (1줄)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

## 매칭 모드

### STEP 1: 의뢰자 정보 수집

이미 제공된 경우 생략. 부족하면 물어본다:

```
1. "어떤 일을 몇 년 하셨나요?"
2. "가장 자신 있는 것은?"
3. "현장에서 나만 느끼는 감각이 있다면?"
```

### STEP 2: DB 검색

1. 의뢰자 입력에서 키워드 추출
2. Notion DB의 "필요역량키워드" 속성에서 검색
   → Notion MCP의 search 기능 활용
3. Notion 미연결 시 → 레퍼런스 파일의 예시 아이템에서 검색

### STEP 3: 매칭 실패 시 폴백 (중요)

DB에 매칭되는 아이템이 없으면:
1. "현재 DB에 [○○] 관련 아이템이 없습니다"라고 알린다
2. **즉석으로 웹 검색**하여 의뢰자 경력에 맞는 아이템 3건을 제안한다
   - 크몽에서 유사 서비스 검색
   - 유튜브에서 "○○ AI 부업" 검색
   - Indie Hackers에서 유사 분야 검색
3. 제안된 아이템을 "DB에 등록할까요?"라고 물어본다
4. 승인 시 → Notion/CSV에 추가

### STEP 4: 매칭 결과 출력

```
━━━━━━━━━━━━━━━━━━━━━━━━━━
  [의뢰자 경력]에 맞는 아이템
━━━━━━━━━━━━━━━━━━━━━━━━━━

입력: ○○ ○년, ○○ 특기

[1] 아이템명
    고객: ○○○
    고통: "○○○"
    AI 80%: ○○○
    당신의 한 끗: ○○○
    ─────
    해외: ○○○
    한국: ○○○

→ Brad와 함께 수익 구조를 설계합니다.
━━━━━━━━━━━━━━━━━━━━━━━━━━
```

## 주의사항

- 대형 플랫폼 자체(Zapier, Notion 등)를 아이템으로 수집하지 않는다. 1인이 만들고 운영할 수 있는 "서비스"를 찾는다.
- Product Hunt 결과가 대형 도구 위주이면 건너뛰고 크몽/유튜브로 간다.
- 한국 시장 근거가 더 중요하다. 해외 MRR은 참고용, 크몽 거래량이 실전 근거다.
- "해외에서 통한 것 = 한국에서도 통한다"고 가정하지 않는다.
- 킬 리스크를 반드시 포함한다. 낙관 편향을 방지한다.
- 인디해커 벤치마크: 첫 매출 평균 3개월, $1K MRR 평균 6~18개월.
