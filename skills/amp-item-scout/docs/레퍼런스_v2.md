## 1. DB 구조

### Notion DB 속성 (Notion 연결 시)

데이터베이스명: **AMP_아이템DB**

| 속성명 | 유형 | 설명 |
|---|---|---|
| item_id | Title | 고유번호 IT001~ |
| 수집일 | Date | YYYY-MM-DD |
| 아이템명 | Text | 서비스 한 줄 이름 |
| URL | URL | 원본 링크 |
| 한줄설명 | Text | 무엇을 하는 서비스인가 |
| 고객 | Text | 누구의 문제인가 |
| 고통 | Text | 고객이 실제로 하는 말 |
| AI_80% | Text | AI가 처리하는 부분 |
| 사람_20% | Text | 마지막 한 끗 |
| 필요역량키워드 | Text | 의뢰자 매칭용 (쉼표 구분, 3~7개) |
| 필요경력 | Text | "재무회계 10년+", "경력 무관" 등 |
| 해외증거 | Text | 해외 유사 서비스 + MRR/이용자 |
| 한국증거 | Text | 한국 유사 서비스 + 거래량 |
| 불만증거 | Text | 커뮤니티 pain point |
| 킬리스크 | Text | 이 아이템이 안 될 이유 |
| 데이터신뢰도 | Select | 실시간검색 / 추정 |
| 소싱출처 | Text | 어디서 발견했나 |

### CSV 구조 (Notion 미연결 시)

위와 동일한 컬럼을 CSV로 출력. 사용자가 Google Sheets 또는 Notion에 수동 업로드.

## 3. 필요역량키워드 예시 사전

| 재무/회계/세무 | 세무신고, 재무제표, 원가분석, 부가세, 자금관리, 결산, 감사 |

## 4. 예시 아이템 카드 5건

### IT001: 창업기업 세무 안심 점검
- 고객: 창업 1~3년차 소기업
- 고통: "부가세 신고 제대로 한 건지 불안"
- AI 80%: 거래 자동 분류, 신고 기한 알림, 이상치 탐지
- 사람 20%: 월 1회 원격 점검, 이상치 판단, 긴급 자문
- 필요역량키워드: 세무신고, 재무제표, 부가세, 원가분석, 자금관리
- 필요경력: 재무회계 10년+
- 해외증거: Bench.co MRR $2M+ [실시간검색]
- 한국증거: 삼쩜삼 이용자 1,500만+ [실시간검색]
- 불만증거: 네이버카페 "세무사 비용 부담" 월 50건+
- 킬리스크: 세무사 저가 구독 서비스 등장 시

### IT002: AI 견적서·제안서 자동 생성
- 고객: 제조업·서비스업 영업 담당자
- 고통: "견적서 매번 처음부터 만든다, 실수도 잦다"
- AI 80%: 과거 견적 패턴 학습, 품목·단가·조건 자동 채움
- 사람 20%: 가격 전략 검토, 경쟁 입찰 시 차별화 포인트
- 필요역량키워드: 영업, 견적, 수주, 거래처관리, 납품, 가격협상
- 필요경력: 영업 10년+
- 해외증거: PandaDoc ARR $100M+ [실시간검색]
- 한국증거: 크몽 "견적서 작성" 서비스 다수 [실시간검색]
- 불만증거: 블라인드 "견적서 만드느라 야근" 다수
- 킬리스크: CRM에 견적 기능 기본 탑재 추세

### IT003: 멀티채널 콘텐츠 자동 생산
- 고객: SNS 마케팅 인력이 없는 자영업·소기업
- 고통: "SNS 해야 하는 건 아는데 뭘 올려야 할지 모르겠다"
- AI 80%: 카드뉴스·블로그·SNS 자동 생성, 멀티채널 변환
- 사람 20%: 업종별 톤 설정, 핵심 메시지 검수, 전략 방향
- 필요역량키워드: 콘텐츠기획, SNS운영, 마케팅, 카피라이팅, 브랜딩
- 필요경력: 마케팅/홍보 5년+
- 해외증거: Buffer, Hootsuite 수천만 사용자 [실시간검색]
- 한국증거: 크몽 숏폼 제작 거래량 전년대비 9배 증가 [실시간검색]
- 불만증거: 네이버카페 "마케팅 대행 비용 부담" 다수
- 킬리스크: 고객이 직접 ChatGPT로 만드는 시대

### IT004: 온라인 코칭 플랫폼 구축 대행
- 고객: 전문성은 있지만 디지털 역량이 없는 시니어·전문가
- 고통: "내 전문성으로 온라인 강의를 하고 싶은데 방법을 모른다"
- AI 80%: 랜딩페이지 자동 생성, 결제 연동, 수강생 관리
- 사람 20%: 가격 전략, 마케팅 퍼널 설계, 콘텐츠 구조
- 필요역량키워드: 교육기획, 커리큘럼, 온라인강의, 코칭, 퍼스널브랜딩
- 필요경력: 교육/강의 5년+ 또는 특정 분야 전문성
- 해외증거: Kajabi ARR $100M+, Teachable [실시간검색]
- 한국증거: 클래스101 누적 수강 1,000만+ [실시간검색]
- 불만증거: "온라인 강의 플랫폼 수수료 너무 높다"
- 킬리스크: 노코드 도구로 직접 구축 가능해지는 추세

### IT005: 소기업 채용 스크리닝 AI
- 고객: 직원 5~20명 규모 소기업
- 고통: "이력서 50개 왔는데 볼 시간이 없다"
- AI 80%: 이력서 자동 파싱, 직무적합도 점수화, 면접 질문 생성
- 사람 20%: 최종 3인 숏리스트 검토, 조직 적합성 판단
- 필요역량키워드: 채용, 인사관리, 면접, HR, 조직관리, 인력운영
- 필요경력: 인사/HR 7년+
- 해외증거: Manatal MRR $500K+ [추정]
- 한국증거: 원티드, 사람인 AI 매칭 도입 [실시간검색]
- 불만증거: 블라인드 "채용 스크리닝에 시간 너무 많이 든다"
- 킬리스크: 잡플랫폼이 AI 스크리닝 기본 제공 시

## 5. 매칭 예시

**입력:** "물류관리 20년, 재고 최적화가 특기"
**키워드 추출:** 물류, 재고, 관리, 최적화, 입출고
**DB 검색:** 필요역량키워드에 "물류" 또는 "재고" 포함된 아이템
**결과:** 현재 DB에 없음 → 폴백: 즉석 웹검색 → 아이템 제안 → DB 등록 여부 확인

**입력:** "세무사 사무실에서 10년"
**키워드 추출:** 세무, 세무신고, 재무, 회계
**DB 검색:** IT001(창업기업 세무 안심 점검) 매칭
**결과:** 해외증거 + 한국증거 + 킬리스크 함께 제시

## 6. 주간 운영 체크리스트

```
매주 월요일 (30분):
□ "이번 주 아이템 스카우트 해줘" 실행
□ 결과 확인 (5~10건 발굴, 3~5건 시장성 검증)
□ DB 저장 확인 (Notion 또는 CSV)
□ 필요 시 추가 소스 지정

격주 수요일 (15분):
□ 킬 테스트 1건 실행
□ 해외 소스 추가 스캔

매월 마지막 금요일 (30분):
□ 한국 불만/수요 채널 스캔
□ DB 전체 현황 확인
□ "이번 달 트렌드 요약" 생성 → 코칭 교재 반영
```
