# AMP 아이템 스카우트 — 수시 스캔 운영체계 v3

## 변경 요약

| 항목 | v2 (기존) | v3 (변경) |
|---|---|---|
| 운영 방식 | 매주/격주/월 정기 스케줄 | **"아이템 스카우트 해줘" 수시 실행** |
| 스캔 범위 | 주기별로 다른 소스 | **매회 30개 소스 전체 스캔** |
| 소요 시간 | 주 30분 + 격주 15분 + 월 30분 | **1회 실행 시 약 10~15분** |
| 검색 구조 | 소스별 개별 검색 (23회) | **5개 묶음 압축 검색 (10~12회)** |
| DB 저장 | Notion 또는 CSV | **XLSX 즉시 생성 + Notion 연동 시 자동** |

---

## 수시 스캔 실행 흐름

### 트리거 명령어
```
"아이템 스카우트 해줘"
"아이템 발굴해줘"
"DB 업데이트 해줘"
"스카우트 전체 스캔"
```

### 실행 STEP (자동 순서)

```
STEP 1: 한국 실거래 + 콘텐츠 묶음 스캔 (검색 3회)
  ↓
STEP 2: 한국 매거진 + 불만/수요 묶음 스캔 (검색 3회)
  ↓
STEP 3: 해외 소스 묶음 스캔 (검색 3회)
  ↓
STEP 4: 시장성 검증 — 상위 3~5건 (검색 2~3회)
  ↓
STEP 5: DB 통합 + 파일 생성
```

## 5개 검색 묶음 설계

### 묶음 1: 한국 실거래 (검색 2회)

| 검색 | 쿼리 | 커버 소스 |
|---|---|---|
| 1-A | `크몽 AI 인기 서비스 2026` | ① 크몽 |
| 1-B | `탈잉 숨고 클래스101 AI 인기 2026` | ② 탈잉 ③ 숨고 ④ 클래스101 |

**긁는 것:** 실제 거래 서비스, 리뷰 수, 가격대, 수강생 수

### 묶음 2: 한국 콘텐츠 (검색 2회)

| 검색 | 쿼리 | 커버 소스 |
|---|---|---|
| 2-A | `AI 1인 창업 부업 아이템 유튜브 2026` | ⑤ 유튜브 일반 |
| 2-B | `동기부여학과 히든리치 조코딩 AI 사업 최근` | ⑥ 유튜브 채널 + 히든리치 |

**보조 (필요 시):**
- `AI 1인 창업 네이버 블로그 브런치 2026` → ⑦ 블로그 ⑧ 브런치

### 묶음 3: 한국 매거진 + 커뮤니티 (검색 2회)

| 검색 | 쿼리 | 커버 소스 |

### 묶음 4: 해외 실매출 (검색 2회)

## 수시 스캔 vs 정기 스캔 비교

### 수시 스캔 (기본)
```
트리거: "아이템 스카우트 해줘"
범위: 30개 소스 전체
검색: 10~12회 (5개 묶음)
결과: 5~10건 발굴, 3~5건 시장성 검증
시간: 약 10~15분
출력: 화면 + XLSX + Notion(연결 시)
```

### 집중 스캔 (소스 지정 시)
```
트리거: "크몽만 봐줘" / "해외 소스만" / "불만 채널 스캔"
범위: 지정 소스만
검색: 2~4회
결과: 해당 채널 집중 3~5건
시간: 약 5분
```

### 킬 테스트 (별도 명령)
```
트리거: "IT001 킬 테스트 해줘" / "세무 점검 아이템 킬 테스트"
범위: Failory + 경쟁사 + 규제 검색
검색: 3~5회
결과: 킬 리스크 심화 분석 1건
```

## 운영 규칙

### 중복 방지
- 매 스캔 시 기존 DB(현재 52건)와 대조
- 유사 아이템 발견 시 → 기존 아이템에 "최신 근거 업데이트"로 처리
- 완전 신규만 새 item_id 부여

### ID 부여 규칙
- IT001~IT030: 경남 10대 분야 기본 DB (고정)
- IT031~: 신규 발굴 순번 (수시 추가)
- 소싱출처에 날짜 포함 → 언제 발견했는지 추적 가능

### 품질 기준
- 1인 또는 소규모 팀(~5명) 운영 가능
- AI 80% + 사람 20% 구조 가능
- 대형 플랫폼 자체가 아닌 "서비스 아이템"
- 한국 시장 근거가 해외보다 우선
- 킬 리스크 반드시 포함 (낙관 편향 방지)
- [실시간검색] / [추정] 태그 필수
