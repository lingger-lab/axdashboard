## 3. SERP 키워드별 점유 설계 상세

### 키워드 10개 생성 공식

`{지역}` = 선거구 지역명, `{직위}` = 시장/군수/구청장/시의원 등, `{후보명}` = 표준 표기

| # | 키워드 패턴 | 예시 |
|---|-----------|------|
| 1 | {지역}{직위} | 창원시장 |
| 2 | {지역}{직위}후보 | 창원시장후보 |
| 3 | {지역}지방선거 | 창원지방선거 |
| 4 | 2026 {지역} 지방선거 | 2026 창원 지방선거 |
| 5 | {지역}{직위} 경선 | 창원시장 경선 |
| 6 | {후보명}+{지역}{직위} | 홍길동 창원시장 |
| 7 | {지역} {핵심정책1} | 창원 교통 개선 |
| 8 | {지역} {핵심정책2} | 창원 일자리 |
| 9 | {후보명} 공약 | 홍길동 공약 |
| 10 | {지역}{직위} 후보 비교 | 창원시장 후보 비교 |

### 키워드별 점유 설계표 형식

각 키워드마다 아래 표를 작성:

```
키워드: [키워드]
검색의도: [정보/인물/정책/뉴스/영상]

| 순위 | 목표 자산 | 제작물 | 내부링크/외부레퍼런스 | 리스크 |
|------|----------|--------|---------------------|--------|
| 1순위 | 홈페이지 허브 | 정책 허브 페이지 | 공약 상세 → FAQ 링크 | LOW |
| 2순위 | 유튜브 영상 | 검색형 해설 영상 | 설명란에 허브 링크 | LOW |
| 3순위 | 뉴스 기사 | 보도자료 1건 | 보도자료에 홈페이지 URL | MED |
| 4순위 | 블로그 포스트 | 정책Q&A 글 | 허브로 내부링크 | LOW |
| 5순위 | 프로필 | 네이버 인물정보 | 공식 자산 URL 등록 | LOW |
```

## 4. 홈페이지 MVP 허브 구조 상세

### 필수 6허브 상세 설계

**허브 1: 정책(공약) 허브**
- URL: `/policy/` 또는 `/공약/`
- 구조: 메인(공약 전체 요약) + 하위 페이지(공약별 상세)
- 필수 요소: 문제 정의 → 해결 방안 → 기대 효과 → 근거/출처
- 구조화 데이터: Article
- 내부 링크: 지역현안 허브, FAQ 허브로 연결

**허브 2: 지역현안 허브**
- URL: `/issues/` 또는 `/지역현안/`
- 구조: 현안별 페이지 (교통/복지/환경/경제 등)
- 필수 요소: 현황 데이터 → 문제점 → 후보 입장 → 구체 해법
- 구조화 데이터: Article
- 내부 링크: 정책 허브, 현장 허브로 연결

**허브 3: 팩트/FAQ 허브**
- URL: `/faq/` 또는 `/팩트체크/`
- 구조: 카테고리별 FAQ (정책/이력/논란대응/일반)
- 필수 요소: 질문 + 2~3문장 답 + 근거 + 업데이트 날짜
- 구조화 데이터: **FAQPage** (최우선 적용)
- 내부 링크: 모든 허브에서 FAQ로 연결

**허브 4: 보도자료/언론 허브**
- URL: `/press/` 또는 `/보도자료/`
- 구조: 날짜순 보도자료 아카이브
- 필수 요소: 제목/날짜/본문/첨부파일
- 구조화 데이터: Article (NewsArticle 가능)

**허브 5: 일정/현장 허브**
- URL: `/activities/` 또는 `/현장/`
- 구조: 날짜순 현장 활동 기록
- 필수 요소: 날짜/장소/내용/사진/결과
- 구조화 데이터: Event

**허브 6: 영상/연설 허브**
- URL: `/videos/` 또는 `/영상/`
- 구조: 유튜브 임베드 + 자막/요약
- 필수 요소: 영상 제목/설명/자막/핵심 발언 텍스트
- 구조화 데이터: Video

## 5. 콘텐츠 생산 공장 상세

### 블로그 템플릿 4종

**① 정책Q&A형**
```
제목: [키워드] - [후보명]이 답합니다
구조: 질문(검색 의도 반영) → 핵심 답변(1문단) → 배경/근거(1~2문단) → 구체 정책(1문단) → 관련 링크
SEO: 제목에 핵심 키워드, 첫 100자에 답변 포함
```

**② 현안해설형**
```
제목: [지역] [현안] 현황과 해법 - [후보명]
구조: 현황 데이터 → 문제의 핵심 → 타 지역 사례 → 후보 해법 → 실행 계획
SEO: H2에 키워드 변형, 데이터 표/그래프 포함
```

**③ 현장스케치형**
```
제목: [날짜] [장소] 현장방문 - [후보명]
구조: 현장 상황 → 주민 의견(익명 가능) → 후보 대응 → 후속 조치
SEO: 이미지 ALT에 장소+활동 설명, 지역명 반복
```

**④ 팩트체크형**
```

```

### 유튜브 템플릿 2종

구조: 훅(15초) → 문제 정의(1분) → 해법 3가지(3~6분) → 정리/CTA(1분)

자막: 반드시 포함 (AI 보조 가능, 출처 검증 필수)

### "안 오를 때" 진단 트리

```
SERP에 우리 자산이 안 보인다
│
├─ 인덱싱 자체가 안 됨?
│  ├─ Yes → 기술 문제 (사이트맵/robots/canonical/서버 에러)
│  └─ No ↓
│
├─ 인덱싱은 됐는데 순위가 낮다?
│  ├─ 콘텐츠 부족 → 키워드 매칭 부족, 콘텐츠 양/질 보강
│  ├─ 권위 부족 → 보도자료/외부 언급/프로필 보강
│  └─ 경쟁 과다 → 롱테일 키워드로 우회, 영상/뉴스 탭 공략
│
├─ 부정/혼동 콘텐츠가 상위?
│  ├─ Yes → 팩트/FAQ 허브 강화, 방어 콘텐츠 발행
│  └─ No ↓
│
└─ 정책 위반 가능성?
   └─ 검색 스팸 판정/패널티 → 즉시 점검, 위반 요소 제거
```
