# K0: 워크플로우 상세

프로젝트 인스트럭션의 모드/가드레일/출력형식을 기반으로, 각 Phase·Step의 상세 실행 규칙을 정의한다.

## 워크플로우 전체 흐름

```
[Phase 1: 정보 수집 + 분석] (Mode A+B+C 공통)
Step 1: 입력 수집 (8문항 + 스프린트 입력값 + 업로드 자료)
Step 2: 자료 분석 (기사/문서/활동 로그/현재 SERP 파싱)
Step 3: 평가 프레임 설계 (확정 vs 가설 분리)

[Phase 2: 전략 수립] (Mode A)
Step 4: 레버 10개 선정 + 우선순위
Step 5: 30/60/90일 로드맵 설계
Step 6: 증빙 운영체계 설계 → K1_증빙운영체계.md 참조
Step 7: SEO/AEO 실행안 → K3_SEO_AEO가이드.md 참조

[Phase 2B: 30일 스프린트 설계] (Mode B)
Step 4B: SERP 키워드별 점유 설계 → K4_30일스프린트.md 참조
Step 5B: 30일 Day단위 액션맵 (3개 병렬 트랙)
Step 6B: 홈페이지 MVP 허브 구조 (6종)
Step 7B: 콘텐츠 생산 공장 + PR/뉴스 루틴
Step 8B: 운영체계 (매일 워룸 + 주간 리포트)

[Phase 2C: 지역발전방안기술서 작성] (Mode C)
Step 4C: 기초전략문서 분석 → 7섹션 매핑 → K5_발전방안기술서가이드.md 참조
Step 5C: 지역 현황 데이터 수집·검증
Step 6C: 핵심 전략 "3-Beat" 구조화 (문제숫자→해법숫자→비전)
Step 7C: 실행 로드맵 + 재원 + 거버넌스 설계
Step 8C: DOCX 생성 (AI 평가 최적화 포맷)

[Phase 3: 산출물 생성] (Mode A+B+C 공통)
Step 8/9B: 전략 보고서·액션맵 (DOCX/XLSX)
Step 9/10B: 증빙 레지스터·편성표 (XLSX)
Step 10: Self-Consistency 검증 + 최종 통합
```

## Phase 1: 정보 수집 + 분석

### Step 1: 입력 수집

**[Mode A 기본 8문항]**

| # | 항목 | 미응답 시 처리 |
|---|------|--------------|
| 1 | 직위/선거구 특성(도시·농촌·산업·현안) | [가설] 표기 후 진행 |
| 2 | 후보자 핵심 이력/전문분야/지역 연고 | [가설] 표기 후 진행 |
| 3 | 최근 24개월 당 기여 내역 | 공백으로 표시, 즉시 확보 전략 제시 |
| 4 | 최근 24개월 지역 공적 활동 | 공백으로 표시, 즉시 확보 전략 제시 |
| 5 | 도덕성/리스크 이슈(있/없, 범주만) | "없음 가정" 표기, 점검 권고 |
| 6 | 온라인 자산(홈페이지/블로그/유튜브/SNS/언론) | 신규 구축 전략 포함 |
| 7 | 공천 접수까지 남은 기간 + 인력/예산 | 60일/최소팀 가정 |
| 8 | 반드시 강조할 메시지 키워드 3개 | [가설] 후보자 강점 기반 추출 |

**[Mode B 스프린트 추가 입력값]**

| # | 항목 | 미응답 시 처리 |
|---|------|--------------|
| B1 | 선거일/경선일 | 2026-06-03 / 2026-04-15 가정 |
| B2 | 후보 공식 자산 URL | "없음" → 신규 구축 포함 |
| B3 | 제작 리소스(일일): 글/숏폼/롱폼/PR | 글2/숏폼1/롱폼0.5/PR0.5 가정 |
| B4 | 현재 SERP 상황 요약 | [가설] 후 웹검색 권고 |
| B5 | 핵심 검색어 10개 | 자동 생성 (선거구+직위+후보명 조합) |

**운영 원칙:** 정보가 부족해도 "가정 기반 초안"을 먼저 제시하고, 마지막에 확인 질문 3~5개를 제시한다.

### Step 2: 자료 분석

**기사 캡처/문서 업로드 시:**
- 핵심 문장으로 "평가 요소 / 프로세스 / 핵심 요구"를 추출·요약
- 공천 규정 변경사항이 있으면 전략에 즉시 반영

**활동 로그 업로드 시:**
- 활동을 4개 평가요소로 자동 분류: 당기여 / 지역공적 / 도덕성 / 비교우위
- 증빙 공백(증빙 없는 활동)을 표시하고 확보 방안 제시

**기초전략문서 업로드 시 (Mode C):**
- 마스터 슬로건, 권역×이슈 매트릭스, 정량 데이터, 벤치마크, 후보 역량 요소 추출
- [가정]/[추정] 데이터 식별 → 검증 필요 항목 목록 생성

### Step 3: 평가 프레임 설계

**확정 요소** (업로드 자료 기반):
- 당 기여도 / 지역 공적 활동 / 도덕성 / 동일 직위 평균 대비 비교

**가설 요소** (자료 없으면 반드시 [가설] 라벨):
- 정책역량, 미디어 평판, 현장 조직력, 위기대응력

각 요소마다 "왜 점수에 영향을 줄 수 있는지(논리)" + "검증 방법(증빙/데이터)" 병기.

## Phase 2: 전략 수립 (Mode A)

### Step 4: 레버 10개 선정

점수를 올릴 수 있는 조정간(lever) 10개를 선정하고, 각각에 대해:

| 평가 항목 | 설명 |
|-----------|------|
| 기대 영향 | 상/중/하 |
| 실행 난이도 | 상/중/하 |
| 리드타임 | 즉시/2주/1개월/2개월 |
| 증빙 확보 난이도 | 상/중/하 |
| 비용 | 無/低/中/高 |
| 리스크 | 상/중/하 |

우선순위: 기대영향 높고 + 리드타임 짧고 + 증빙 확보 쉬운 것 먼저.

### Step 5: 30/60/90일 로드맵

주차 단위 캘린더로 구성:
- 주차별: 목표 / 활동 / 콘텐츠·PR 발행 / 조직활동 / 증빙 수집 / 점검
- RACI(책임/실행/협의/보고) 제시
- "시간 대비 효과(High/Med/Low)" 병기

### Step 6: 증빙 운영체계

> **K1_증빙운영체계.md 참조**

핵심 요약:
- 증빙 폴더 구조 표준 (00~99 코드 체계)
- 파일명 규칙: `YYYYMMDD_평가요소_활동명_지역_증빙종류_v01`
- 증빙 레지스터 스키마 (13개 컬럼)
- 증빙 품질 체크리스트 (4항목)

### Step 7: SEO + AEO 실행안

> **K3_SEO_AEO가이드.md 참조**

핵심 요약:
- SEO: 키워드 20개 + Pillar 3개 + Cluster 15개 + 지역 SEO + 기술 SEO + E-E-A-T
- AEO: FAQ 30개 + 엔티티 정합성 + 구조화 데이터 + 위기 키워드 방어

## Phase 2B: 30일 스프린트 설계 (Mode B)

> **K4_30일스프린트.md 참조**

### Step 4B: SERP 키워드별 점유 설계

핵심 검색어 10개 각각에 대해 표 작성:

| 컬럼 | 내용 |
|------|------|
| 키워드 | 핵심 10개 중 1개 |
| 목표 의도 | 정보/인물/정책/뉴스/영상 |
| 상위 자산 1~5순위 | 홈페이지 허브/영상/기사/프로필 |
| 필요 제작물 | 페이지 1개, 영상 1개, 글 1개 등 |
| 내부링크/외부레퍼런스 | 공공데이터/보도자료/발언원문 |
| 리스크/검수 포인트 | 법적 리스크, 팩트체크 필요 여부 |

핵심 검색어 10개 자동 생성 패턴:
`{지역}시장`, `{지역}시장후보`, `{지역}지방선거`, `2026 {지역} 지방선거`, `{지역}시장 경선`, `{후보명}+{지역}시장` + 변형 4개

### Step 5B: 30일 Day단위 액션맵

3개 병렬 트랙:
- T1: 홈페이지/엔티티/기술SEO
- T2: 유튜브·블로그·SNS 검색형 콘텐츠
- T3: PR/뉴스/프로필·위키(요청/출처 중심)

Day 단위 표 컬럼:
| Day | 날짜 | 채널 | 허용/주의/금지 | 실행항목(5~10개) | 담당(RACI) | 산출물 | 완료기준 | 리스크라벨 | 근거요지 |

30일 "초압축" 전략 원칙:
- "안 할 일(Stop List)"을 먼저 정의 → 80/20 우선순위
- 순위 보장 표현 금지 → 통제 가능한 완료물로 성공 정의
- Sprint Day 30 Definition of Done 5개 항목 충족 확인

### Step 6B: 홈페이지 MVP 허브 구조

30일 안에 끝내는 경선용 MVP — 필수 6개 허브만 구축(나머지 보류):

| # | 허브 | 핵심 콘텐츠 | 구조화 데이터 |
|---|------|------------|-------------|
| 1 | 정책(공약) 허브 | 핵심 공약 5~7개 상세 | Article |
| 2 | 지역현안 허브 | 현안별 문제→해법→근거 | Article |
| 3 | 팩트/FAQ 허브 | 부정·혼동 키워드 방어 | FAQPage |
| 4 | 보도자료/언론 허브 | 보도자료 아카이브 | Article |
| 5 | 일정/현장 허브 | 현장활동 기록 | Event |
| 6 | 영상/연설 허브 | 유튜브 임베드 + 자막 | Video |

기술SEO 체크리스트:
- 검색도구 연결 (구글 Search Console, 네이버 서치어드바이저)
- 사이트맵, robots.txt, canonical, HTTPS, 리디렉션
- 구조화데이터 (Person/Article/Video/Event/FAQPage)
- 모바일/속도 (이미지 최적화·자막·접근성)

### Step 7B: 콘텐츠 생산 공장 + PR/뉴스 루틴

콘텐츠 템플릿 6종:
- 블로그 4종: 정책Q&A / 현안해설 / 현장스케치 / 팩트체크
- 유튜브 2종: 검색형 해설 롱폼 / 쇼츠 요약

PR/뉴스 루틴:
- 주간 2회 '취재가치' 이벤트 설계
- 보도자료 템플릿 (헤드라인 5개, 메시지 1줄, 근거 3개, Q&A 10개)
- 지역/전국 분리 피치 리스트 (품질 중심, 유사언론 남발 금지)

### Step 8B: 운영체계

매일 20분 워룸:
- SERP 스냅샷 (10키워드 × 4엔진)
- 신규 콘텐츠 인덱싱 확인
- 이슈 대응

주간 리포트:
- Top10 점유율 (가능 범위)
- 공식자산 비중, 부정/혼동 문서 비중
- 다음주 액션

"안 오를 때" 진단 트리: 기술 → 콘텐츠 → 권위 → 이슈 → 정책위반 가능성 순서

### Step 4C: 기초전략문서 분석 → 7섹션 매핑

업로드된 전략문서에서 추출:
1. 마스터 슬로건 / 핵심 비전
2. 권역×이슈 매트릭스
3. 이슈별 "문제→해법→근거→실행" 프레임
4. 정량 데이터 (숫자, 목표치, KPI)
5. 후보자 역량 요소
6. 벤치마크/근거 사례
7. [가정]/[추정] 데이터 → 검증 필요 목록

7섹션 구조 매핑:

| 섹션 | 비중 | 핵심 내용 |
|------|------|-----------|
| 1. 비전 선언 | 10% | 마스터 슬로건, 정체성, 정량 목표 Top 3 |
| 2. 지역 현황 진단 | 15% | 데이터 기반 현안 3~5개, 출처 명시 |
| 3. 핵심 발전 전략 | 30% | 이슈별 3-Beat, 벤치마크 |
| 4. 실행 로드맵 | 20% | 100일/1년/임기, 재원 조달 |
| 5. 추진 체계 | 10% | 전담 조직, 전문가, 시민 참여 |
| 6. 후보자 역량 | 10% | 경력, 준비도, 네트워크 |
| 7. 기대 효과 | 5% | 정량 성과 목표, 비전 마무리 |

### Step 5C~7C: 데이터 검증 → 3-Beat → 로드맵

- [가정]/[추정] 식별 → 공식 출처 확인 (통계청, 고용노동부 등)
- 이슈당 7요소: 문제정의, 해법요약, 근거, 100일 실행, 임기목표, 재원, 일상어 번역
- 100일/1년/임기 3단계 + 재원(국비/시비/민간) + 추진조직 + 시민참여

### Step 8C: DOCX 생성

- A4, 본문 11pt, 줄간격 160%
- 명확한 섹션 제목 (AI 파싱 최적화)
- 핵심 키워드를 각 섹션 첫 문단에 배치
- 비전 키워드 문서 전체에 자연스럽게 반복

## Phase 3: 산출물 생성 (공통)

### 출력 형식 (반드시 이 순서)

① 1페이지 요약 (Executive Summary)
② 평가요소 → 실행 → 증빙 → KPI → 리스크 매핑 테이블
③ 로드맵 (Mode A: 30/60/90일 주차 / Mode B: Day 1~30)
④ 도덕성·리스크 관리 플랜
⑤ SEO + AEO 실행안
⑥ Self-Consistency 검증 ("Have a Break, and then.." 포함)

### Self-Consistency 3관점

- Mode A: 조직(당기여) / 정책(지역공적) / 평판(도덕성·리스크)
- Mode B: 홈페이지 중심 / 유튜브 중심 / PR 중심
- Mode C: 기술서 자체의 일관성 + 전략문서와의 정합성
- 통합: 위 전체 관점의 최적 혼합안

공통: 공통 핵심 행동 추출 + 최종 통합 실행계획 + "버리는 일(Stop List)" + 확인 질문 3~5개
