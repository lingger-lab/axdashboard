# SEO + AEO 실행 가이드

## SEO 실행안

### 핵심 키워드 전략

키워드군 20개를 아래 조합으로 선정한다:

| 유형 | 조합 패턴 | 예시 |
|------|-----------|------|
| 지역+후보자명 | {선거구} {후보자명} | "○○구 홍길동" |
| 지역+직위 | {선거구} {직위} 후보 | "○○구 시의원 후보" |
| 지역+정책 | {선거구} {핵심정책} | "○○구 교통 개선" |
| 후보자+전문분야 | {후보자명} {전문분야} | "홍길동 도시재생" |
| 지역+현안 | {선거구} {현안 키워드} | "○○구 재개발 현황" |

### Pillar-Cluster 구조

**Pillar 3개:** 후보자 비전 / 지역 정책 / 활동 성과

**Cluster 15개 (Pillar당 5개):**

| Pillar | Cluster 예시 | 의도 | 추천 포맷 |
|--------|-------------|------|-----------|
| 후보자 비전 | 인사말/소개 | 정보형 | 소개 페이지 |
| 후보자 비전 | 핵심 공약 | 정보형 | 정책 상세 페이지 |
| 후보자 비전 | 이력/경력 | 정보형 | 프로필 페이지 |
| 후보자 비전 | 언론 보도 | 신뢰형 | 보도자료 아카이브 |
| 후보자 비전 | FAQ | 정보형 | FAQ 페이지 |

| ... | ... | ... | ... |

### 지역 SEO

1. **NAP 통일**: 이름/주소/연락처를 모든 채널에서 동일하게 표기
2. **지도/프로필 정비**: 네이버 플레이스, 구글 비즈니스 프로필 (사무소)
3. **지역 매체·단체 페이지 활용**: 합법적 협력 범위 내 링크/소개

### 기술 SEO

1. **사이트 구조**: 정책 / 성과 / FAQ / 보도자료 4대 섹션
2. **페이지 속도**: 모바일 Core Web Vitals 기준 충족
3. **메타 태그**: title, description에 핵심 키워드 포함
4. **내부 링크**: 정책→성과→보도자료 간 상호 링크
5. **이미지 ALT**: 모든 활동 사진에 설명적 ALT 텍스트
6. **보도자료 아카이브**: 날짜순 정리, 구조화 데이터 적용

### E-E-A-T 강화

- **Experience**: 현장 활동 기록, 실명 인터뷰, 직접 체험 콘텐츠
- **Expertise**: 전문분야 이력, 자격/수료, 정책 분석 능력 증명
- **Authoritativeness**: 공식 문서, 기관 협업 기록, 언론 보도
- **Trustworthiness**: 출처 명시, 팩트체크 가능한 수치, 투명한 정보 공개

## AEO (답변엔진 최적화) 실행안

### FAQ 30개 생성 가이드

각 FAQ는 아래 형식으로 작성:

```
Q: [질문]
A: [2~3문장 답변]
근거(증빙): [증빙 종류 - 예: 봉사확인서, 활동보고서]
업데이트: YYYY-MM-DD
```

**FAQ 카테고리 분포 (30개):**
| 카테고리 | 개수 | 예시 |
|---------|------|------|
| 후보자 소개/이력 | 5개 | "○○ 후보는 어떤 사람인가요?" |
| 핵심 공약/정책 | 8개 | "○○구 교통 문제 해결 방안은?" |
| 지역 활동 성과 | 7개 | "지역에서 어떤 봉사 활동을 했나요?" |
| 당 활동/기여 | 5개 | "당에서 어떤 역할을 했나요?" |
| 도덕성/신뢰 | 3개 | "재산 공개는 어떻게 하나요?" |
| 논란 대응 (방어) | 2개 | "[이슈]에 대한 입장은?" |

### 엔티티 정합성

후보자명·직함·지역·핵심정책 키워드 표기를 **모든 채널에서 100% 동일**하게 유지:

| 요소 | 통일 표기 예시 |
|------|--------------|
| 후보자명 | "홍길동" (한글 풀네임) |
| 직함 | "국민의힘 ○○구 시의원 예비후보" |
| 지역 | "○○시 ○○구" |
| 핵심정책 | "3대 핵심공약" (동일 명칭 사용) |

### 구조화 데이터

가능하면 아래 Schema.org 마크업 적용:
- `FAQPage`: FAQ 페이지
- `Person`: 후보자 프로필
- `Organization`: 캠프/사무소
- `Article`: 보도자료, 블로그 포스트

### 위기 키워드 방어

논란 가능 질문에 대해:
1. **사실 기반 Q&A** 작성 (추측 금지)
2. **타임라인** 정리 (날짜별 사실관계)
3. 검색 결과에서 방어 콘텐츠가 상위에 노출되도록 SEO 최적화
4. 정기 업데이트 (최소 월 1회)

### 콘텐츠 운영 루틴

| 빈도 | 콘텐츠 유형 | 핵심 요소 |
|------|------------|-----------|
| 주 2~3건 | 성과 리포트 | 숫자 + 현장사진 + 확인자 |
| 주 1건 | 정책 설명 | 문제→해결→기대효과 구조 |
| 주 1회 | Q&A 요약 | 주민 질문 + 답변 + 후속조치 |
| 월 1회 | 종합 활동 보고 | 월간 KPI + 하이라이트 |
