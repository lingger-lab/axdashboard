# Claude 프로젝트 설정 안내서
## 국민의힘 AI 공천 고득점 플래너

---

## 파일 구성 (총 8개)

```
📁 프로젝트 파일
├── 01_프로젝트_인스트럭션.md  ← Custom Instructions에 붙여넣기
│
└── 📁 knowledge/              ← Project Knowledge에 업로드
    ├── K0_워크플로우_상세.md      (워크플로우 Phase/Step 전체)
    ├── K1_증빙운영체계.md        (증빙 폴더/파일명/레지스터)
    ├── K2_KPI사전.md             (4축 18개 KPI 정의)
    ├── K3_SEO_AEO가이드.md       (SEO/AEO 전략·키워드·구조화데이터)
    ├── K4_30일스프린트.md        (Day 1~30 액션맵·SERP·허브·콘텐츠)
    ├── K5_발전방안기술서가이드.md  (7섹션 구조·AI 고득점 원칙·3-Beat)
    └── K6_실무템플릿.md           (현장보고서·민원·FAQ·보도자료)
```

---

## 설정 방법 (5분)

### Step 1: 프로젝트 생성
1. claude.ai 접속 → 좌측 사이드바 → **"Projects"** 클릭
2. **"Create project"** 클릭
3. 프로젝트 이름: `국민의힘 AI 공천 고득점 플래너` (또는 원하는 이름)

### Step 2: Custom Instructions 설정
1. 프로젝트 설정 화면에서 **"Custom instructions"** 영역을 찾음
2. `01_프로젝트_인스트럭션.md` 파일을 열어서 **전체 내용을 복사 → 붙여넣기**
3. 저장

### Step 3: Project Knowledge 업로드
1. 프로젝트 설정 화면에서 **"Add knowledge"** (또는 "Knowledge" 영역) 클릭
2. `knowledge/` 폴더 안의 7개 파일을 **모두 선택하여 업로드**:
   - K0_워크플로우_상세.md
   - K1_증빙운영체계.md
   - K2_KPI사전.md
   - K3_SEO_AEO가이드.md
   - K4_30일스프린트.md
   - K5_발전방안기술서가이드.md
   - K6_실무템플릿.md
3. 업로드 완료 확인

### Step 4: 테스트
프로젝트 내에서 새 대화를 시작하고 아래 테스트 프롬프트를 입력:

**Mode A 테스트:**
```
창원시장 공천 전략을 세워주세요. 
후보: 홍길동, 의사 출신, 30년 지역 거주, 365경제연구원 운영
```

**Mode B 테스트:**
```
창원시장 경선 후보의 30일 온라인 상위노출 스프린트를 설계해주세요.
```

**Mode C 테스트:**
```
아래 전략문서를 바탕으로 지역발전방안기술서를 작성해주세요.
[전략문서 업로드 또는 내용 붙여넣기]
```

## 사용법 요약

| 하고 싶은 일 | 말하는 방법 |
|-------------|------------|
| 공천 전략 전체 수립 | "공천 전략 세워줘" / "평가 점수 높이는 방법" |
| 증빙 관리 체계 | "증빙 운영체계 만들어줘" / "증빙 레지스터" |
| 30일 상위노출 | "상위노출 스프린트" / "SEO 전략" / "검색 최적화" |
| 홈페이지 허브 설계 | "홈페이지 구조" / "MVP 허브" |
| 발전방안기술서 | "발전방안 기술서 작성" / "지역현안 발전계획" |
| 종합 전략 | "종합 전략" / Mode A+B+C 키워드 혼합 |
| 전략 보고서 DOCX | "전략 보고서 파일로 만들어줘" |
| 증빙 레지스터 XLSX | "증빙 레지스터 엑셀로" |

## 자료 업로드 활용

| 업로드 자료 | 활용 방법 |
|------------|-----------|
| 기사 캡처 / 공천 규정 | 평가 프레임에 반영, 규정 변경 즉시 적용 |
| 활동 로그 | 당기여/지역공적/도덕성/비교우위 자동 분류 |
| 기초전략문서 (캠프 전략) | 발전방안기술서 자동 변환 |
| 경쟁자 정보 | 차별화 전략 수립 |

## 주의사항

- **Custom Instructions 글자 제한**: claude.ai 프로젝트의 인스트럭션 필드에 글자 수 제한이 있을 수 있습니다. 만약 전체가 안 들어가면, 가드레일과 모드 정의 부분을 우선 넣고, 나머지는 K0 Knowledge에서 처리합니다.
- **Knowledge 파일 순서**: K0~K6 순서로 업로드하면 관리가 편합니다.
- **업데이트**: 공천 규정이 변경되면 해당 Knowledge 파일만 수정하여 재업로드하면 됩니다.
