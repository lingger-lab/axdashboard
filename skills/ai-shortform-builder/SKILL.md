# AI시대예감 숏폼 변환기 (스킬 3)

완성된 「AI시대예감」 글을 유튜브 숏츠용 9:16 영상 제작 패키지로 바꾼다.

## 이 스킬의 정확한 범위

```
[스킬 1: 큐레이터]  뉴스 후보 3건 선별            (ai-news-curator)
[스킬 2: 빌더]     선택된 1건 → 발행 가능한 글     (ai-newsletter-builder)
        ↓ 글 확정
[스킬 3: 숏폼]     완성된 글 → 영상 제작 패키지     ← 지금 이 스킬
```

**입력:** 빌더가 만든 완성 글 (또는 발행된 「AI시대예감」 글 1편)
**출력:** 씬별 내레이션 대본 + 자막 + 이미지 프롬프트 + CapCut 가이드
**대상:** 유튜브 숏츠 중심, 45~60초

## 영상 포맷 (확정)

- **형태:** 텍스트+이미지 슬라이드형. **얼굴 안 나옴.**
- **소리:** 내레이션 음성 (NotebookLM 또는 TTS) + 화면 자막
- **비율:** 9:16 세로
- **길이:** 45~60초 (씬 5~7개)
- Brad 유튜브 채널 포맷(화면 녹화+음성, 노 페이스)과 일치

## 툴체인 (Brad 확립 워크플로우에 얹음)

```
이 스킬 → [내레이션 대본 + 자막 + 이미지 프롬프트 + 편집 가이드]
                ↓
        이미지: Nano Banana (9:16 JPEG, 한글 텍스트 직접 삽입 가능)
        음성:   NotebookLM 내레이션 (또는 TTS)
        편집:   CapCut (자막·전환·음성 합성)  ← 김동현 또는 Brad
```

> 이 스킬은 '제작 재료'를 만든다. 실제 이미지 생성·음성·편집은
> 위 툴에서 사람이 한다. 스킬이 영상 파일을 직접 만들지 않는다.

## 핵심 원칙: 글을 요약하지 말고 '재구성'하라

숏폼은 글의 축소판이 아니다. **다른 매체**다.
- 글은 천천히 읽지만, 숏폼은 3초 안에 안 잡으면 넘어간다.
- 글의 논리 순서 ≠ 숏폼의 순서. 숏폼은 *결론·충격을 먼저* 던진다.
- 글의 핵심 1개만 살린다. 욕심내면 다 놓친다.

그래서 변환의 본질은 "줄이기"가 아니라 **"가장 강한 한 점을 골라 앞세우기"**다.
빌더 글에서 [💬 Brad의 시선]의 가장 날선 한 문장 = 숏폼의 심장.

## 훅 우선 구조 (45~60초)

```
[씬 1 · 0~3초] 훅 ★ 가장 중요
  - 글의 결론/반전/충격 수치를 질문이나 단언으로 먼저
  - 예: "AI로 월 천만원? 근데 0.2%만 성공합니다."
  - 절대 "안녕하세요" "오늘은~"으로 시작 안 함

[씬 2~3 · 3~20초] 맥락
  - 무슨 뉴스인지 한 입 크기로. 사실만, 빠르게

[씬 4~5 · 20~45초] Brad의 시선 ★ 차별점
  - 코멘트의 핵심. "30년 본 사람"의 한마디
  - 여기가 이 숏폼이 남들과 다른 이유

[씬 6 · 45~55초] 질문/여운
  - 독자에게 던지는 물음. 댓글 유도

[씬 7 · 55~60초] CTA
  - "집출 오픈채팅" 또는 "구독" 한 줄. 짧게
```

## 씬 설계 규칙

| 항목 | 규칙 |
|------|------|
| 씬당 길이 | 5~10초. 한 씬 = 한 메시지 |
| 내레이션 | 구어체. 문어체 금지. 한 씬 1~2문장 |
| 자막 | 내레이션의 핵심 키워드만. 전체 받아쓰기 ❌ |
| 자막 길이 | 한 화면 1줄, 최대 2줄. 9:16 좁은 폭 고려 |
| 이미지 | 씬당 1장. Nano Banana 9:16 |
| 수치 처리 | 빌더의 '미검증' 플래그 계승 — 과장 수치는 자막에 단정 ❌ |

## 이미지 프롬프트 규칙 (Nano Banana 9:16)

- 비율: 9:16 세로 명시
- 스타일: 미니멀, 따뜻한 톤. Brad 브랜드(콘크리트 틈 민들레, 새벽빛, 모니터 불빛)
- 인물: 얼굴 없이. 뒷모습/손/실루엣
- **한글 텍스트 삽입 가능** (Nano Banana는 한글 렌더링 양호) — 핵심 키워드를 이미지에 직접 넣어도 됨
- 씬 내용과 직결. 추상적 장식 이미지 금지

상세 프롬프트 형식은 references/scene-format.md 참조.

## 워크플로우

```
[Step 1] 글 분석     — 핵심 1점 + 가장 날선 문장 추출 (= 숏폼 심장)
[Step 2] 훅 설계     — 결론/충격을 0~3초로. 3안 만들어 최선 택
[Step 3] 씬 분할     — 5~7씬, 훅 우선 구조로 배치
[Step 4] 씬별 작성   — 내레이션 + 자막 + 이미지 프롬프트
[Step 5] 과장 점검   — 미검증 수치 단정 제거
[Step 6] 출력        — references/scene-format.md 템플릿
```

## 사용 시나리오

```
A: "방금 그 글로 숏폼 만들어줘"  → 직전 빌더 출력을 입력으로 변환
B: "이 AI시대예감 글로 숏츠 [붙여넣기]"  → 붙인 글을 입력으로 변환
C: "훅만 3개 다시"  → Step 2만 재실행 (Brad가 훅 고르게)
```
