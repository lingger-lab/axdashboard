# AX 숏폼 증폭기 (스킬 5 · ③ 증폭)

생성된 콘텐츠를 숏폼으로 재포장해 **유입을 키운다.** 새로 만들지 않는다 —
이미 만든 글의 가장 인용 가능한 한 조각을 영상 훅으로 변환한다.

## 경계: 이 스킬은 '재포장'만 한다

```
[스킬 2~4]    완성된 글 (블로그/에세이/뉴스레터)
        ↓
[스킬 5: 숏폼]  가장 강한 4기둥 통증 한 문장 → 15~30초 숏폼  ← 지금 이 스킬
              (새 콘텐츠 X, 재포장 O)
```

**입력:** 완성된 글 1건 (특히 블로그 결론 문장 재활용)
**출력:** 9:16 숏폼 스크립트 + 자막 타이밍 + 썸네일 문구 + 이미지 프롬프트

## 핵심: 통증 한 조각 = 훅

블로그 전체가 아니라 **4기둥 통증 한 문장**만 떼어낸다.
예: "직원 안 뽑고 버티는 법" → 첫 3초 훅 → 30초 안에 핵심 1개 → 오픈채팅 CTA.

## 채널 전략 (기존 유지)

| 채널 | 타깃 | 포맷 |
|---|---|---|
| YouTube Shorts / Reels | 2030 (2차) | 빠른 컷, 자막 중심 |
| 페이스북 영상 | 4050 (1차, 4,500망) | 차분한 내레이션 |

포맷: 화면녹화 + 음성내레이션(얼굴 노출 없음), BOYA BY-M3, Bandicam 캡처, CapCut 편집.

## 자동화 경계 (유튜브 인간 병목)

- 대본·자막·썸네일 문구 = **반자동** (스킬이 초안)
- 목소리·최종 편집 = **사람** (김동현 / Brad)
- "키워드→영상 100개 자동"은 불가. 그건 유튜브가 거르는 AI 슬롭이다.

## 워크플로우

```
[Phase 0] 입력 확인        — 완성된 글 + 4기둥 태그
[Phase 1] 통증 추출        — 가장 강한 한 문장 (블로그 결론 재활용)
[Phase 2] 훅 설계          — 첫 3초에 통증 (질문 or 충격 사실)
[Phase 3] 본론 압축        — 30초 안에 핵심 1개만
[Phase 4] CTA             — 오픈채팅 유도 (자막 + 음성)
[Phase 5] 자막 타이밍 + 썸네일 문구 + 이미지 프롬프트
```

## Phase 2: 훅 설계

첫 3초가 전부다. 통증을 질문으로: "직원 한 명 값, 매달 아깝지 않으세요?"
또는 충격 사실로: "이 일, 사람 없이 AI가 합니다." 설명 금지, 통증 직격.

## ⚠️ 검증 단계에서는 보류

숏폼(증폭)은 인용·전환이 증명된 뒤 합류. 증폭은 *증명된 것*을 키우는 도구지,
검증의 일부가 아니다. 최소 루프(큐레이터+블로그)부터 돌린다.
