# 선거 경선전략 실행문서 생성 스킬

## 개요

이 스킬은 **모든 지역, 모든 후보**에 적용 가능한 경선 승리 전략 실행문서를 생성합니다.
핵심 프레임워크는 8-STEP 구조이며, 맘다니(Mamdani) 벤치마킹 방법론을 기반으로 합니다.

**산출물**: 전문적으로 포맷된 .docx 파일 (50~80페이지급 전략 실행문서)

## 워크플로우

### Phase 1: 입력 수집 (파일 추출 우선 원칙)

**핵심 원칙: 업로드 파일에서 최대한 추출하고, 추출 불가능한 변수만 사용자에게 질문한다.**

사용자가 후보 분석 파일(제3의눈, 후보 리포트, 프로필 등)을 업로드하면:

#### Step 1. 파일 분석 및 변수 자동 추출

업로드 파일을 읽고 아래 변수를 최대한 추출합니다.
파일 형식(.docx, .pdf, .txt, .md 등)에 관계없이 텍스트를 추출한 후 분석합니다.

**자동 추출 대상 (파일에서 높은 확률로 추출 가능한 변수):**
- 후보명, 선거구/지역명, 정당명, 선거 유형
- 경쟁자 이름, 경쟁자별 강점/약점/예상 지지율
- 후보 경력/배경, 학력, 주요 이력
- 현재 지지율 추정치 (언급된 경우)
- 핵심 정책/공약 목록
- 권역 구분 및 권역별 이슈 (언급된 경우)
- 선거구 인구, 당원 수 등 정량 데이터

**추출 방법:**
```
1. 파일 텍스트 전체를 읽는다
2. 후보명 → 문서에서 가장 빈번하게 등장하는 인물명 또는 제목에서 추출
3. 경쟁자 → 비교 분석 섹션, 표, 대립 구도에서 추출
4. 정책 → 공약, 비전, 핵심 메시지 섹션에서 추출
5. 수치 → 지지율, 인구, 당원수 등 숫자+단위 패턴에서 추출
6. 지역 → 지명, 권역, 선거구명에서 추출
```

#### Step 2. 추출 결과 확인 + 미추출 변수 질문

추출된 변수를 사용자에게 표로 보여주며 확인을 받고,
**추출되지 않은 변수만** 카테고리별로 묶어 질문합니다.

**질문 형식 예시:**
```
📋 파일에서 다음 정보를 추출했습니다. 확인해 주세요:

| 항목 | 추출 값 | 확인 |
|------|---------|------|
| 후보명 | 홍길동 | ✅/수정 |
| 선거구 | 대전시장 | ✅/수정 |
| 경쟁자 | 김갑수(25%), 이을순(20%) | ✅/수정 |
| ... | ... | ... |

❓ 다음 항목은 파일에서 확인되지 않아 입력이 필요합니다:
1. 경선 방식은 무엇입니까? (예: 당원ARS 50% + 시민ARS 50%)
2. 캠페인 기간은 며칠입니까? (기본값: 30일)
3. 핵심 슬로건은 무엇입니까?
...
```

#### Step 3. 변수 확정

사용자 확인/수정을 반영하여 모든 변수를 확정합니다.
확정 후 바로 Phase 2(문서 생성)로 진행합니다.

#### 전체 변수 목록

아래 표의 모든 변수를 파일에서 추출 시도하고, 추출 실패한 항목만 질문합니다.
질문 시 가능한 한 기본값이나 선택지를 제공하여 사용자 부담을 최소화합니다.

### Phase 2: 문서 생성

`references/document-structure.md`를 읽고 8-STEP 프레임워크에 따라 문서를 생성합니다.

#### 생성 순서

1. **docx 스킬 참조**: `/mnt/skills/public/docx/SKILL.md`의 docx-js 패턴을 따름
2. **구조 참조**: `references/document-structure.md`의 8-STEP 템플릿 로드
3. **변수 치환**: 수집된 변수를 템플릿에 적용
4. **AI 생성 구간**: 
   - 승리조건 역산 모델의 구체적 수치 계산
   - 경쟁자별 1:1 차별화 프레임 생성
   - 자기소개 스크립트 (10초/30초/120초/3분) 생성
   - 숏폼 스크립트 10개 생성
   - FAQ 답변 TOP5 생성
   - 카피라이팅 6-Layer 아키텍처 구성
   - 30일 실행 캘린더 생성
   - 리스크 시나리오 및 방어 전략 생성
5. **테이블 생성**: 62개 이상의 전략 테이블 자동 구성
6. **문서 출력**: .docx 포맷으로 최종 문서 생성

### Phase 3: 검증 및 수정

문서 생성 후:
1. 사용자에게 파일 전달
2. 피드백 수집
3. 수정 사항 반영

## 8-STEP 프레임워크 요약

문서를 생성할 때 반드시 아래 8개 STEP을 모두 포함해야 합니다.
각 STEP의 상세 구조는 `references/document-structure.md`를 참조하세요.

| STEP | 제목 | 핵심 산출물 |
|------|------|------------|
| 1 | 승리조건 역산 모델 | 승리공식, 3개 시나리오, 역전경로, 표이동 시뮬레이션 |
| 2 | 이슈브레이킹 & 인지도 폭발 전략 | 5대 전술, 바이럴 체계, KPI |
| 3 | 이미지메이킹 설계 | 정체성 1문장, 상황별 매트릭스, 자기소개 스크립트 |
| 4 | 카피라이팅 체계 | 6-Layer 아키텍처, 변형패턴, 숏폼 스크립트 |
| 5 | 이슈선점 전략 | 권역별 이슈, 정량적 해법, 9-매트릭스 |
| 6 | 경쟁우위 프레이밍 | 본선경쟁력, 1:1 차별화, 공격방어 |
| 7 | N일 실행 캘린더 | 주차별 로드맵, 트랙별 액션, KPI |
| 8 | 리스크 관리 & 준법 | TOP5 리스크, 대응 프로토콜, 준법 체크 |

추가 섹션:
- 교차검증 (6개 관점)
- 부록 1: 캠프 전원 암기 카드
- 부록 2: 주간 점검 체크리스트

## 문서 포맷 규격

- **용지**: A4
- **헤더**: "극비 · CONFIDENTIAL"
- **표지**: 슬로건 + 후보명 + 부제 + 선거정보 + 벤치마킹 대상 + 작성일
- **Heading 1**: STEP 제목 (8개 + 교차검증 + 부록)
- **Heading 2**: 세부 섹션 (a), (b), (c)...
- **테이블**: 각 STEP마다 핵심 결론 박스(💡), 맘다니 공식 적용 박스(🗽), 즉시 실행 액션 박스(🎯) 포함
- **테이블 스타일**: 헤더행 회색 배경, 1pt 테두리, 셀 패딩 적용

## 핵심 원칙

이 스킬의 모든 산출물은 다음 원칙을 지켜야 합니다:

1. **일반화**: 특정 후보에 종속되지 않는 프레임워크. 변수 치환으로 어떤 후보든 적용 가능
2. **정량적**: 모든 목표와 KPI는 구체적 숫자로 제시
3. **실행 가능성**: 각 STEP마다 "즉시 실행 액션 TOP3" 포함
4. **벤치마킹 기반**: 성공 사례(맘다니 등)의 공식을 체계적으로 적용
5. **3초 기억 원칙**: 후보의 핵심 메시지가 3초 안에 전달되도록 설계
6. **준법**: 선거법 준수 가이드 반드시 포함
