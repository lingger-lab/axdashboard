## 섹션별 추출 지침

### [섹션 1] 헤더 메타데이터

**추출 대상 패턴:**
```
후보자: [이름]
지역: [도명] [시군구]
선거모드: [모드 설명]
경선발표 예상일: [날짜] (D-[N])
본선일: [날짜] (D-[N])
생성일: [날짜]
버전: [버전]
```

**분석모드 결정 규칙:**
- `당원N:여론N` 형식 포함 → `primary_multi`
- `양자경선` 또는 `양자구도` → `primary_bilateral`  
- `본선` 또는 `일반선거` → `general_bilateral`
- 판별 불가 → `primary_multi` (기본값, 사용자에게 확인)

**campaign_days 추출:**
- `(D-19)` → 19일
- `D-`가 없으면 `election_date - today`로 계산
- 여전히 불명확하면 30일 기본값 사용

**party_name 추출:**
- `국민의힘`, `더불어민주당`, `민주당`, `개혁신당`, `조국혁신당` 키워드 검색
- 없으면 `election_type_raw`에서 추출 시도

---

### [섹션 2] 경쟁자 지지율

**1차 추출 패턴 (괄호형):**
```
[이름](X.X%) 또는 [이름] X.X%
```
예: `박동식(36.9%)`, `정대웅 11.8%`

**오탐 필터링 규칙:**
- 후보 본인 이름 제외
- 숫자/영문만으로 된 오탐 제거
- `트랙`, `방향` 등 명사는 인물명이 아니므로 제외
- 지지율이 0% 또는 100%이면 제외
- **"현직"** 키워드가 앞에 있으면 `is_incumbent: true` 표시

**경쟁자별 추가 추출:**
- 강점: `조직력`, `뉴스노출`, `친화력` 등 키워드 근처 텍스트
- 약점: `경험 부재`, `비전 부재` 등
- 예상 슬로건: 경쟁자 섹션에서 따옴표 내 텍스트

**현재 후보 지지율:**
- `[후보명] [N.N%]` 또는 `후보명(N.N%)` 패턴으로 추출
- `경선 핵심 구도` 섹션에서 추출 우선

---

### [섹션 3] 승리공식 테이블

**추출 대상:** `승리 공식` ~ `핵심 프레임` 사이 테이블

**추출 필드:**
- 트랙명 (여론조사 트랙, 당원투표 트랙, 변화진영 통합, 합산 목표)
- 목표 (퍼센트)
- 핵심 전술
- 현재→목표 (N%→M%)

---

### [섹션 4] 슬로건 & 핵심 프레임

**슬로건 추출 패턴:**

| 항목 | 패턴 |
|------|------|
| 메인 슬로건 | `메인 슬로건` 다음 따옴표/「」 안 텍스트 |
| 서브 슬로건 | `서브 슬로건` 다음 따옴표/「」 안 텍스트 |
| 3초 차별화 | `3초 차별화` 다음 따옴표/「」 안 텍스트 |

**따옴표 없는 경우:**  
`메인 슬로건` 키워드 이후 2칸 이상 공백 전까지 텍스트 추출

**핵심 프레임 테이블 추출:**  
`핵심 프레임` ~ `브랜딩` 사이의 표에서 `축 / 후보자 / 경쟁자` 3열 추출

### [섹션 5] 권역별 이슈

**추출 대상:** `유권자 관심 의제 맵` 섹션

**권역 패턴:**
```
N권역 (지역명)   X.X%   Y.Y%   의제1, 의제2, 의제3   대응공약
```

**추출 필드:**
- `region`: `N권역`
- `area_name`: 괄호 안 지명
- `population_ratio`: 인구비 퍼센트
- `negative_rate`: 시정 부정평가율 (`최고`/`최저` 태그 포함)
- `is_priority`: 부정평가 최고이거나 인구비 최대이면 true
- `issues`: `①②③` 앞 텍스트 최대 3개
- `solution`: 유해남/후보 대응 공약 컬럼

**권역 없는 경우 (Fallback):**  
`sub_regions: []`로 설정하고 STEP 5를 **이슈 기반 구조**로 전환 (지역별 → 정책분야별)

### [섹션 6] 시나리오 수치

**추출 대상:** `표 이동 시뮬레이션` 또는 `시나리오` 테이블

**S1/S2/S3 패턴:**
```
보수적 시나리오 (S1)   [상대%]   [후보%]   [±격차%p]
중도 시나리오 (S2)     [상대%]   [후보%]   [±격차%p]
낙관적 시나리오 (S3)   [상대%]   [후보%]   [±격차%p]
```

**현재 상황:**
```
현재 다자구도   [상대%]   [후보%]   [-격차%p]
```

**양자구도 승리 조건:**  
`중도 시나리오 (S2)` 에서 후보 > 상대이면 → 역전 가능 플래그 설정

---

### [섹션 7] 즉시 실행 액션 TOP10

**추출 대상:** `즉시 실행 액션 TOP` 섹션

**행 패턴:**
```
[우선순위번호]   [액션 설명]   [시한]   [담당]   [KPI]
```

**추출 필드:**
- `priority`: 1~10 정수
- `action`: 액션 설명 (최대 60자)
- `deadline`: 시한 (D-N형식 또는 기간)
- `owner`: 담당 (팀명)
- `kpi`: KPI 지표

**주의:** 헤더행("우선순위 / 액션 / 시한...") 및 섹션 제목행 제외

---

### [섹션 8] 비판적 자기검증 (EXT-8)

**추출 대상:** `EXT-8. 비판적 자기검증` 섹션

**행 패턴:**
```
[①~⑤]   [영역]   [현재방향]   [편향리스크]   [대안]   [판정]
```

**추출 필드:**
- `area`: 영역명 (브랜딩, 이미지, 카피, 공약, 경력 활용)
- `current_direction`: 현재 방향
- `bias_risk`: 편향 리스크
- `recommendation`: 대안 A 또는 판정 권고
- `verdict`: 판정 (유지/부분수정/전면수정)
