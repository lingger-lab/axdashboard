# AI시대예감 뉴스레터 빌더 (스킬 2)

집출(zipchul.cloud)의 「AI시대예감」 코너에 발행할 뉴스레터 글을 만드는 스킬.

## 이 스킬의 정확한 범위 (경계가 핵심)

전체 시스템은 3개 스킬로 분리되어 있다. 이 스킬은 **가운데 조각**이다.

```
[스킬 1: 큐레이터]  매일 9시 뉴스 3건 추출 → Brad에게 전달   (이 스킬 아님)
        ↓ Brad가 1건 선택 + 코멘트 입력
[스킬 2: 빌더]     선택된 1건 + 코멘트 → 발행 가능한 글       ← 지금 이 스킬
        ↓ 글 확정
[스킬 3: 숏폼]     완성된 글 → 9:16 숏폼 스크립트 + 이미지 프롬프트  (이 스킬 아님)
```

**입력:** 뉴스 1건(링크 또는 제목+출처) + Brad의 코멘트(한 줄~한 문단)
**출력:** 「AI시대예감」 발행용 글 1편 (블로그 본문 + 카드형 요약 + 메타데이터)
**발행 리듬:** 주 3회 (매일 아님 — 아래 "발행 철학" 참조)

## 핵심 철학: 코멘트가 심장이다

뉴스 요약은 누구나(어떤 AI나) 한다. 대체 불가능한 단 하나의 자산은
**경남 30년 + AI를 통과한 59세 기획자 Brad의 관점**이다. ("경험 × AI = 증폭")

그래서 이 스킬의 무게중심은 "뉴스 요약"이 아니라 **"Brad 코멘트의 증폭"**에 있다.
브런치 스킬의 "인간 씨앗"과 같은 원리 — Brad의 코멘트가 곧 씨앗이고,
스킬은 그 씨앗을 발행 가능한 글로 키운다. 코멘트를 *대체*하지 않고 *확장*한다.

> Brad 코멘트가 있으면 코멘트를 중심으로 글을 구성한다.
> 코멘트가 없으면 뉴스 자체의 시사점을 Brad의 톤으로 분석한다.
> (코멘트가 있는 글이 더 차별화되지만, 없어도 발행 가능한 글을 만든다.)

## 발행 철학: 왜 주 3회인가

"정기적 = 매일"이 아니다. AI 수익모델에서 *논평할 가치가 있는* 뉴스는 매일 나오지 않는다.
매일을 강제하면 평범한 뉴스에 억지 코멘트가 붙고, 그건 브랜드 희소성을 깎는다.
큐레이터(스킬 1)는 매일 후보를 던지되, **발행은 Brad가 고른 주 3회만** 한다.
침묵할 권리가 곧 신뢰다. → 이 스킬은 "발행하기로 한 1건"에만 호출된다.

## 워크플로우

```
[Phase 0] 입력 확보       — 뉴스 + 코멘트 확인 (없으면 요청)
[Phase 1] 뉴스 소화        — 링크 fetch / 사실 검증 / 카테고리 분류
[Phase 2] 코멘트 증폭      — Brad 씨앗 → 관점 단락으로 (★ 무게중심)
[Phase 3] 글 조립          — 본문 + 카드 요약 구조화
[Phase 4] AI 냄새 제거 검증 — 7항목 (references/anti-ai-checklist.md)
[Phase 5] 출력             — 발행용 패키지 + 메타데이터
```

## Phase 0: 입력 확보

요청이 들어오면 두 가지가 있는지 확인한다:
1. **뉴스** — 링크, 또는 (제목 + 출처)
2. **Brad 코멘트** — 이 뉴스에 대한 Brad의 생각 한 줄 이상

코멘트가 없으면 — 뉴스 원문(있는 경우)과 제목을 기반으로 Brad의 톤·관점으로
소상공인·중소기업 관점의 시사점을 분석하여 글을 구성한다.
코멘트가 있으면 — 코멘트를 씨앗으로 확장하는 기존 방식 그대로.

큐레이터(스킬 1)가 뉴스를 던질 때 "Brad라면 던질 반론 3개"를 함께 줬다면,
Brad는 0에서 쓰지 않고 그중 하나에 *반응*하면 된다. 코멘트 품질이 올라간다.

## Phase 1: 뉴스 소화

- 링크가 있으면 `web_fetch`로 본문 확인. 제목만 있으면 `web_search`로 사실 확인.
- **사실 검증**: 수익 금액·기간·주체가 과장/미확인이면 글에 단정적으로 옮기지 않는다.
  ("월 1,000만원"류 주장은 "~라고 주장한다 / ~라는 사례가 소개됐다"로 처리)
- **카테고리 분류** (6종, 아이콘은 references/format-spec.md):
  📡 AI트렌드 / 🚀 창업 / 📢 마케팅 / 💰 정부자금 / 🔧 도구 / ⌨️ 바이브코딩
- 뉴스 핵심을 **2~3문장**으로 압축 (저작권: 원문 인용 금지, 전면 의역, 15자 이상 직접인용 금지).

## Phase 2: 코멘트 증폭 또는 자체 관점 분석 ★ (이 스킬의 무게중심)

**코멘트가 있을 때:** Brad의 씨앗 코멘트를 **3~5문장의 관점 단락**으로 키운다.
**코멘트가 없을 때:** 뉴스 원문과 제목을 기반으로 Brad의 톤과 시각(경남 30년 기획자, 소상공인 관점)으로 **자체 관점 단락**을 구성한다. 규칙:

| 원칙 | 내용 |
|------|------|
| 씨앗 보존 | Brad가 쓴 표현/핵심 단어를 단락 안에 살려둔다. 매끈하게 덮어쓰지 않는다 |
| 관점 우선 | 뉴스 재설명이 아니라 "그래서 1인 창업자에게 뭔데"로 전개 |
| 지역성 | 가능하면 경남/4050/현장 감각으로 착지 (억지로는 금지) |
| 겸손한 확신 | "정답"이 아니라 "30년 본 사람의 시선"으로. 단정보다 질문으로 여는 게 낫다 |
| 길이 | 뉴스 요약 < 코멘트 단락. 코멘트가 더 길고 무거워야 한다 |

> Brad 코멘트가 평범하거나 짧아도, 스킬이 *대신 의견을 지어내지 않는다.*
> 씨앗을 키우되 새로운 주장을 창작해 넣지 않는다. 확장은 하되 날조는 안 한다.

## Phase 3: 글 조립

`references/format-spec.md`의 두 산출물을 만든다:
1. **블로그 본문** — 「AI시대예감」 발행용 (제목 + 도입 + 뉴스 요약 + Brad 코멘트 + CTA)
2. **카드형 요약** — 카톡/SNS 미리보기용 압축 버전 (기존 GeekNews 스타일)

CTA 블록은 `references/format-spec.md`의 고정 포맷 사용
(크몽 서비스 + 집출 오픈채팅, 책 2권). 3편 중 2편에만 강한 CTA, 1편은 가볍게.

## Phase 4: AI 냄새 제거 검증

`references/anti-ai-checklist.md`의 7항목으로 검사 → 오염 문장 교체.
뉴스레터는 에세이보다 정보성이 높으므로 "고유 감각"보다 **"관점의 날" 보존**에 집중.

## Phase 5: 출력

`references/format-spec.md`의 최종 출력 템플릿을 따른다.
메타데이터(카테고리, 출처, 발행 추천 요일, 태그)와 퇴고 가이드를 포함한다.

```
시나리오 A: 링크 + 코멘트 동시
  Brad: "이 뉴스로 뉴스레터 [링크] — 코멘트: 결국 경험 있는 사람이 이긴다"
  → fetch → 소화 → 코멘트 증폭 → 조립 → 검증 → 출력

시나리오 B: 코멘트 누락
  Brad: "이 뉴스로 AI시대예감 써줘 [링크]"
  → Phase 0 코멘트 요청 → (Brad 응답) → 진행

시나리오 C: 큐레이터 연계
  Brad: "어제 3번 뉴스로 발행, 두 번째 반론으로 가자"
  → 해당 뉴스 + 큐레이터가 제시했던 반론을 씨앗으로 → 진행
```
