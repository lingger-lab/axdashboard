# AX 뉴스레터 빌더 (스킬 2 · ② 생성 > 인식)

「AI시대예감」 뉴스레터 글을 만드는 스킬. 퍼널에서 잠재고객의 **인식**을 담당한다 — "AI가
내 일과 무슨 상관?"을 "아, 이게 내 인건비/생산성 얘기구나"로 바꾼다.

## 경계

```
[스킬 1: 큐레이터]  신호 + 4기둥 태그 → Brad 1건 선택 + 씨앗
        ↓
[스킬 2: 뉴스레터]  선택 1건 + 씨앗 → 인식형 발행글        ← 지금 이 스킬
        ↓ (선택) 같은 글을 AI검색용으로 → 스킬 3
```

**입력:** 신호 1건(+기둥 태그) + Brad 코멘트(씨앗)
**출력:** 「AI시대예감」 발행글 1편 (본문 + 카드 요약 + 메타)

## 핵심 철학: 코멘트가 심장, 4기둥이 척추

뉴스 요약은 누구나 한다(Perplexity가 이미 해준다). 대체 불가 자산은 **경남 30년 + AI를
통과한 59세 Brad의 관점**이다("경험 × AI = 증폭"). 무게중심은 "요약"이 아니라 "코멘트 증폭".

여기에 더해, 모든 글은 **4기둥 중 하나에 착지**한다. 척추 없는 코멘트는 흩어진다 —
Brad의 한마디를 *어떤 사장님 통증*에 연결하는지가 글의 골격이다.

> ⚠️ Brad 코멘트 없으면 글을 완성하지 않는다. 없으면 Phase 0에서 먼저 요청한다.

## 표면 언어 규칙 (격상의 함정 회피)

- 제목·첫 문장 = **고객 언어**만 ("직원 안 뽑고 버티려면?")
- "AX전환" 개념어 = **글당 1회**, 후반부에 "이걸 통틀어 AX전환이라 부른다"로 회수
- 컨설턴트 보고서체 금지. Brad는 "기관"이 아니라 "직접 해본 59세 사장"이다.

## 워크플로우

```
[Phase 0] 입력 확보       — 신호 + 씨앗 확인 (없으면 요청)
[Phase 1] 신호 소화        — fetch / 사실 검증 / 4기둥 확정
[Phase 2] 코멘트 증폭      — 씨앗 → 관점 단락 (★ 무게중심)
[Phase 3] 통증 연결        — 코멘트를 해당 기둥의 고객 통증에 포갬
[Phase 4] 글 조립          — 본문 + 카드 요약
[Phase 5] AI냄새 제거 + 개념 회수 1회 확인
[Phase 6] 출력             — 발행 패키지
```

## Phase 0: 씨앗 요청 (코멘트 없을 때)

```
이 신호에 대한 Brad의 한마디를 주세요. 글의 심장입니다.
🗣 직관 / 🗣 반론 / 🗣 연결(경남 경험) / 🗣 질문(독자에게) 중 하나면 충분합니다.
한 줄이면 됩니다. 나머지는 제가 키웁니다.
```

## Phase 3: 통증 연결 (AX 격상의 실체)

씨앗을 그 신호의 기둥 통증에 연결한다. 예: 인건비 기둥이면 — Brad의 한마디를
"그래서 직원 한 명 값이 아까운 사장님이라면…"으로 자연스럽게 착지시킨다.
개념(AX)은 Brad 머릿속, 통증(고객언어)은 글 표면.

## Phase 5: 발행 전 체크

- AI냄새 7항목 (상투어·과장·균일한 문장리듬 제거)
- 4기둥 착지 1개로 명확한가
- "AX전환" 개념어가 정확히 1회만 등장하는가
- 끝 CTA = "30분 1:1"(오픈채팅), 개념어로 끝내지 않음
