# ENTERLABS 견적서 자동생성 스킬

## 역할
ENTERLABS의 견적서를 DOCX 파일로 자동 생성합니다.
사용자의 자연어 입력을 구조화된 견적 데이터로 변환하고,
Word에서 바로 편집, 서명, PDF 저장 가능한 파일로 출력합니다.

---

## ★ 공급자 기본값 (ENTERLABS 고정 — 사용자에게 묻지 않음)

아래 정보는 매번 자동으로 입력됩니다. 절대 사용자에게 재확인 요청 불필요.

```
상호:       ENTERLABS (엔터랩스)
대표자:     김동현
사업자번호: 405-02-46113
주소:       경남 김해시 대청로26번안길 25-8, 202호(관동동)
이메일:     hello@enterlabs.ai
```

---

## Step 1 — 사전 정보 수집 (견적서 생성 전 반드시 수행)

견적 요청이 들어오면 파일 생성 전에 아래 체크리스트로 필요한 정보를 먼저 수집합니다.

### 1-A. 메시지에서 자동 추출 가능한 항목 (있으면 그대로 사용)
| 항목 | 예시 |
|------|------|
| 고객사명 | 디에스메카 |
| 담당자 | 황은수 대표 |
| 프로젝트명 | 모터드라이브 PLC프로그램 공급 |
| 견적 항목 및 금액 | 품목명, 수량, 단가 |

### 1-B. 자동 추출 불가 항목 → 사용자에게 한 번에 질문

메시지에 누락된 정보가 있으면 아래 형식으로 한 번에 정리해서 물어보세요.
여러 번 나눠서 묻지 말고, 한 번의 메시지로 모든 항목을 요청합니다.

**질문 템플릿 예시:**
```
견적서 작성을 위해 아래 정보를 알려주세요 😊

[필수]
1. 고객사명:
2. 담당자 성함 / 직책:
3. 프로젝트(솔루션)명:
4. 견적 항목 (품목명 / 수량 / 단가):

[선택 — 없으면 생략 가능]
5. 고객사 사업자번호:
6. 고객사 주소:
7. 고객사 연락처:
8. 고객사 이메일:
9. 특이사항 / 결제조건 (기본값 사용 시 생략):

[날인 도장]
10. 도장 이미지 파일을 첨부해 주세요 (없으면 텍스트로 대체).
    지원 형식: PNG, JPG / 권장 크기: 80x80 ~ 120x120px 투명 배경
```

### 1-C. 기본값 적용 규칙
| 항목 | 미입력 시 기본값 |
|------|----------------|
| 견적일 | 오늘 날짜 |
| 유효기간 | 견적일 + 30일 |
| 통화 | KRW (원) |
| 상태 | 초안 (Draft) |
| 부가세 | VAT 10% 별도 |
| 결제조건 | 계약금 50% / 잔금 50% (납품 시) |
| 날인 이미지 | [날인] 텍스트 박스로 대체 |

## Step 3 — DOCX 파일 생성

### 3-1. 생성 도구
`/mnt/skills/user/enterlabs-quote/assets/make_quote_template.js` 를 복사한 뒤
DATA 섹션을 수집한 정보로 채워 실행합니다.

### 3-2. 수행 순서
```bash
# 1. 템플릿 복사
cp /mnt/skills/user/enterlabs-quote/assets/make_quote_template.js /home/claude/make_quote.js

# 2. DATA 섹션을 str_replace로 수정

# 3. 실행
cd /home/claude && node make_quote.js

# 4. 검증
python3 /mnt/skills/public/docx/scripts/office/validate.py /mnt/user-data/outputs/ENTERLABS_[번호].docx

# 5. 전달
present_files
```

### 3-3. DATA 섹션 채우기 예시

```javascript
const DATA = {
  quoteNo:   'EL-20260319-0001',
  quoteDate: '2026-03-19',
  validDate: '2026-04-18',
  currency:  'KRW (원)',
  status:    '초안 (Draft)',

  client: {
    name:    '디에스메카',       // 필수
    contact: '황은수 대표',      // 필수
    bizno:   '',
    address: '',
    phone:   '',
    email:   '',
  },

  project: {
    title: '모터드라이브 PLC프로그램 공급',
    desc:  '모터드라이브 PLC프로그램 공급에 관한 견적입니다.',
  },

  items: [
    { cat: '개발', desc: '모터드라이브 PLC프로그램 공급', qty: 1, unit: '식', price: 2900000 },
  ],

  notes: [
    '견적 유효기간: 발행일로부터 30일',
    '납품 일정: 계약 후 협의',
    '결제 조건: 계약금 50%, 잔금 50% (납품 시)',
    '본 견적금액은 VAT 별도입니다',
    '견적 범위 외 추가 업무는 별도 협의',
  ],

  // 날인 이미지: 업로드 시 경로 입력, 없으면 null
  stampImagePath: null,
  // 업로드된 경우 예시: '/mnt/user-data/uploads/stamp.png'
};
```

### 3-4. 날인 이미지 처리 규칙
- 사용자가 PNG/JPG 업로드 → `stampImagePath: '/mnt/user-data/uploads/파일명'`
- 이미지 없음 → `stampImagePath: null` (자동으로 [날인] 텍스트 표시)
- 권장 사양: 80~120px 정사각형, 투명 배경 PNG

## Step 4 — Markdown 요약 (빠른 확인용)

"간단히", "대략", "이 정도면?" 표현이 있을 때만 사용합니다.

```markdown
## 견적서 — ENTERLABS × {{client_name}}

**프로젝트**: {{project_name}}  
**견적일**: {{quote_date}} | **유효기간**: {{valid_date}}까지

| # | 구분 | 항목 | 수량 | 단위 | 단가 | 금액 |
|---|------|------|------|------|------|------|
| 1 | 개발 | ... | 1 | 식 | 2,900,000 | 2,900,000 |

**공급가액**: ₩ X,XXX,XXX  
**부가세 (10%)**: ₩ XXX,XXX  
**합계금액**: **₩ X,XXX,XXX**

> 결제조건: 계약금 50% / 잔금 50% (납품 시)  
> 본 견적은 발행일로부터 30일간 유효합니다.
```

## 품질 체크리스트

- [ ] 견적번호 형식: `EL-YYYYMMDD-XXXX`
- [ ] 고객사명(공급받는자)이 왼쪽 첫 번째 칸
- [ ] 공급자(ENTERLABS) 정보가 오른쪽 두 번째 칸에 자동 입력
- [ ] 금액이 0이 아닌 실질적 값
- [ ] VAT 별도 명확히 표기
- [ ] 합계 계산 정확 (소계 + VAT = 합계)
- [ ] validate.py 검증 통과

```

  │  날인 이미지 업로드됐으면 경로 삽입

```
