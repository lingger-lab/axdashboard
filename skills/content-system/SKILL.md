# 콘텐츠 시스템 스킬 v3.0

브런치 에세이 생성 + 5채널 변환 배포를 하나의 워크플로우로 통합.
"인간 씨앗 한 줄 → 브런치 에세이 1편 + 5채널 콘텐츠 + 이미지 프롬프트 7종"

---

## 실행 모드 (3가지)

사용자 요청에 따라 자동 판별:

| 요청 키워드 | 모드 | 실행 범위 |
|------------|------|-----------|
| "브런치 써줘", "에세이 써줘" | A. 브런치 전용 | Phase 0~3만 |
| "채널 배포해줘", "네이버용으로" | B. 변환 전용 | Phase 4~6만 (에세이 원본 필요) |
| "전체 배포", "오늘 콘텐츠 돌려줘" | C. 통합 | Phase 0~6 전체 |
| "브랜드 설계", "계정 세팅" | D. 브랜드 | brand-voice-guide.md 로드 |
| "글감 추천", "캘린더" | E. 캘린더 | content-calendar.md 로드 |

## Phase 0: 인간 씨앗 수집

에세이 생성 요청이 들어오면 **반드시 먼저** 사용자에게 요청한다:

```
오늘 에세이의 "인간 씨앗"을 주세요.
AI가 만들 수 없는, 오늘 Brad만의 재료입니다.

아래 중 하나만 주셔도 됩니다:

🌱 오늘의 장면 — "아까 이런 일이 있었다"
   예) "동현이가 '형, 이거 왜 이렇게 만들었어요?' 하길래 말문이 막혔다"

🌱 오늘의 감각 — 지금 보이는/들리는/느껴지는 것
   예) "비가 오는데 베란다 창문에 물방울이 흘러내린다"

🌱 오늘의 한 문장 — 카카오톡 메모에 적어둔 생각
   예) "프롬프트를 잘 쓰는 게 결국 상대를 잘 이해하는 거 아닌가"

🌱 오늘의 감정 키워드 — 단어 하나도 좋습니다
   예) "허탈" / "의외로 뿌듯" / "묘하게 설렘"

한 줄이면 충분합니다. 이 씨앗이 에세이의 심장이 됩니다.
```

**씨앗 유형별 배치:**

| 씨앗 유형 | 에세이 내 역할 | 배치 위치 |
|-----------|---------------|-----------|
| 장면 | 중심 에피소드 | 본문 중간 (현실 경험 구간) |
| 감각 | 첫 장면 또는 마무리 | 도입부 또는 결말 |
| 한 문장 | 통찰/제목 후보 | 통찰 구간 또는 제목 |
| 감정 키워드 | 전체 톤 결정 | 에세이 전체에 스며들게 |

**씨앗 거부 시:**
1. 메모리/대화에서 Brad 근황 단서를 찾는다
2. 없으면 `content-calendar.md`에서 글감 3개 제안
3. 씨앗 없이 쓸 경우 빈칸 표기: `[🌱 여기에 실제 경험을 넣어주세요]`

## Phase 1: 뉴스 연동 (자동 실행)

**매번 자동으로** 실행:

```
web_search: "AI 뉴스 오늘" (현재 연도 포함)
```

씨앗 내용에 따라 추가 검색:
```
web_search: "[씨앗 관련 키워드] 뉴스 오늘"
```

**선별 기준:** Brad 세계관(경험×AI, 중년 적응, 지역성)과 연결 가능 + 4050 독자 체감 가능 + 비기술적

**뉴스 삽입 3가지 방식:**
1. 도입부 훅 — 뉴스로 시작 → 씨앗 장면으로 전환
2. 중간 대비 — 경험 서술 중 "마침 오늘 이런 뉴스가..." 삽입
3. 마무리 여운 — 씨앗+뉴스 엮어 질문으로 마무리

> 뉴스가 씨앗과 자연스럽게 연결되지 않으면 **넣지 않는다.**

## Phase 2: 브런치 에세이 생성

### 에세이 유형 (6가지)

씨앗에서 자동 판단. 상세 구조는 `essay-templates.md` 참조.

| 유형 | 적합 주제 | 핵심 감정 |
|------|----------|-----------|
| 인간경험형 | 실제 체험, 일상 에피소드 | 공감, 위로 |
| 시행착오형 | 실패담, 학습 과정 | 용기, 연대감 |
| 통찰형 | 깨달음, 관점 전환 | 놀라움, 성찰 |
| AI철학형 | AI 시대 의미, 인간 역할 | 불안+희망 |
| 뉴스반응형 | 시사 이슈+개인 경험 | 현실감, 각성 |
| 짧은몰입형 | 감정 한 조각, 순간 포착 | 여운, 잔상 |

### 에세이 공통 구조

```
[제목] 검색형: 키워드 포함 / 팬형: 감성 후킹
[첫 3줄] 장면 묘사 또는 질문. 절대 "요즘~" "최근~"으로 시작 안 함
[감정 몰입] 솔직한 감정
[현실 경험] ★ 인간 씨앗 핵심 구간
[뉴스 연결] 자연스러운 삽입 (해당 시)
[통찰] "그때 알았다" (거창하지 않게)
[인간적 흔들림] 확신 없는 솔직함
[마무리] 질문 또는 여운. "~합시다" "~하세요"로 끝내지 않음
[CTA] funnel-strategy.md 참조. 3편 중 2편에만.
```

### 문체 핵심 규칙

상세: `brand-voice-guide.md` 참조. 요약:

| 항목 | 규칙 |
|------|------|
| 문장 | 15~25자 위주. 간혹 5자로 리듬 |
| 단락 | 2~4문장. 5문장 이상 금지 |
| 톤 | 독백+대화 혼합. "~했다" "~일까" |
| 금지 | "여러분", "꼭", "반드시", "성공", "월N억", "결국", "사실" |
| 필수 | 구체적 시간, 감각 묘사, 자기 의심 |
| 길이 | 1,500~2,500자 |

## Phase 3: AI 냄새 제거 검증

본문 작성 후, 출력 전 반드시 실행.

### 7대 징후 체크

```
[1] 균일한 문장 길이 → 연속 3문장 비슷하면 오염 → 한 문장을 극단적으로 짧거나 길게
[2] 완벽한 기승전결 → 교훈이 깔끔하면 오염 → 교훈을 질문으로 / 해결을 불완전하게
[3] 대칭 구조 → "A하지만 B" 반복이면 오염 → 한쪽으로 기울어진 서술
[4] 일반적 감각 → "따뜻한 커피" 등 누구나 쓸 표현 → Brad만의 감각(씨앗 활용)
[5] 전환어 남발 → "결국/사실/중요한 것은" 3회 이상 → 전환어 없이 줄바꿈
[6] 감정 직접 명명 → "두려웠다" → 행동으로: "손이 키보드 위에서 멈췄다"
[7] 깔끔한 마무리 → "오늘도 걸어간다" → "커피가 식었다." / "모르겠다. 그래도."
```

- 오염 0~1개: 그대로 출력
- 오염 2~3개: 해당 문장 교체 후 출력
- 오염 4+개: 전체 재작성

## Phase 4: 5채널 변환

브런치 에세이를 원본으로 5개 채널 콘텐츠 생성.
채널별 상세 스펙: `channel-specs.md`
채널별 Brad 톤: `brand-voice-guide.md`

### 변환 요약

| 채널 | 형태 | 길이 | 정보:감성 | 핵심 변환 |
|------|------|------|----------|-----------|
| 네이버 블로그 | Q&A 정보형 | 1,000~1,500자 | 7:3 | 에세이→단계별 가이드. 소제목+AEO 구조 |
| 워드프레스 | 원본+SEO 보강 | 원본 동일 | 4:6 | H2 소제목 추가+메타 태그+구조화 데이터 |
| 카카오 채널 | 카드뉴스 요약 | 500~800자 | 3:7 | 감성 압축+브런치 링크 |
| 오픈채팅 | 대화 씨앗 | 50~150자 | 1:9 | 씨앗 한 줄+열린 질문 |
| 페이스북 | 공유 유도형 | 300~500자 | 5:5 | 에피소드 압축+통찰+해시태그 |

### 네이버 블로그 변환 핵심

제목 변환: 감성→검색 키워드
```
브런치: "새벽에 Claude와 싸웠다"
네이버: "AI 프롬프트 작성법 — 58세 비개발자가 알려주는 핵심 3가지"
```

첫 문단: AEO '정답-먼저' 구조
```
Q. 코딩을 모르는 50대도 앱을 만들 수 있나요?
A. 네, 가능합니다. 저는 58세 비개발자이고, Claude로 26개 웹앱을 만들었습니다.
```

Brad 경험 30% 유지: 정보 사이에 "저는 이걸 5번 실패하고 나서야 알았습니다" 삽입

### 워드프레스 변환 핵심

브런치 원문 그대로 + 다음 추가:
- H2 소제목 2~3개 (구글 크롤링용)
- SEO 메타 태그 (title, description, OG)
- 구조화 데이터 (Article Schema JSON-LD)
- 하단 CTA: 코칭/워크숍 + 책 구매 링크

### 카카오 채널 변환 핵심

에세이의 가장 인상적인 문장으로 시작, 감성 압축, 브런치 유도
```
새벽에 Claude와 세 번째 같은 대화를 하다가 깨달았어요.
AI가 못 알아듣는 게 아니라,
내가 질문을 못 하고 있었다는 걸.
→ 전체 이야기: [브런치 링크]
```

### 오픈채팅 변환 핵심

대화체. 씨앗+질문. 브런치 링크는 3번 중 1번만.
```
오늘 브런치에 쓴 건데요,
프롬프트 쓰는 게 결국 기획서 쓰는 거랑 같더라고요.
혹시 AI 쓰면서 "이거 내가 원래 하던 거랑 비슷한데?" 싶었던 적 있으세요?
```

### 페이스북 변환 핵심

후킹 첫 줄→에피소드 3~5줄→통찰 1문장→브런치 링크+해시태그 4~6개

## Phase 5: 이미지 프롬프트 (7종)

상세: `image-prompt-guide.md` 참조.

| # | 채널 | 비율 | 스타일 |
|---|------|------|--------|
| 1 | 브런치 커버 | 16:9 | 수채화, 감성 |
| 2 | 브런치 본문 | 4:3 | 미니멀 수채화 |
| 3 | 네이버 대표 | 16:9 | 깔끔한 인포그래픽풍 |
| 4 | 네이버 본문1 | 4:3 | 설명 보조 |
| 5 | 네이버 본문2 | 4:3 | 설명 보조 |

**공통 비주얼:** 새벽빛, 모니터불빛, 콘크리트 민들레, 중년남성 뒷모습(얼굴없음)

**씨앗 반영:** 시각 요소 있으면 이미지에 반영

## 참조 파일 가이드
