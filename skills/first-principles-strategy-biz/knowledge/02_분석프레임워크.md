# 6단계 제1원칙 분석 프레임워크 — 사업 전략 상세 가이드

이 문서는 프로젝트 지침서의 각 단계를 실행할 때 참조하는 상세 질문 리스트, 판단 매트릭스, 템플릿입니다.

---

## 1단계: 현재 상식 파악 — 질문 리스트

### 전제 추출 질문 7가지

1. 이 문서는 어떤 "당연한 것"을 전제로 하고 있는가?
2. "~해야 한다", "~이 필요하다"는 주장의 근거가 제시되어 있는가?
3. 비교 대상이 있는가? 비교 기준은 타당한가? (경쟁사 벤치마킹의 적절성)
4. 숫자가 제시된 경우, 그 숫자의 출처와 맥락이 명확한가? (TAM, 성장률, 전환율)
5. 문서 작성자가 암묵적으로 가정하는 고객상(像)은 무엇인가?
6. "보통", "일반적으로", "업계 평균"이라는 표현 뒤에 숨은 전제는?
7. 경쟁 상대에 대한 가정이 검증된 것인가? (경쟁자가 현상 유지할 것이라는 가정)

### 전제 분류 매트릭스

| 전제 유형 | 식별 방법 | 도전 가능성 |
|----------|----------|-----------|
| 규제/법적 제약 | 법률, 인허가, 규정 인용 여부 확인 | 낮음 (법 개정 전까지) |
| 구조적 관행 | "원래 그래요" 테스트 — 근거 없이 업계에서 반복 | 높음 |
| 데이터 기반 가정 | 표본 크기, 조사 시점, 방법론 검증 | 중간 (데이터 품질에 따라) |
| 직관적 판단 | "경험상", "보통", "업계에서는" 등의 표현 | 매우 높음 |
| 이론적 전제 | 학술/이론 프레임 적용 여부 (포터 5 Forces 등) | 중간 (맥락 적합성 검증 필요) |

### 전제 추출 시 주의사항

- **"[가정]" 또는 "[추정]" 표기된 수치**: 작성자조차 검증하지 않은 데이터. 반드시 미검증으로 분류
- **벤치마킹 사례**: 원본 사례와 적용 대상의 구조적 차이(시장, 규모, 시기)를 반드시 검토
- **복합 전제**: 하나의 전략 안에 여러 전제가 중첩된 경우, 개별 분리하여 각각 평가

## 2단계: 원자 단위 분해 — 분해 템플릿

### 사업모델 분해 구조
```
[사업 목표]
  ├── [핵심 전략 1: 고객 획득]
  │     ├── [세부 활동: 채널, 캠페인, 파트너십]
  │     │     ├── 필요 자원: 인력, 비용, 기술
  │     │     ├── 핵심 가정: "이 채널에서 고객이 온다"는 믿음
  │     │     └── 시장 제약: 경쟁 강도, 규제, 플랫폼 정책
  │     └── [세부 활동 1-2]
  │           └── ...
  └── [핵심 전략 2: 수익화]
        └── ...
```

### 수익모델 분해 구조
```
[총 매출 목표]
  ├── 매출 공식 (단가 × 고객 수 × 구매 빈도)
  ├── 비용 구조 (고정비 + 변동비)
  ├── 유닛 이코노믹스 (CAC, LTV, LTV/CAC 비율)
  ├── 성장 가정 (MoM/YoY 성장률의 근거)
  └── 브레이크이븐 시점 (BEP 도달 조건)
```

### 비용 분해 구조
```
[총 사업 비용]
  ├── 고정비 (사무실, 인건비, 인프라, SaaS 구독료)
  ├── 변동비 (마케팅, 물류, 원자재, 트랜잭션 비용)
  ├── 기회비용 (창업자 시간, 자본 배분 선택)
  └── 숨겨진 비용 (관행적으로 포함되지만 효과 미검증)
```

**숨겨진 비용 식별 체크리스트:**
- 기회비용: A에 투자한 자본/시간은 B를 못 한 것
- 고객 이탈의 재획득 비용: 이탈률 1% 증가 = CAC × 이탈 고객 수
- 기술 부채: 빠른 출시의 대가로 누적되는 유지보수 비용
- 팀 스케일링 비용: 인원 2배 ≠ 산출물 2배 (커뮤니케이션 오버헤드)
- 플랫폼 의존 비용: 구글/앱스토어/네이버 정책 변경 리스크
- 디지털 마케팅의 타겟 불일치: 노출수 ≠ 타겟 고객 도달 (봇, 비타겟 유입)

## 3단계: 전제 제거 및 검증 — 5Why 템플릿

### 5Why 실행 형식

각 전제에 대해 다음 형식으로 분석합니다:

```
전제: "[전제 내용]"
근거: 문서 [페이지/섹션]에서 "[인용]"

Why 1: 왜 이것이 사실이라고 가정하는가?
→ [답변]

Why 2: 왜 [Why 1의 답]인가?
→ [답변]

Why 3: 왜 [Why 2의 답]인가?
→ [답변]

Why 4: 왜 [Why 3의 답]인가?
→ [답변]

Why 5: 왜 [Why 4의 답]인가?
→ [답변]

★ 판정: [절대 제약 / 구조적 관행 / 미검증 가정 / 자기실현 전제]
도전 시 기대효과: [높음/중간/낮음] — [설명]
```

### 판정 기준 상세

**절대 제약 — 모든 조건을 충족해야 판정:**
- 물리적/기술적으로 변경 불가능하거나
- 법적으로 변경에 수년 이상 소요되거나
- 변경 시 분석 범위를 벗어나는 비용 발생

**구조적 관행 — 식별 신호:**
- "업계에서는 보통...", "SaaS에서는 원래..."
- 최근 5년간 변화가 없었던 관행
- 다른 산업/지역에서는 다르게 하는 사례 존재

**미검증 가정 — 식별 신호:**
- 데이터 근거 없이 제시된 인과관계
- "~일 것이다", "~로 보인다" 등의 추정 표현
- 반대 사례가 존재할 수 있지만 언급되지 않음

**자기실현 전제 — 식별 신호:**
- 모든 경쟁자가 같은 가정을 하고 있어 검증 기회가 없음
- "그렇게 하면 위험하다"지만 실제 시도한 사례가 없음
- 가정 자체가 행동을 제약하여 가정이 유지되는 순환 구조

### 5Why 품질 기준

좋은 5Why vs 나쁜 5Why:
- ✅ 각 Why가 이전 답변을 깊이 파고듦 (수직적 탐색)
- ❌ 같은 수준의 다른 이유를 나열 (수평적 나열)
- ✅ 마지막 Why에서 근본 원인이 드러남
- ❌ 3번째 Why에서 이미 결론이 나고 4~5는 형식적

## 4단계: 근본 원리 도출 — 체크리스트

### 변경 불가능성 테스트
- [ ] 이 원리는 기술 발전으로도 변하지 않는가?
- [ ] 이 원리는 산업/지역에 관계없이 성립하는가?
- [ ] 이 원리를 부정하면 논리적 모순이 발생하는가?

### 최소성 테스트
- [ ] 이 원리를 더 작은 원리로 분해할 수 있는가? (가능하면 분해)
- [ ] 이 원리에서 불필요한 수식어를 제거할 수 있는가?

### 사업 맥락의 전형적 근본 원리
1. 수익 = (고객 수 × 객단가 × 구매빈도) - 총비용
2. 고객 획득 = 노출 × 관심 × 전환 × 유지 (퍼널)
3. 가치 = 고객이 "돈을 내고 해결하고 싶은" 문제 해결
4. 성장 = 유지율 > 이탈률일 때만 지속 가능 (리키 버킷)
5. 경쟁우위 = 모방 비용이 높은 차별화 요소 (moat)

## 5단계: 새로운 해결책 설계 — 발상 프레임

### 창의적 발상을 위한 질문 6가지

1. **제거**: 기존 전략에서 아무것도 바뀌지 않으면서 제거할 수 있는 것은?
2. **역전**: 전제의 반대가 사실이라면 어떤 전략이 나오는가?
3. **극단**: 자원이 10배라면? 1/10이라면?
4. **차용**: 완전히 다른 산업에서 이 문제를 어떻게 해결하는가?
5. **결합**: 서로 무관해 보이는 두 요소를 결합하면?
6. **시간축**: 런칭이 내일이라면? 3년 뒤라면?

### 대비표 작성 가이드

| 영역 | 기존 접근 | 제1원칙 접근 | 차이의 근거 | 실현 가능성 | 리스크 |
|------|----------|-------------|-----------|-----------|--------|
| [영역명] | [현재 하는 방식] | [새로운 방식] | [왜 다른가 — 근본 원리 연결] | 높/중/하 | [주요 리스크] |

### 해결책 품질 기준

각 해결책은 다음을 반드시 포함:
- 핵심 아이디어 (1문장)
- 근본 원리와의 연결 (어떤 원리에서 도출되었는가)
- 기존 방식과의 본질적 차이 (표면적 변형이 아닌)
- 필요 조건과 리스크
- 기대 효과 (가능한 정량적)

## 6단계: 실행 가능한 첫 단계 — 실험 설계 템플릿

### MVP 실험 설계
```
실험명: [구체적 명칭]
검증 대상: [어떤 가설을 검증하는가]

실행 계획:
- 누가: [담당자/역할]
- 무엇을: [구체적 행동 — 동사로 시작]
- 언제까지: [기한 — "내일", "이번 주 금요일까지" 수준]
- 어떻게: [실행 방법 3단계 이내]
- 비용: [필요 자원 — 최소화 원칙]

성공 기준:
- 정량: [측정 가능한 KPI]
- 정성: [관찰 가능한 변화]

실패 시 학습:
- [이 실험이 실패하면 무엇을 알 수 있는가]
```

### 우선순위 매트릭스

| 우선순위 | 기준 | 설명 |
|---------|------|------|
| 🔴 즉시 | 비용 낮음 + 효과 높음 + 내일 가능 | 무조건 실행 |
| 🟡 이번 주 | 비용 낮음 + 효과 중간 | 계획 후 실행 |
| 🟢 2주 내 | 비용 중간 + 효과 높음 | 자원 확보 후 실행 |
| ⚪ 검토 | 비용 높음 또는 효과 불확실 | 추가 분석 필요 |
