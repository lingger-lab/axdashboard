# 프롬프트 템플릿 모음

이 프로젝트에서 자주 사용하는 프롬프트를 모아두었습니다. 복사하여 바로 사용하세요.

---

## 1. 기본 분석 — 문서 업로드 + 전체 6단계

**사용 시점**: 전략 문서/사업계획서를 처음 분석할 때

```
[문서 업로드]

이 문서를 제1원칙으로 분석해주세요.

분석 맥락:
- 분석 대상: [기업명 / 사업명]
- 산업/시장: [SaaS / 이커머스 / 핀테크 / 제조 / F&B 등]
- 분석 목적: [전략 점검 / 전제 검증 / 새로운 접근법 발굴 / 투자 판단]

특히 주목해줄 부분: [예: 수익 모델의 지속 가능성, TAM 추정의 근거, 유닛 이코노믹스 등]
```

---

## 2. 빠른 분석 — 전제만 추출

**사용 시점**: 전체 분석 전에 핵심 리스크만 빠르게 파악할 때

```
[문서 업로드]

이 문서의 숨겨진 전제만 찾아주세요.
각 전제에 대해:
1. 전제 내용
2. 문서 근거 (구체적 문장/숫자)
3. 전제 유형 (절대 제약 / 구조적 관행 / 미검증 가정 / 자기실현 전제)
4. 도전 가능성 (높음/중간/낮음)

표 형식으로 정리해주세요.
```

---

## 3. 해결책 집중 — 4~6단계만 실행

**사용 시점**: 전제는 이미 파악했고, 새로운 접근법이 필요할 때

```
[문서 업로드 또는 이전 분석 결과 참고]

이 전략의 핵심 문제는 [문제 설명]입니다.
기존 접근과 본질적으로 다른 해결책을 제시해주세요.

요구사항:
- 근본 원리에서 출발
- 기존 vs 제1원칙 대비표 포함
- 내일 당장 실행 가능한 첫 실험 3개
```

---

## 4. 교차 분석 — 복수 문서 비교

**사용 시점**: 사업계획 + 시장 데이터 등 2개 이상의 문서를 비교할 때

```
[문서 1 업로드] — 사업계획서
[문서 2 업로드] — 시장 분석 보고서 / 실적 보고서

이 두 문서를 교차 분석해주세요.

분석 초점:
1. 사업 전략이 시장/실적 데이터에 실제로 뒷받침되는가? (전략-데이터 정합성 매트릭스)
2. 데이터에서 전략이 놓치고 있는 신호가 있는가?
3. 두 문서의 전제가 충돌하는 부분은?
```

---

## 5. 특정 관점 분석

**사용 시점**: 특정 각도에서만 집중 분석하고 싶을 때

```
[문서 업로드]

이 문서를 [관점] 관점에서 제1원칙 분석해주세요.

관점 예시:
- "유닛 이코노믹스" 관점: CAC, LTV, 마진의 전제가 검증된 것인지
- "시간 제약" 관점: 6개월 안에 실행 가능한 것과 불가능한 것 구분
- "경쟁자" 관점: 경쟁자에 대한 가정이 검증된 것인지 분석
- "고객 행동" 관점: 고객 행동에 대한 가정이 실제 데이터와 일치하는지
- "스케일링" 관점: 10배 성장 시 깨지는 가정은 무엇인지
- "리스크" 관점: 최악의 시나리오에서 생존 가능한지
```

## 6. 간단 요약 — 3줄 + 즉시 실행

**사용 시점**: 바쁠 때, 핵심만 빠르게 파악할 때

```
[문서 업로드]

이 문서를 제1원칙으로 간단히 분석해주세요.
핵심 인사이트 3줄 + 내일 당장 실행할 것 3개만 알려주세요.
```

---

## 7. 문서 없이 주제 분석

**사용 시점**: 특정 사업 모델/전략에 대해 제1원칙 사고를 적용할 때

```
[사업 모델/전략 주제]에 대해 제1원칙 분석을 해주세요.

맥락:
- 현재 상황: [상황 설명]
- 일반적 접근법: [업계에서 통상 사용하는 방법]
- 내가 의심하는 전제: [있다면 기술]

이 분야의 일반적 상식과 관행을 1단계에서 다루고,
6단계 전체를 적용하여 새로운 접근법을 제안해주세요.
```

---

## 8. 피벗/전략 변경 분석

**사용 시점**: 기존 전략에서 방향을 전환할 때

```
[기존 전략 문서 또는 설명]

현재 [기존 전략]에서 [새로운 방향]으로 피벗을 고려하고 있습니다.

분석해줄 것:
1. 기존 전략의 전제 중 여전히 유효한 것 / 무효화된 것
2. 새로운 방향의 숨겨진 전제
3. 전환 비용과 리스크
4. 즉시 실행 가능한 검증 실험
```

---

## 9. 반복 분석 — 이전 분석 업데이트

**사용 시점**: 상황이 바뀌거나 새로운 데이터가 나왔을 때

```
[새로운 문서 또는 데이터 업로드]

이전 분석에서 [변경된 사항]이 바뀌었습니다.

확인해줄 것:
1. 이전 분석의 전제 중 여전히 유효한 것 / 무효화된 것
2. 새로운 데이터로 인해 추가된 전제
3. 해결책 업데이트 (변경 필요한 것만)
4. 즉시 실행 항목 재우선순위
```

## 10. DOCX 보고서 요청

**사용 시점**: 정식 보고서가 필요할 때

```
[문서 업로드]

이 문서를 제1원칙으로 분석하고 DOCX 보고서로 만들어주세요.

보고서 요구사항:
- 6단계 전체 포함
- 표/대비표 포함
- 핵심 인사이트 3줄 요약으로 시작
- 즉시 실행 로드맵으로 마무리
```

## 프롬프트 작성 팁

**좋은 프롬프트의 3요소:**
1. **맥락**: 분석 대상 기업/사업, 산업/시장, 목적을 명시
2. **초점**: "특히 주목해줄 부분"을 지정하면 분석 깊이가 달라짐
3. **출력 형식**: 표/대화/DOCX 중 원하는 형식을 명시

**분석 깊이를 조절하는 방법:**
- "간단히" → 3줄 요약 + 즉시 실행 3개
- "분석해줘" → 6단계 전체 (기본)
- "철저하게" → 전제 7개+ / 5Why 5개+ / 해결책 5개+
