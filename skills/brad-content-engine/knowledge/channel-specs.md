# 채널별 상세 스펙

## 1. 네이버 블로그

### 계정 정보
- 블로그명: "Brad Kim | 58세 기획자의 바이브코딩"
- 카테고리: IT·컴퓨터 > AI (단일 주제 집중 — 토픽 어쏘리티)
- 프로필: "30년 사업기획자. 코드 못 쓰지만 AI로 26개 앱을 만들었습니다."

### 콘텐츠 스펙

| 항목 | 규격 |
|------|------|
| 글 길이 | 1,000~1,500자 |
| 이미지 | 3장 (대표1 + 본문2) |
| 소제목 | H2 2~3개, H3 필요 시 |
| 발행 주기 | 주 3회 (월/수/금) |
| 발행 시간 | 오전 9~10시 (네이버 인덱싱 최적) |
| 태그 | 검색 키워드 5~10개 |

### 문체 규칙

**기본 톤:** 정보형 + 경험 삽입 (7:3)

```
✅ 허용:
- "~입니다", "~습니다" 존댓말
- 소제목, 번호 매기기
- "이렇게 하면 됩니다" 가이드형
- "저는 이렇게 했습니다" 경험 삽입
- 질문-답변(Q&A) 구조
- 요약표, 비교표

❌ 금지:
- 브런치식 독백체 ("했다. 그랬다.")
- 과도한 감성 ("새벽 4시의 적막 속에서...")
- 내용 없는 키워드 반복
- "여러분" 남발
- AI가 쓴 티 나는 나열형 ("첫째, 둘째, 셋째")
```

**Brad 경험 삽입 위치:**
- 도입부: "저는 58세 비개발자입니다. 이 방법을 5번 실패하고 나서야 알았습니다."
- 중간: 실용 정보 사이에 1~2문장 경험
- 결론: "아직 완벽하진 않지만, 이 방법으로 26개 앱을 만들 수 있었습니다."

### AEO/SEO 구조 (2026년 네이버 대응)

```
[제목] 검색 키워드 포함. "~하는 법", "~가이드", "~비교"
[첫 문단] 질문 + 2~3문장 핵심 답변 (AI 브리핑 인용 구조)
[H2] 단계별/항목별 상세 설명
[경험 삽입] 각 H2 사이에 Brad 경험 1문장
[결론] 핵심 요약 + 하단 링크
```

### 하단 링크 템플릿
```
━━━
📖 이 글의 전체 에세이 버전:
→ 브런치에서 읽기 [링크]

🔔 AI 시대 이야기를 카카오에서 받아보세요:
→ 카카오 채널 [링크]
━━━
```

## 2. 워드프레스 (enterlab.cloud/blog)

### 사이트 정보
- URL: enterlab.cloud/blog
- 역할: 구글 SEO + 콘텐츠 자산 소유 + 서비스 연결
- 프로필: Enterlabs 공식 블로그

### 콘텐츠 스펙

| 항목 | 규격 |
|------|------|
| 글 길이 | 1,500~2,500자 (브런치 원본과 동일) |
| 이미지 | 브런치 이미지 재사용 (2장) |
| 소제목 | H2 2~3개 추가 (SEO용) |
| 발행 주기 | 브런치 발행 당일 또는 다음 날 |
| 메타 태그 | 필수 — title, description, OG |

### 문체 규칙

**기본 톤:** 브런치 원문 유지 (감성 에세이 그대로)

변경 사항:
- H2 소제목 2~3개 추가 (구글 크롤링 최적화)
- 소제목은 자연스럽게: "그날 밤의 일" / "프롬프트가 기획서였다" 식
- 하단에 Enterlabs 서비스 소개 추가
- 관련 글 링크 추가

### SEO 체크리스트
- [ ] title 태그에 검색 키워드 포함
- [ ] meta description 140자 이내
- [ ] OG 태그 (title, description, image)
- [ ] 구조화 데이터 (Article Schema)
- [ ] 이미지 alt 텍스트
- [ ] 내부 링크 (이전 글 연결)
- [ ] 외부 링크 (브런치, 부크크 등)

### 하단 영역 템플릿
```
━━━

📚 Brad Kim의 책
→ 『늦은 출발』 구매하기 [부크크 링크]
→ 『만들어줘』 구매하기 [부크크 링크]

🤝 AI 시대 바이브코딩, 함께 시작하고 싶다면
```

- 채널명: 엔터랩스 (기존 채널 활용)
- 프로필 문구: "AI 시대, 경험이 무기가 되는 법 | 58세 기획자의 기록"
- 자동 응답: "반갑습니다. 새벽에 쓰고 AI와 만드는 사람, Brad입니다."

### 콘텐츠 스펙

| 항목 | 규격 |
|------|------|
| 글 길이 | 500~800자 |
| 이미지 | 1장 (1:1 카드뉴스 또는 3:4 세로형) |
| 발행 주기 | 주 3회 (브런치 발행 직후) |
| 발행 시간 | 오전 8시 또는 저녁 9시 |

### 문체 규칙

**기본 톤:** 감성 요약 (3:7 정보:감성)

```
✅ 허용:
- 브런치 독백체 유지
- 짧은 문장, 많은 줄바꿈
- 에세이의 핵심 감정 유지
- "→ 전체 읽기" 링크

❌ 금지:
- 긴 문단 (3줄 이상)
- 정보 나열
- "지금 바로!", "놓치지 마세요"
- 과도한 이모지
```

### 카드뉴스 이미지 규칙
- 비율: 1:1 (1080×1080px) 또는 3:4 (1080×1440px)
- 텍스트 오버레이: 에세이의 통찰 문장 1개
- 폰트: 깔끔한 고딕체 (Pretendard, Noto Sans KR)
- 색상: 흰색 텍스트 + 반투명 어두운 배경 (60% 불투명)
- 하단: "새벽메모 by Brad Kim" 워터마크 (작고 은은하게)

## 4. 오픈채팅 (집으로 출근하기)

### 채팅방 정보
- 이름: "집으로 출근하기" (기존 채팅방)
- 역할: 커뮤니티 + 씨앗 수집 + 관계 심화
- 인원 목표: 초기 20~50명 → 3개월 후 100명

### 콘텐츠 스펙

| 항목 | 규격 |
|------|------|
| 글 길이 | 50~150자 (대화체) |
| 이미지 | 없음 |
| 발행 주기 | 매일 아침 (씨앗) + 브런치 발행일 (링크) |
| 톤 | 편한 존댓말 또는 반말 혼합 |

### 메시지 유형

**유형 A: 아침 씨앗 (매일)**
```
오늘 새벽 메모에서 —
"AI에게 질문을 잘 하려면, 먼저 나에게 질문을 잘 해야 한다"

여러분은 오늘 아침에 어떤 생각이 떠올랐나요?
```

**유형 B: 브런치 알림 (주 3회)**
```
오늘 브런치에 새 글 올렸어요.
"프롬프트가 결국 기획서였다"

→ [링크]

혹시 읽어보시고 공감 가는 부분 있으면 여기서 이야기해요 😊
```

**유형 C: 순수 대화 (비정기)**
```
요즘 AI 관련해서 가장 궁금한 게 뭐예요?
저도 아직 모르는 게 많아서, 같이 찾아보면 좋겠어요.
```

**브런치 링크 빈도:** 3번 중 1번만. 매번 넣으면 광고 채널이 됨.

| 발행 시간 | 오전 8시 또는 오후 7시 |
| 해시태그 | 4~6개 |

### 문체 규칙

**기본 톤:** 통찰 공유 + 공유 유도 (5:5 정보:감성)

```
✅ 허용:
- 대화체

- 핵심 통찰 1문장 강조
- "비슷한 경험 있으세요?" 공유 유도
- 이모지 1~2개 (의미 전달용)
- 해시태그 4~6개

❌ 금지:
- 긴 글 (500자 초과)
- 링크만 던지기 (반드시 텍스트 + 링크)
- 이모지 남발
- "좋아요/공유 부탁드립니다"
- 과도한 해시태그 (7개 이상)
```

### 해시태그 전략

고정 태그 (매번 포함):
```

```

변동 태그 (에세이 주제에 따라 2~3개):
```

#중년의도전 #새벽메모 #경험이무기다
#AI와걷는사람 #콘크리트민들레

```
