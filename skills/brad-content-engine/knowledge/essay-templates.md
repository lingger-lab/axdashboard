# 에세이 구조 템플릿 (6가지)

각 템플릿은 실제 작성 시 유연하게 적용합니다.
구조에 갇히지 않되, 뼈대는 유지합니다.

---

## 1. 인간경험형

일상의 한 장면에서 시작해 AI 시대의 의미로 확장하는 구조.

```
[제목] 감각적 장면 또는 질문
[첫 3줄] 구체적 장면 묘사. 시간+장소+행동.
         "새벽 4시 반, 장유 아파트. 커피를 내리다가 멈췄다."
[일상 에피소드] 그날 있었던 일. 디테일하게.
[감정 포착] 그 순간 느낀 것. 솔직하게.
[확장] 이 경험이 AI 시대와 어떻게 연결되는지.
       억지로 연결하지 않음. 자연스러운 연상.
[흔들림] "확신은 없다. 그래도..."
[여운] 질문 또는 이미지로 마무리.
```

**예시 제목:** "새벽에 Claude와 싸웠다"
**예시 첫 3줄:**
"새벽 3시 47분. Claude가 내 말을 못 알아들었다.
세 번째 같은 프롬프트를 고쳐 쓰면서 생각했다.
이건 AI 문제가 아니라, 내 질문의 문제구나."

## 2. 시행착오형

실패 → 깨달음 → 재시도의 호를 그리는 구조.
Brad의 핵심 서사 "5번 실패 후 덴뿌라" 구조.

```
[제목] 실패를 암시하는 제목. 유머 가능.
[첫 3줄] 실패의 순간. 구체적.
[1차 시도] 뭘 했는지. 왜 안 됐는지.
[감정] 좌절, 자괴감, 자기 의심.
[전환점] 작은 깨달음. 거창하지 않은.
         "그때 문득 생각났다. 내가 30년간 해온 건..."
[재시도] 다르게 접근한 방법.
[결과] 성공이든 진행 중이든, 과장하지 않음.
[현재] "지금도 완벽하진 않다. 그런데..."
```

**예시 제목:** "다섯 번째 앱이 죽었을 때"
**예시 첫 3줄:**
"다섯 번째 만든 웹앱이 또 안 됐다.
에러 메시지를 읽을 줄도 몰랐다.
58세 비개발자의 체면 같은 건 이미 없었다."

---

## 3. 통찰형

하나의 깨달음을 중심으로 전개하는 구조.
"아, 이거였구나" 순간을 독자와 공유.

```
[제목] 깨달음을 암시하되 정답을 주지 않는 제목
[첫 3줄] 통찰의 계기가 된 상황
[배경] 왜 이 생각을 하게 됐는지
[관찰] 무엇을 보았는지 / 들었는지 / 읽었는지
[연결] 다른 영역의 경험과 연결
       "30년간 사업계획서를 쓸 때도 같은 패턴이었다"
[통찰] 핵심 깨달음. 한 문장으로.
       (이 한 문장이 에세이 전체의 무게중심)
[적용] 이 통찰이 지금 내 삶에서 어떻게 작동하는지
[열린 결말] "물론, 이것도 내일이면 바뀔 수 있다."
```

**예시 제목:** "프롬프트가 결국 기획서였다"
**예시 첫 3줄:**
"Claude에게 열 번째 프롬프트를 고쳐 쓰는데 깨달았다.
이건 30년 전 기획서 쓰던 것과 똑같다.
상대를 설득하는 글. 맥락을 주는 글. 원하는 걸 명확히 하는 글."

---

## 4. AI 철학형

AI 시대의 본질적 질문을 던지는 구조.
정보가 아닌 성찰. 답이 아닌 질문.

```
[제목] 철학적 질문 형태
[첫 3줄] 일상에서 마주친 AI 장면
[질문] 그 장면이 던지는 근본 질문
       "AI가 다 해주면, 나는 뭘 하는 거지?"
[경험 대비] 과거와 현재의 대비
          "예전에는 기획서를 3일 걸려 썼다. 지금은 30분."
[깊은 사유] 이 변화가 의미하는 것
[불안 인정] "솔직히 두렵다. 내가 불필요해지는 것 같아서."
[반전] "그런데, 오히려..."
       불안을 인정한 뒤 발견한 다른 가능성
[열린 질문] 독자에게 질문을 던지며 마무리
          "당신은 어떤가요?"
```

**예시 제목:** "AI가 내 30년을 삼킨다면"
**예시 첫 3줄:**
"어제 Claude가 3분 만에 사업계획서 초안을 썼다.
나는 같은 일을 30년간 해왔다.
3분. 30년. 그 사이에 내가 서 있다."

## 5. 뉴스반응형

당일 뉴스에서 시작해 개인 경험으로 착지하는 구조.
Phase 2 뉴스 연동과 연계.

```
[제목] 뉴스 키워드 + 개인 감정 결합
[첫 3줄] 뉴스를 본 순간의 개인적 반응
         "오늘 아침, 기사 하나가 눈에 걸렸다."
         뉴스 내용을 1~2문장으로 요약 (출처 표기)
[나의 반응] 이 뉴스가 나에게 의미하는 것
[과거 연결] 이 이슈와 관련된 내 경험
[현재] 지금 내가 하고 있는 것과의 연결
[해석] 58세 기획자의 눈으로 본 이 뉴스의 의미
       (정보 분석이 아닌 삶의 맥락에서 해석)
[질문] 독자에게 던지는 질문
[여운] "내일은 또 다른 뉴스가 올 것이다. 그래도."
```

**예시 제목:** "GPT-5가 나오는 날, 나는 커피를 내렸다"

## 6. 짧은몰입형 (숏에세이)

800~1,200자. 감정 한 조각을 포착하는 구조.
"스크롤을 멈추게 하는 글"

```
[제목] 짧고 강한 이미지
[전체] 하나의 장면, 하나의 감정, 하나의 여운.
       설명하지 않고 보여준다.
       시작과 끝이 하나의 이미지로 연결.
```

**예시:**

"새벽 4시의 프롬프트

모니터 불빛이 유일한 조명이다.
Claude에게 또 질문을 던진다.
답이 돌아온다. 빠르고 정확하게.

그런데 가끔, 이 대화가 끝나면
방 안이 너무 조용하다.

AI는 절대 "오늘 힘들었어?"라고 묻지 않는다.
그 질문은 아직 인간의 몫이다.

새벽 4시 23분.
커피를 한 모금 마시고,
다시 프롬프트를 쓴다.

이번엔 질문을 바꿔본다.
'나는 왜 이걸 만들고 있을까?'

Claude가 답한다. 길고 정확하게.
하지만 진짜 답은
내 안에서 아직 만들어지는 중이다.

커피가 식었다."

## 에세이 유형 선택 가이드

| 상황 | 추천 유형 |
|------|-----------|
| 오늘 있었던 일을 쓰고 싶다 | 인간경험형 |
| 뭔가 안 돼서 고생했다 | 시행착오형 |
| "아하!" 순간이 있었다 | 통찰형 |
| AI에 대해 깊이 생각했다 | AI철학형 |
| 뉴스가 마음에 걸린다 | 뉴스반응형 |
| 짧게 감정만 담고 싶다 | 짧은몰입형 |
| 글감이 없다 | content-calendar.md 참조 |

## "다음 글 보게 만드는" 연재 설계

### 연결 장치
- 에세이 말미에 "다음 새벽엔..." 또는 "내일은..."으로 다음 글 암시
- 연재물은 "n일차" 대신 감정 흐름으로 연결
- 3편 단위 미니 시리즈: 불안 → 시도 → 성찰

### 매거진 구성안
- 매거진 1: "늦은 출발" — AI 시대 적응 에세이 (메인)
- 매거진 2: "새벽 메모" — 짧은몰입형 모음
- 매거진 3: "만들어줘" — 바이브코딩 시행착오기
