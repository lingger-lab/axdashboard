# 6단계 제1원칙 분석 프레임워크 — 상세 가이드

이 문서는 프로젝트 지침서의 각 단계를 실행할 때 참조하는 상세 질문 리스트, 판단 매트릭스, 템플릿입니다.

---

## 1단계: 현재 상식 파악 — 질문 리스트

각 문서를 읽으며 다음 질문을 적용합니다:

### 전제 추출 질문 7가지

1. 이 문서는 어떤 "당연한 것"을 전제로 하고 있는가?
2. "~해야 한다", "~이 필요하다"는 주장의 근거가 제시되어 있는가?
3. 비교 대상이 있는가? 비교 기준은 타당한가?
4. 숫자가 제시된 경우, 그 숫자의 출처와 맥락이 명확한가?
5. 문서 작성자가 암묵적으로 가정하는 유권자상(像)은 무엇인가?
6. "보통", "일반적으로", "전통적으로"라는 표현 뒤에 숨은 전제는?
7. 경쟁 상대에 대한 가정이 검증된 것인가?

### 전제 분류 매트릭스

| 전제 유형 | 식별 방법 | 도전 가능성 |
|----------|----------|-----------|
| 법적 제약 | 선거법, 규정 인용 여부 확인 | 낮음 (법 개정 전까지) |
| 구조적 관행 | "원래 그래요" 테스트 — 근거 없이 반복 | 높음 |
| 데이터 기반 가정 | 표본 크기, 조사 시점, 방법론 검증 | 중간 (데이터 품질에 따라) |
| 직관적 판단 | "경험상", "보통" 등의 표현 | 매우 높음 |
| 이론적 전제 | 학술/이론 프레임 적용 여부 | 중간 (맥락 적합성 검증 필요) |

### 전제 추출 시 주의사항

- **"[가정]" 표기된 수치**: 문서 작성자조차 검증하지 않은 데이터. 반드시 미검증으로 분류
- **벤치마킹 사례**: 원본 사례와 적용 대상의 구조적 차이를 반드시 검토
- **복합 전제**: 하나의 전략 안에 여러 전제가 중첩된 경우, 개별 분리하여 각각 평가

## 2단계: 원자 단위 분해 — 분해 템플릿

### 전략 분해 구조
```
[최종 목표]
  ├── [핵심 전략 1]
  │     ├── [세부 활동 1-1]
  │     │     ├── 필요 자원: 인력, 비용, 시간
  │     │     ├── 핵심 가정: "이것이 효과가 있다"는 믿음
  │     │     └── 물리적 제약: 변경 불가능한 조건
  │     └── [세부 활동 1-2]
  │           └── ...
  └── [핵심 전략 2]
        └── ...
```

### 여론 데이터 분해 구조
```
[여론조사 결과]
  ├── 표본 설계 (크기, 추출법, 가중치)
  ├── 측정 항목 (인지도, 호감도, 지지율)
  ├── 분석 방법 (교차분석, 추세분석)
  ├── 해석 레이어 (작성자의 판단)
  └── 권고사항 레이어 (작성자의 제안)
```

### 비용 분해 구조
```
[총 캠페인 비용]
  ├── 고정비 (사무실, 인건비, 장비)
  ├── 변동비 (인쇄물, 광고, 행사)
  ├── 기회비용 (후보자 시간, 자원배분 선택)
  └── 숨겨진 비용 (관행적으로 포함되지만 효과 미검증)
```

**숨겨진 비용 식별 체크리스트:**
- 후보자 시간의 기회비용: A 활동에 투입한 시간은 B 활동을 못 한 것
- 인력 교육/관리 비용: 자원봉사자 100명 = 관리자 3명 필요
- 디지털 콘텐츠의 타겟 불일치: 조회수 ≠ 유권자 도달 (타 지역/연령 유입)
- 전수 접촉의 비효율: 확정 지지자에 대한 반복 접촉은 낭비

## 3단계: 전제 제거 및 검증 — 5Why 템플릿

### 5Why 실행 형식

각 전제에 대해 다음 형식으로 분석합니다:

```
전제: "[전제 내용]"
근거: 문서 [페이지/섹션]에서 "[인용]"

Why 1: 왜 이것이 사실이라고 가정하는가?
→ [답변]

Why 2: 왜 [Why 1의 답]인가?
→ [답변]

Why 3: 왜 [Why 2의 답]인가?
→ [답변]

Why 4: 왜 [Why 3의 답]인가?
→ [답변]

Why 5: 왜 [Why 4의 답]인가?
→ [답변]

★ 판정: [절대 제약 / 구조적 관행 / 미검증 가정 / 자기실현 전제]
도전 시 기대효과: [높음/중간/낮음] — [설명]
```

### 판정 기준 상세

**절대 제약 — 모든 조건을 충족해야 판정:**
- 물리적으로 변경 불가능하거나
- 법적으로 변경에 수년 이상 소요되거나
- 변경 시 분석 범위를 벗어나는 비용 발생

**구조적 관행 — 식별 신호:**
- "업계에서는 보통...", "선거에서는 원래..."
- 최근 5년간 변화가 없었던 관행
- 다른 나라/업계에서는 다르게 하는 사례 존재

**미검증 가정 — 식별 신호:**
- 데이터 근거 없이 제시된 인과관계
- "~일 것이다", "~로 보인다" 등의 추정 표현
- 반대 사례가 존재할 수 있지만 언급되지 않음

**자기실현 전제 — 식별 신호:**
- 모든 경쟁자가 같은 가정을 하고 있어 검증 기회가 없음
- "그렇게 하면 위험하다"지만 실제 시도한 사례가 없음
- 가정 자체가 행동을 제약하여 가정이 유지되는 순환 구조

### 5Why 품질 기준

좋은 5Why vs 나쁜 5Why:
- ✅ 각 Why가 이전 답변을 깊이 파고듦 (수직적 탐색)
- ❌ 같은 수준의 다른 이유를 나열 (수평적 나열)
- ✅ 마지막 Why에서 근본 원인이 드러남
- ❌ 3번째 Why에서 이미 결론이 나고 4~5는 형식적

## 4단계: 근본 원리 도출 — 체크리스트

### 변경 불가능성 테스트
- [ ] 이 원리는 기술 발전으로도 변하지 않는가?
- [ ] 이 원리는 문화/지역에 관계없이 성립하는가?
- [ ] 이 원리를 부정하면 논리적 모순이 발생하는가?

### 최소성 테스트
- [ ] 이 원리를 더 작은 원리로 분해할 수 있는가? (가능하면 분해)
- [ ] 이 원리에서 불필요한 수식어를 제거할 수 있는가?

### 선거 맥락의 전형적 근본 원리
1. 당선 = 유효 득표수에서 상대를 초과하는 것
2. 투표 행위 = 유권자의 주관적 판단 + 행동 실행(투표소 방문)
3. 유권자의 관심(attention) = 유한한 자원, 경쟁적 배분
4. 신뢰 = 일관성 × 진정성 × 시간
5. 정보 전달 = 메시지 × 도달률 × 수용도

## 5단계: 새로운 해결책 설계 — 발상 프레임

### 창의적 발상을 위한 질문 6가지

1. **제거**: 기존 전략에서 아무것도 바뀌지 않으면서 제거할 수 있는 것은?
2. **역전**: 전제의 반대가 사실이라면 어떤 전략이 나오는가?
3. **극단**: 자원이 10배라면? 1/10이라면?
4. **차용**: 완전히 다른 분야에서 이 문제를 어떻게 해결하는가?
5. **결합**: 서로 무관해 보이는 두 요소를 결합하면?
6. **시간축**: 선거일이 내일이라면? 1년 뒤라면?

### 대비표 작성 가이드

| 영역 | 기존 접근 | 제1원칙 접근 | 차이의 근거 | 실현 가능성 | 리스크 |
|------|----------|-------------|-----------|-----------|--------|
| [영역명] | [현재 하는 방식] | [새로운 방식] | [왜 다른가 — 근본 원리 연결] | 높/중/하 | [주요 리스크] |

### 해결책 품질 기준

각 해결책은 다음을 반드시 포함:
- 핵심 아이디어 (1문장)
- 근본 원리와의 연결 (어떤 원리에서 도출되었는가)
- 기존 방식과의 본질적 차이 (표면적 변형이 아닌)
- 필요 조건과 리스크
- 기대 효과 (가능한 정량적)

## 6단계: 실행 가능한 첫 단계 — 실험 설계 템플릿

### MVP 실험 설계
```
실험명: [구체적 명칭]
검증 대상: [어떤 가설을 검증하는가]

실행 계획:
- 누가: [담당자/역할]
- 무엇을: [구체적 행동 — 동사로 시작]
- 언제까지: [기한 — "내일", "이번 주 금요일까지" 수준]
- 어떻게: [실행 방법 3단계 이내]
- 비용: [필요 자원 — 최소화 원칙]

성공 기준:
- 정량: [측정 가능한 지표]
- 정성: [관찰 가능한 변화]

실패 시 학습:
- [이 실험이 실패하면 무엇을 알 수 있는가]
```

### 우선순위 매트릭스

| 우선순위 | 기준 | 설명 |
|---------|------|------|
| 🔴 즉시 | 비용 낮음 + 효과 높음 + 내일 가능 | 무조건 실행 |
| 🟡 이번 주 | 비용 낮음 + 효과 중간 | 계획 후 실행 |
| 🟢 2주 내 | 비용 중간 + 효과 높음 | 자원 확보 후 실행 |
| ⚪ 검토 | 비용 높음 또는 효과 불확실 | 추가 분석 필요 |
