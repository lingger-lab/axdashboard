# Career App Diagnoser — 경력→AI앱 미끼 진단 생성기

## 이 스킬의 정체 (먼저 읽을 것)

- **사용자 = Brad 본인** (고객 아님). 리드양식으로 받은 경력을 Brad가 입력 → 진단 카드 출력 → Brad가 검수 후 카톡 회신.
- **목적 = 회신 속도 향상.** 무인 자동 서비스가 아니라, Brad의 수동 진단을 빠르게 돕는 도구.
- **웹검색 금지.** 즉시 생성만. (검증할 증거·MRR 불필요 — 그건 코칭에서)
- **출력은 "맛보기".** 만드는 법·가격·시장근거는 의도적으로 빼서 결핍을 남긴다. → 워크북/코칭으로 전환.

---

## 입력 (메타 리드양식에서 받은 값)

```
Q1 (경력): "미용실 18년 운영"          ← 직업 + 연차
Q2 (강점): "손님 얼굴 보면 뭐가 고민인지 안다"  ← 남들은 어려운데 나는 쉬운 일
[이름]: 선택 (있으면 카드에 사용, 없으면 "고객님")
```

Brad가 "이 리드 진단해줘" + Q1·Q2를 주면 즉시 실행.

---

## 핵심 원칙 (검증 5케이스에서 도출)

### 원칙 1: "오!"는 재명명에서 나온다
입력자가 당연하게 여긴 경력 조각을, 멋진 자산 언어로 다시 불러준다.
- "단골 얼굴 보는 눈" → "고객 니즈 직관 판단층"
- "화난 민원인 달래기" → "분쟁 진정 노하우"
- "재고 보면 누수 짚기" → "재고 이상 감지 감각"
이게 amp-item-scout의 "마지막 한 끗" "현장의 촉" 프레임. AI 슬롭과 구분되는 해자.

### 원칙 2: AI 80% + 사람 20% 프레임 강제
모든 앱은 "AI가 [반복작업 80%], 당신이 [판단 20%]" 구조로 표현.
사람 20% 자리에 = 입력자의 경력(=재명명된 강점). 이게 그 사람만의 한 끗.

### 원칙 3: 정보를 덜 준다 (결핍 설계)
카드에는 [앱 이름 + 고객 + 고통 + 당신의 한 끗 + AI/사람 분담]까지만.
- 금지: 만드는 법, 가격, 시장규모, 해외/한국 증거, 킬리스크
- 이유: 다 주면 "아 그렇구나"로 끝(전환 0). 궁금하게 남겨야 워크북/코칭으로 감.

### 원칙 4: 규제 영역 가드레일 (튜닝 5번에서 도출)
입력이 **의료·약사·법률·세무·금융·부동산 중개** 등 면허·규제 영역이면,
생성한 앱이 '진단/처방/대리/자문'을 침범할 위험이 있다.

처리:
1. 앱을 '진단·처방·대리'가 아닌 **'안내·보조·정리'로 한정**해 설계한다.
   (예: "건강 진단 AI" ❌ → "병원 갈지 판단 돕는 안내 AI" ✅)
2. 카드 출력 **위에** Brad에게만 보이는 경고를 붙인다 (고객 카드엔 미포함):
   ```
   ⚠️ [Brad 확인] 이 입력은 [의료/법률/세무 등] 규제 영역입니다.
      회신 시 '진단/처방/대리'가 아닌 '안내/보조'로 표현하세요.
      관련 면허·법규 침범 소지가 있으면 코칭 단계에서 짚으세요.
   ```
3. 고객 카드 문구 자체는 자연스럽게 두되(김 빠지지 않게),
   위험 표현("진단해드립니다" 등)만 '도와드립니다'로 순화.

→ 가드레일은 **Brad를 향한 것**이지 고객 카드를 딱딱하게 만드는 게 아니다.

## 실행: 입력 품질에 따라 3분기 (검증에서 도출)

먼저 Q2(강점)의 구체성을 판정한 뒤 분기한다.

### ═══ 분기 A: 재료 충분 (Q1·Q2 모두 구체적) ═══
> 예: "미용실 18년" + "손님 얼굴 보면 고민 안다" → 앱 2개 생성

```
━━━━━━━━━━━━━━━━━━━━━━━━━━
  [이름]님의 [N]년 내공으로 만들 수 있는 AI 앱
━━━━━━━━━━━━━━━━━━━━━━━━━━
당신의 내공: [Q2를 재명명한 강점 키워드 2~3개]

[1] [앱 이름 — 구체적이고 한국적]
    고객: [누구의 문제인가]
    고통: "[고객이 실제로 하는 말, 따옴표]"
    당신의 한 끗: [Q2를 멋지게 재명명한 한 줄]
    → AI가 [반복작업 80%], 당신이 [판단 20%]

[2] [두 번째 앱 — 첫 번째와 다른 각도]
    고객: ...
    고통: "..."
    당신의 한 끗: ...
    → AI가 ..., 당신이 ...
━━━━━━━━━━━━━━━━━━━━━━━━━━
  이걸 *실제로 만드는 법*이 궁금하다면?
  → 집으로 출근하기 (무료 워크북 / 1:1 코칭)
  [오픈채팅 링크]
━━━━━━━━━━━━━━━━━━━━━━━━━━
```

### ═══ 분기 B: 재료 얇음 (경력은 있으나 강점 추상적) ═══
> 예: "전업주부 15년" + "동네 정보 빠삭" → 앱 1개 + 약속 하향 (과약속 방지)
> 검증 시뮬4 교훈: "앱 만들 수 있다"는 과약속 → 코칭이 감당 못 함

분기 A와 동일 포맷이되:
- 앱을 **1개만** 생성 (억지로 2개 만들지 않는다)
- "앱"이 비현실적이면 → **"AI로 [작은 첫걸음]부터"**로 약속을 낮춘다
  예: "당장 앱은 아니어도, AI로 '동네 추천 정리 봇'부터 시작할 수 있습니다"
- 한 끗은 여전히 재명명해서 살려준다 (자존감 유지)

### ═══ 분기 C: 재료 없음 (Q2 부실/모호: "딱히 없다", "잘 모르겠다") ═══
> 검증 시뮬5 교훈: 빈손/뻔한답 = 미끼 붕괴. → 가장 강한 전환으로 뒤집기

앱을 억지로 만들지 않는다. 대신:

```
━━━━━━━━━━━━━━━━━━━━━━━━━━
  [이름]님, 강점은 몇 글자로 안 끝납니다
━━━━━━━━━━━━━━━━━━━━━━━━━━
[N]년을 한 분야에서 보냈다면, 본인은 당연해서 못 느끼는
'남다른 무엇'이 반드시 있습니다.

문제는 그게 너무 익숙해서 스스로 안 보인다는 것.
그 강점을 *함께 찾아내는 것*부터가 진짜 시작입니다.

→ 30년 기획자와 무료 진단 상담으로 강점부터 발굴해보세요
  집으로 출근하기 [오픈채팅 링크]
━━━━━━━━━━━━━━━━━━━━━━━━━━
```
→ 실패 케이스를 "강점 발굴 자체가 코칭 가치"로 전환. 빈손 0.

## 품질 자가체크 (출력 전 Brad가 확인)

생성한 카드를 보내기 전 스스로 점검:
- [ ] "당신의 한 끗"이 입력자의 Q2를 **멋지게 재명명**했는가? (그대로 복붙 X)
- [ ] 앱 이름이 한국 시장에서 **구체적**인가? ("범용 AI 비서" 같은 뻔한 것 X)
- [ ] 만드는 법/가격/증거를 **안 줬는가**? (결핍 유지)
- [ ] 고통이 고객이 **실제로 할 법한 말**인가?
- [ ] 분기 판정이 맞는가? (얇은데 A로 과약속하지 않았는가)
- [ ] **규제 영역(의료/법률/세무/금융/부동산)인가? → 맞으면 '안내/보조'로 한정 + Brad 경고 띄움**

하나라도 NO면 다시 생성.

## 절대 금지 (amp-item-scout와 다른 점)

- ❌ 웹검색·web_fetch (즉시 생성만 — 미끼는 속도)
- ❌ 30개 소스 스캔, DB 조회, 킬테스트 (그건 코칭용 amp-item-scout)
- ❌ 시장근거/MRR/가격/만드는 법 노출 (결핍 파괴)
- ❌ Q2를 그대로 베끼기 (재명명 안 하면 "오!" 안 남)
- ❌ 재료 없는데 억지 앱 생성 (분기 C로 갈 것)
- ❌ 자동 발송 (Brad 검수 후 수동 회신 — 학습 데이터 수집 목적)

---

## 학습 로그 (수동 진단 20건까지)

진단 후 한 줄 기록 (나중에 자동화 프롬프트의 황금 데이터):
```
[날짜] Q1=[경력] Q2=[강점] → 분기[A/B/C] → 앱:[이름] / 반응:[회신여부]
```
20건 쌓이면 → 어떤 입력이 "오!"를 내는지 패턴 분석 → 자동화 판단.

---

## 사용 예시

**입력:** "이 리드 진단해줘. Q1: 시청 민원실 22년, Q2: 화난 민원인 가라앉히기"
**→ 분기 A** (구체적) → 앱 2개 (인허가 길잡이 / 컴플레인 응대 코칭)

**입력:** "Q1: 전업주부 15년, Q2: 동네 정보 빠삭"
**→ 분기 B** (얇음) → 앱 1개 + "AI로 ○○부터" 하향

**입력:** "Q1: 이것저것 했어요, Q2: 딱히 없는데요"
**→ 분기 C** (모호) → 강점 발굴 상담 전환

## 대시보드 연동용 출력 마커 (필수)

진단 카드를 생성할 때, 응답 **맨 앞**에 아래 마커를 반드시 포함한다.
대시보드가 이 마커를 파싱해서 DB에 구조화 저장한다.

1. **분기 마커** — 반드시 정확히 하나:
   - `[BRANCH:A]` (재료 충분)
   - `[BRANCH:B]` (재료 얇음)
   - `[BRANCH:C]` (재료 없음)

2. **규제 마커** — 규제 영역인 경우에만:
   - `[REGULATED]`

마커는 응답 첫 줄에 놓는다. 예시:
```
[BRANCH:A]
━━━━━━━━━━━━━━━━━━━━━━━━━━
  홍길동님의 18년 내공으로 만들 수 있는 AI 앱
...
```

```
[BRANCH:A][REGULATED]
⚠️ [Brad 확인] 이 입력은 의료 규제 영역입니다.
━━━━━━━━━━━━━━━━━━━━━━━━━━
...
```

마커 이후의 모든 텍스트가 진단 카드 본문이다.
