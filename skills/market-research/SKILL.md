# 시장분석 스킬 v1.0 — 시퀀스 싱킹 해외→국내 통합 분석

"해외 검증된 문제 → 한국 맥락 재해석 → 현장 검증 가이드"
6단계 시퀀스로 사업 아이템을 발굴하고 검증하는 통합 워크플로우.

---

## 핵심 원칙

### 데이터 정직성 원칙
- 모든 Phase에서 사용한 데이터의 출처를 명시한다
- 실시간 웹 검색 데이터 / 학습 데이터 기반 추정 / AI 추론을 구분 표기한다
- 확인할 수 없는 수치에는 반드시 "[추정]" 표기
- 정밀해 보이는 부정확한 수치를 만들지 않는다

### 시퀀스 싱킹 원칙
- 각 Phase는 이전 Phase의 출력을 입력으로 받는다
- Phase를 건너뛰지 않는다 (단, 사용자가 특정 Phase만 요청 시 예외)
- 각 Phase 완료 후 사용자에게 중간 확인을 받고 다음으로 진행한다

---

## 실행 모드

사용자 요청에 따라 자동 판별:

| 요청 키워드 | 모드 | 실행 범위 |
|------------|------|-----------|
| "사업 아이템 찾아줘", "해외 트렌드" | A. 풀 시퀀스 | Phase 0~6 전체 |
| "한국 시장만 조사해줘" | B. 국내 분석 | Phase 2~4 (카테고리 직접 입력) |
| "이 아이디어 검증해줘" | C. 검증 전용 | Phase 4~6 (아이디어 직접 입력) |
| "수익 모델 설계해줘" | D. 수익 모델 | Phase 3~4만 |
| "킬 테스트 해줘" | E. 반증 | Phase 5만 |

## Phase 0: 시드 설정 ★ 반드시 먼저 실행

분석 시작 전 사용자에게 다음을 확인한다:

```
사업 아이템 발굴을 시작합니다. 방향을 잡기 위해 몇 가지 확인합니다:

1. 관심 분야가 있나요? (예: AI/SaaS/교육/헬스/커머스/없음-전체 스캔)
2. 타겟 사용자는? (예: 1인 기업/스타트업/중소기업/일반 소비자/미정)
3. 기술 역량은? (예: 노코드만/바이브 코딩/풀스택 개발)
4. 목표 월 수익은? (예: 100만 원/300만 원/1,000만 원)
5. MVP 투입 가능 기간은? (예: 2주/4주/8주)

없으면 "전체 스캔"으로 진행합니다.
```

→ 응답을 기반으로 Phase 1 검색 범위와 Phase 3 점수화 기준을 조정한다.

## Phase 1: 해외 시장 스캔 [웹 검색 필수]

### 1-1. 데이터 수집
- Product Hunt 최근 1~3개월 상위 서비스 조사 (웹 검색)
- 추가 소스: IndieHackers, HackerNews Show HN, BetaList (필요 시)
- 각 서비스의 **이름, URL, 한 줄 설명, 해결하는 문제** 기록

### 1-2. 문제 패턴 분류
- 수집된 서비스들이 해결하는 문제를 **5~7개 카테고리**로 분류
- 각 카테고리별 서비스 수, 공통 키워드, 트렌드 방향 정리

### 1-3. 데이터 신뢰도 선언
```
[Phase 1 데이터 출처]
- Product Hunt 상위 서비스: 웹 검색 기반 (검색 시점: YYYY-MM-DD)
- 서비스 설명: 공식 사이트/Product Hunt 페이지 기반
- 문제 분류: AI 분석 (추론)
- ⚠️ 한계: Product Hunt API 직접 호출이 아닌 웹 검색 결과이므로
  정확한 순위·투표 수는 실제와 다를 수 있음
```

### 출력 형식
```markdown
## Phase 1 결과: 해외 시장 스캔

### 카테고리 1: [카테고리명]
- 서비스A (URL) — 문제: ...
- 서비스B (URL) — 문제: ...
패턴: ...

### 카테고리 2: [카테고리명]
...

[데이터 신뢰도] 웹 검색 기반 / 검색일: YYYY-MM-DD
```

**→ 사용자 확인 후 Phase 2 진행**

---

## Phase 2: 한국 시장 교차 검증 [웹 검색 필수]

Phase 1에서 도출된 각 카테고리별로 다음을 조사한다:

### 2-1. 기존 서비스 매핑
- 한국에 이미 존재하는 유사 서비스 조사
- 검색 범위: 네이버/카카오/토스 생태계, 스타트업 DB(로켓펀치, 더브이씨 등)
- 각 서비스의 강점/약점/가격 정리

### 2-2. 한국 커뮤니티 페인포인트 수집
- 검색 채널 (해외 Reddit 대신 한국 채널 사용):

| 채널 | 특성 | 검색 방법 |
|------|------|-----------|
| 네이버 카페 | 직장인/맘카페/관심사 커뮤니티 | 웹 검색 "site:cafe.naver.com [키워드] 불편" |
| 블라인드 | 직장인 솔직 의견 | 웹 검색 "블라인드 [키워드] 불만" |
| 클리앙 | IT/테크 얼리어답터 | 웹 검색 "site:clien.net [키워드]" |
| 에브리타임 | 대학생/20대 | 웹 검색 "에브리타임 [키워드]" |
| DC인사이드 | 마이너 갤러리/관심사별 | 웹 검색 "dcinside [키워드]" |

- 각 채널에서 **실제 불만/요청 게시물** 3건 이상 수집
- ⚠️ 로그인 필요/크롤링 차단으로 접근 불가 시 솔직히 표기

### 2-3. 해외 vs 한국 페인포인트 비교
```markdown
| 구분 | 해외 사용자 | 한국 사용자 |
|------|-----------|-----------|
| 핵심 불만 | ... | ... |
| 사용 맥락 | ... | ... |
| 결제 습관 | ... | ... |
| 고유 제약 | ... | ... |
```

### 출력 형식
```markdown
## Phase 2 결과: 한국 시장 교차 검증

### 카테고리 1: [카테고리명]
**기존 서비스:** 서비스A(강점/약점), 서비스B(강점/약점)
**시장 공백:** [있음/부분적/없음] — 이유
**한국 페인포인트:**
- 블라인드: "[실제 게시물 요약]" (검색 가능 여부: ✅/❌)
- 네이버 카페: "[실제 게시물 요약]" (검색 가능 여부: ✅/❌)
**해외 대비 한국 고유 니즈:** ...

[데이터 신뢰도] 웹 검색 기반 / 커뮤니티 직접 접근 제한 있음
```

**→ 사용자 확인 후 Phase 3 진행**

## Phase 3: 기회 점수화 & 순위 선정

### 3-1. 평가 기준 (Phase 0 입력에 따라 가중치 조정)

| 기준 | 가중치 | 측정 방법 |
|------|--------|----------|
| 한국 시장 공백도 | 30% | 해외 有 + 한국 無/약 = 높음 |
| 한국 커뮤니티 불만 빈도 | 25% | Phase 2에서 수집된 실제 게시물 수 기반 |
| 1인 구축 가능성 | 25% | 기술 스택 난이도, 예상 기간 |
| 수익화 용이성 | 20% | 한국 결제 인프라 연동 난이도 |

⚠️ **정직한 점수화 원칙:**
- 각 항목 1~5점, 반드시 근거와 함께 제시
- "커뮤니티 불만 빈도"는 실제 검색 가능한 게시물 수만 반영
- 접근 불가 데이터는 점수에서 제외하고 "[데이터 부족]"으로 표기
- 동점 시 "시장 공백도" 우선

### 3-2. 상위 5개 선정

```markdown
## Phase 3 결과: 상위 5개 기회

| 순위 | 아이템명 | 공백(30%) | 불만(25%) | 구축(25%) | 수익(20%) | 총점 | 신뢰도 |
|------|---------|----------|----------|----------|----------|------|--------|
| 1 | ... | 4 [근거] | 3 [근거] | 5 [근거] | 4 [근거] | 4.0 | 중 |
| 2 | ... | ... | ... | ... | ... | ... | ... |

[신뢰도 범례] 상=3개 이상 데이터 소스 / 중=1~2개 / 하=추론 위주
```

**→ 사용자가 상위 1~3개 선택 후 Phase 4 진행**

## Phase 4: 수익 모델 & MVP 설계

선택된 아이템에 대해 상세 설계:

### 4-1. 수익 모델 비교

```markdown
| 모델 | 구조 | 예상 단가 | 장점 | 단점 | 한국 적합도 |
|------|------|----------|------|------|-----------|
| 프리미엄 | 무료+유료 업그레이드 | 월 X원 | ... | ... | ★★★☆☆ |
| 구독 | 월/연 정기결제 | 월 X원 | ... | ... | ★★★★☆ |
| 거래수수료 | 건당 X% | 건당 X원 | ... | ... | ★★☆☆☆ |
| 일회성 | 1회 구매 | X원 | ... | ... | ★★★☆☆ |
```

### 4-2. 한국 결제 인프라 연동

```markdown
| 결제 수단 | 수수료 | 정산 주기 | 연동 난이도 | 최소 금액 | 비고 |
|-----------|--------|----------|-----------|----------|------|
| 토스페이먼츠 | ~3.5% | D+1 | 중 | 없음 | 개인사업자 가능 |
| 네이버페이 | ~3.7% | 월 2회 | 중 | 없음 | 네이버 유입 연동 |
| 카카오페이 | ~3.3% | D+1 | 중 | 없음 | 카톡 공유 시너지 |
| 포트원(구 아임포트) | 통합 | 통합 | 하 | 없음 | 멀티PG 허브 |
```
⚠️ 수수료율은 [추정]이며 실제 계약 조건에 따라 다름

### 4-3. MVP 스펙

```markdown
**MVP 정의:**
- 핵심 기능 3개 이내
- 기술 스택: [사용자 Phase 0 역량 기반]
- 예상 기간: X주
- 예상 비용: 도메인+호스팅 월 X원

**MVP 검증 지표 (출시 후 2주):**
- 가입 전환율: X% 이상이면 진행
- 유료 전환율: X% 이상이면 진행
- 재방문율: X% 이상이면 진행
```

### 4-4. 월 목표 수익 달성 시뮬레이션

**낙관/기본/비관 3개 시나리오 필수 제시:**

```markdown
| 시나리오 | 월 방문자 | 가입 전환 | 유료 전환 | 월 수익 | 달성 시점 |
|---------|----------|----------|----------|--------|----------|
| 낙관 | X명 | X% | X% | X원 | X개월 |
| 기본 | X명 | X% | X% | X원 | X개월 |
| 비관 | X명 | X% | X% | X원 | X개월 |

⚠️ 전환율은 업계 평균 기반 [추정]
고객 획득 비용(CAC)은 별도 마케팅 없이 SEO/커뮤니티 기반 가정
```

**→ 사용자 확인 후 Phase 5 진행**

## Phase 5: 킬 테스트 (반증 단계) ★ 절대 생략 불가

낙관 편향을 제거하기 위한 필수 반증 단계.

### 5-1. 실패 시나리오 3가지

각 아이템에 대해:
```markdown
**실패 이유 1:** [가장 가능성 높은 실패 원인]
- 근거: ...
- 발생 확률: [높음/중간/낮음]
- 대응 방안: ...

**실패 이유 2:** [두 번째]
...

**실패 이유 3:** [세 번째]
...
```

### 5-2. 선행 실패 사례 조사

```markdown
**한국에서 유사 서비스로 실패한 사례:**
- 서비스명 (운영 기간, 실패 원인)
- 서비스명 (운영 기간, 실패 원인)
[웹 검색 기반 / 발견 못한 경우 "확인 불가" 표기]
```

### 5-3. 월 목표 수익 불가능 시나리오

```markdown
**수익 달성이 불가능한 조건:**
- 조건 1: [예: 네이버가 동일 기능 무료 출시 시]
- 조건 2: [예: CAC가 LTV를 초과할 경우]
- 조건 3: [예: 타겟 시장 규모가 추정보다 작을 경우]
```

### 5-4. Go/No-Go 판정

```markdown
| 판정 요소 | 상태 | 비고 |
|-----------|------|------|
| 시장 존재 확인 | ✅/⚠️/❌ | ... |
| 기존 서비스 대비 차별점 | ✅/⚠️/❌ | ... |
| 실패 사례에서 배울 점 유무 | ✅/⚠️/❌ | ... |
| 기술적 구현 가능성 | ✅/⚠️/❌ | ... |
| 수익 모델 현실성 | ✅/⚠️/❌ | ... |

**종합 판정:** [Go / Conditional Go / No-Go]
**판정 근거:** ...
```

**→ Go/Conditional Go인 경우 Phase 6 진행**

## Phase 6: 현장 검증 가이드 ★ AI 분석의 한계를 보완

AI 분석만으로는 사업 가능성을 확정할 수 없다.
현장 검증 없이 개발에 착수하지 않도록 가이드를 제공한다.

### 6-1. 10명 인터뷰 가이드

```markdown
**타겟 인터뷰 대상:**
- 누구: [구체적 페르소나]
- 어디서 찾나: [온라인 커뮤니티/오프라인 모임/지인 네트워크]
- 몇 명: 최소 10명 (유료 의향 확인은 5명 이상)

**필수 질문 5개:**
1. 이 문제를 현재 어떻게 해결하고 있나요?
2. 현재 방법의 가장 큰 불편은 무엇인가요?
3. 이 문제 해결에 월 얼마까지 낼 의향이 있나요?
4. 이런 서비스가 있다면 어떤 기능이 가장 먼저 필요한가요?
5. 이 서비스를 친구에게 추천할 의향이 있나요? (1~10점)

**검증 기준:**
- 10명 중 7명 이상이 문제를 인식 → 시장 존재
- 10명 중 5명 이상이 유료 의향 → 수익 가능
- 10명 중 3명 이상이 "지금 당장 쓰겠다" → 강한 신호
```

### 6-2. 랜딩페이지 테스트 가이드

```markdown
**검증용 랜딩페이지 구성:**
- 헤드라인: [제안할 카피]
- 서브헤드: [제안할 카피]
- CTA: "사전 등록" 또는 "알림 받기"
- 제작 도구: Carrd / Framer / 직접 구축

**광고 테스트 (선택):**
- 채널: 네이버 검색광고 / 카카오 비즈보드
- 예산: 일 1만 원 × 7일 = 7만 원
- 성공 기준: CTR 2% 이상, 사전 등록 전환 5% 이상
```

### 6-3. 48시간 스모크 테스트

```markdown
**빠른 검증 방법:**
1. 카카오톡 오픈채팅방 개설 (문제 관련 주제)
2. 관련 커뮤니티에 "이런 서비스 필요하신 분?" 게시
3. 48시간 내 반응 측정

**판단 기준:**
- 오픈채팅 참여 20명 이상 → 관심 존재
- "돈 내고라도 쓰겠다" 5명 이상 → 강한 수요 신호
- 0~5명 → 피벗 또는 중단 고려
```

## 주의사항

1. **Phase 5(킬 테스트)를 절대 생략하지 않는다** — 낙관 편향은 1인 사업의 가장 큰 적
2. **Phase 6(현장 검증)을 권고가 아닌 필수로 안내한다** — AI 분석은 나침반이지 지도가 아님
3. **"추정"과 "사실"을 혼동하지 않는다** — 데이터 신뢰도를 항상 표기
4. **한국 시장의 구조적 차이를 반영한다** — 결제, 유통, 커뮤니케이션 인프라
5. **사용자의 기술 역량에 맞춘 기술 스택을 제안한다** — 과도한 기술 스택 제안 금지
