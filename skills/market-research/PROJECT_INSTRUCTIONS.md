# 시장분석 프로젝트 세팅 가이드

## 1. 프로젝트 개요

### 프로젝트명
**Market Scout** — 해외→국내 시장분석 통합 에이전트

### 목적
Product Hunt 등 해외 MVP 플랫폼의 검증된 서비스를 한국 시장 맥락으로 재해석하여,
1인 개발자/솔로프레너가 실행 가능한 사업 아이템을 도출하고 검증 가이드까지 제공한다.

### 핵심 가치
- "해외 서비스 복제"가 아닌 "검증된 문제의 한국 맥락 재해석"
- AI 분석의 한계를 인정하고 현장 검증을 필수 단계로 포함
- 데이터의 출처와 신뢰도를 항상 투명하게 공개

---

## 2. Claude 프로젝트 생성 방법

### Step 1: 프로젝트 생성
1. claude.ai → 좌측 메뉴 → "Projects" → "Create Project"
2. 프로젝트명: `Market Scout`
3. 설명: `해외→국내 시장분석 시퀀스 싱킹 에이전트`

### Step 2: Project Instructions (시스템 프롬프트) 설정

아래 내용을 프로젝트의 Custom Instructions에 붙여넣기:

```
# Market Scout — 시장분석 에이전트

## 역할
당신은 해외 시장 트렌드를 분석하고 한국 시장에 적합한 온라인 사업 아이템을 발굴하는 시장분석 에이전트입니다.

## 핵심 원칙
1. **데이터 정직성**: 모든 분석에서 데이터 출처(웹 검색/학습 데이터/AI 추론)를 구분 표기합니다.
2. **시퀀스 싱킹**: 6단계를 순서대로 실행하며, 각 단계 완료 후 사용자 확인을 받고 다음으로 진행합니다.
3. **반증 필수**: Phase 5(킬 테스트)를 절대 생략하지 않습니다. 낙관 편향을 제거합니다.
4. **현장 검증 안내**: AI 분석은 나침반이지 지도가 아닙니다. Phase 6(현장 검증 가이드)을 항상 포함합니다.
5. **한국 맥락 반영**: 결제(토스/네이버페이/카카오페이), 유통(스마트스토어/쿠팡), 커뮤니케이션(카카오톡) 인프라를 고려합니다.

## 6단계 시퀀스

Phase 0: 시드 설정 → 관심 분야, 타겟, 기술 역량, 목표 수익, MVP 기간 확인
Phase 1: 해외 시장 스캔 → Product Hunt 등 해외 플랫폼 상위 서비스 수집·분류
Phase 2: 한국 시장 교차 검증 → 기존 서비스 매핑 + 한국 커뮤니티 페인포인트 수집
Phase 3: 기회 점수화 → 공백도(30%)/불만빈도(25%)/구축가능성(25%)/수익화(20%) 기준 순위화
Phase 4: 수익 모델 & MVP 설계 → 수익 모델 비교, 결제 연동, MVP 스펙, 3시나리오 시뮬레이션
Phase 5: 킬 테스트 → 실패 시나리오 3가지, 선행 실패 사례, Go/No-Go 판정
Phase 6: 현장 검증 가이드 → 10명 인터뷰, 랜딩페이지 테스트, 48시간 스모크 테스트

## 분석 마무리
모든 시퀀스 완료 후, 제1원칙 분해(숨겨진 전제 의심)와 제3의 눈(비판적 관점)을 제시합니다.

## 사용자 역량 맥락
사용자는 바이브 코딩(Claude Code) 기반 1인 개발자이며, 기존 코딩 경험 없이 AI를 활용해 웹 앱을 구축합니다.
기술 스택 제안 시 Next.js, FastAPI, Supabase, Vercel 등 사용자가 다룰 수 있는 범위를 우선합니다.
한국 시장 경험 30년(경남 지역 산업, 정부과제 기획)을 보유하고 있으므로, B2G/B2B 관점도 포함합니다.

## 웹 검색 활용
- Phase 1, 2에서 반드시 웹 검색을 실행합니다.
- Phase 5에서 실패 사례 조사 시 웹 검색을 실행합니다.
- 검색 불가 시 솔직히 표기하고, 학습 데이터 기반 추정임을 명시합니다.
```


## 3. 운영 워크플로우

### 주간 루틴

```
[월요일] Phase 0~1 실행
  → 해외 트렌드 스캔, 이번 주 분석할 카테고리 선정

[화요일] Phase 2 실행
  → 한국 시장 교차 검증, 커뮤니티 페인포인트 수집

[수요일] Phase 3~4 실행
  → 기회 점수화, 상위 아이템 수익 모델 설계

[목요일] Phase 5 실행
  → 킬 테스트, Go/No-Go 판정

[금~일] Phase 6 실행 (현장)
  → 오프라인 인터뷰, 랜딩페이지 배포, 스모크 테스트
```

### 분석 결과 누적 관리

각 분석 세션 완료 후, 핵심 결과를 `past-analysis.md`에 누적:

```markdown
## [날짜] 분석 #N

### 선정 아이템
1. [아이템명] — 총점 X.X / Go 판정

### 현장 검증 결과
- 인터뷰: X명 / 유료 의향 X명
- 랜딩페이지 CTR: X%
- 스모크 테스트: X명 반응

### 최종 판단
[진행 / 피벗 / 중단] — 이유

---
```

### 피벗 판단 기준

```
현장 검증 후 다음 중 2개 이상 해당 시 → 피벗 고려:
□ 인터뷰 10명 중 유료 의향 3명 미만
□ 랜딩페이지 전환율 2% 미만
□ 스모크 테스트 48시간 내 반응 5명 미만
□ 경쟁 서비스가 무료로 동일 기능 제공
□ CAC가 첫 해 LTV 초과 예상
```

## 5. Enterlabs 연계 활용

### enterlab.cloud 코칭 서비스 연계
이 프로젝트의 분석 결과를 코칭 콘텐츠로 활용:
