# AI 도구 워크숍 실습 가이드 DOCX 생성 스킬 (v1.0)

원본 콘텐츠를 기반으로, 비전문가가 즉시 따라할 수 있는 워크숍 실습 가이드 DOCX를 생성합니다.

## 핵심 특징
- **범용 템플릿**: 변수 7개만 교체하면 모든 AI 도구 워크숍에 적용
- **2단계(Phase) 프로세스**: 구조 확인 후 생성 (lecture-script-docx와 동일 패턴)
- **Few-shot 품질 보장**: 골든 샘플 5종으로 섹션별 품질 기준 제공
- **인쇄+디지털 겸용**: 빈칸 워크시트 + 복사용 프롬프트 박스 공존

---

## 워크플로우 개요

```
[Phase 1: 설계 — 사용자 확인 필요]
Step 1: 입력 파싱 + 변수 추출
Step 2: 원본 콘텐츠 분석 + 핵심 요소 분류
Step 3: 문서 구조 설계 (9개 섹션 배분)
  → 사용자에게 구조안 제시 → 승인/수정 요청

[Phase 2: 생성 — 승인 후 실행]
Step 4: 콘텐츠 생성 (골든 샘플 품질 기준)
Step 5: DOCX 빌드
Step 6: 최종 검증
Step 7: 출력 + 전달
```

---

## Phase 1: 설계 (사용자 확인 단계)

### Step 1: 입력 파싱 + 변수 추출

사용자 요청에서 **7개 변수**를 추출합니다.

| 변수 | 필수 | 미추출 시 처리 |
|------|------|---------------|
| `TOOL_NAME` (도구명) | ✅ 필수 | 반드시 확인 질문 |
| `TOOL_TAGLINE` (부제) | ⚠️ | 도구 특성에서 자동 생성 후 고지 |
| `WORKSHOP_DURATION` (시간) | ⚠️ | 기본 2시간 고지 후 진행 |
| `TARGET_AUDIENCE` (대상) | ⚠️ | 기본 "AI 도구 초보자" 고지 후 진행 |
| `BRAND_COLOR` (브랜드 색상) | ⚠️ | 도구별 기본값 적용 (아래 표 참조) |
| `SOURCE_CONTENT` (원본 콘텐츠) | ✅ 필수 | 반드시 확인 질문 |
| `FILENAME` (파일명) | ⚠️ | `{도구명}-workshop-guide.docx` 자동 |

**도구별 기본 브랜드 색상:**
| 도구 | HEX | 비고 |
|------|-----|------|
| 수노 AI | #6B21A8 | 보라 |
| 미드저니 | #5865F2 | 디스코드 블루 |
| 감마 | #8B5CF6 | 퍼플 |
| 캔바 | #00C4CC | 틸 |
| 클로드 | #D97706 | 앰버 |
| ChatGPT | #10A37F | 그린 |
| 기타 | #2563EB | 블루 (기본값) |

**입력 등급 판정:**
- **A등급** (7개 변수 + 상세 원본 콘텐츠): 바로 Step 2
- **B등급** (도구명 + 원본 콘텐츠): 미추출 변수 자동 설정 고지 후 Step 2
- **C등급** (도구명만): 원본 콘텐츠 요청
- **D등급** (정보 부족): 확인 질문 후 진행

### Step 2: 원본 콘텐츠 분석 + 핵심 요소 분류

원본 콘텐츠(텍스트/PDF/URL)를 읽고 다음 5개 카테고리로 분류합니다:

| 카테고리 | 설명 | 문서 배치 섹션 |
|----------|------|---------------|
| **기본 개념** | 도구의 모드/기능 구분 | §3 핵심 개념 이해 |
| **핵심 공식/프레임워크** | 단계별 방법론, 템플릿 | §4 핵심 공식 |
| **상세 기법** | 태그, 파라미터, 고급 옵션 | §5 상세 기법 |
| **실습 소재** | 예시, 템플릿, 카테고리별 샘플 | §6 실습 워크시트 |
| **문제 해결** | 트러블슈팅, 팁 | §7 문제 해결 가이드 |

분류 결과를 내부적으로 정리한 뒤 Step 3에서 활용합니다.

### Step 3: 문서 구조 설계

**참조:** `references/document-structure.md` — 9개 섹션별 상세 요구사항

**설계 원칙:**
1. 전체 분량: 15~20페이지 (A4)
2. 텍스트:시각요소 비율 = 6:4 (표, 박스, 워크시트 포함)
3. 실습 워크시트(§6)는 전체의 최소 20% 비중
4. 각 섹션 시작 시 페이지 나누기

**Phase 1 출력물 — 사용자에게 제시:**

```
📋 워크숍 실습 가이드 구조안 (Phase 1)

■ 기본 설정
- 도구: [TOOL_NAME]
- 부제: [TOOL_TAGLINE]
- 대상: [TARGET_AUDIENCE]
- 워크숍 시간: [WORKSHOP_DURATION]
- 브랜드 색상: [BRAND_COLOR]

■ 섹션 구성 (예상 페이지)
| # | 섹션명 | 예상 페이지 | 주요 콘텐츠 |
|---|--------|------------|------------|
| 1 | 표지 | 1p | 제목, 부제, 주최 정보 |
| 2 | 시작하기 전에 | 1p | 목표 3개, 준비물 체크리스트 |
| 3 | 핵심 개념 이해 | 1~2p | 모드 비교표, 시나리오 |
| 4 | 핵심 공식 | 2~3p | 단계별 분해, 완성 예시 N개 |
| 5 | 상세 기법 | 2p | 기능 일람표, 좋은/나쁜 예 비교 |
| 6 | 실습 워크시트 | 2~3p | 빈칸형 실습 3개 |
| 7 | 문제 해결 | 1p | FAQ 최소 5개 |
| 8 | 고급 팁 | 1p | 최신 버전/고급 기능 |
| 9 | 부록 | 1~2p | 용어집, 메모 공간 |
| 합계 | | 15~20p | |

■ 원본 콘텐츠 활용 계획
- 추출된 핵심 개념: [N]개
- 공식/프레임워크: [이름]
- 예시 프롬프트: [N]개
- 문제 해결 팁: [N]개

■ 확인 요청
위 구조로 진행할까요? 수정사항이 있으면 말씀해주세요.
```

**사용자 응답 처리:**
- "좋아요 / 진행해줘" → Phase 2 시작
- 구체적 수정 요청 → 반영 후 구조안 재제시
- "전체 다시" → Step 2부터 재분류

## Phase 2: 생성 (승인 후 실행)

### Step 4: 콘텐츠 생성

**생성 전 필수 참조:**
1. `references/golden-samples.md` — 5종 골든 샘플의 품질 수준 확인
2. `references/document-structure.md` — 9개 섹션별 상세 요구사항
3. `references/design-specs.md` — DOCX 디자인 사양

**섹션별 품질 체크 (골든 샘플 기준):**
- [ ] 섹션 도입부: "🎯 이 섹션에서 배울 것:" 패턴 적용
- [ ] 비교표: 비유 행 포함 (예: "자판기 커피 vs 핸드드립 커피")
- [ ] 공식 단계: 좋은/나쁜 예 대비표 + TIP 박스
- [ ] 워크시트 빈칸: 힌트 포함 + 최종 조합 공간
- [ ] FAQ: 😣→🔍→✅ 이모지 구조
- [ ] 전문 용어: 첫 등장 시 "한글명(English)" 형식
- [ ] 격려 문구: 섹션 전환마다 자연스럽게 삽입
- [ ] 한 문단 3~4줄 이내

### Step 5: DOCX 빌드

**기술 스택:** Node.js + docx 라이브러리
**참조:** `references/design-specs.md` — 색상, 폰트, 표, 특수 요소 사양
**추가 참조:** `/mnt/skills/public/docx/SKILL.md` — docx 스킬 가이드라인

**DOCX 구현 필수 요소:**
1. A4 세로 (11906 x 16838 DXA)
2. 페이지 번호: 하단 중앙 (표지 제외)
3. 색상 체계: `BRAND_COLOR` 기반 자동 파생 (주색/연한색/진한색)
4. 표: 듀얼 폭 설정 (columnWidths + cell width)
5. TIP 박스: 왼쪽 색상 바 + 연한 배경
6. 워크시트 빈칸: 점선 밑줄 + 충분한 높이
7. 복사용 프롬프트 박스: 회색 배경 (#F5F5F5) + 모노스페이스
8. 각 섹션 시작: PageBreak

**색상 자동 파생 로직:**
```javascript
// BRAND_COLOR에서 파생
const PRIMARY = BRAND_COLOR;           // 제목, 강조
const LIGHT = lighten(BRAND_COLOR, 40); // 표 헤더 배경
const PALE = lighten(BRAND_COLOR, 80);  // TIP 박스 배경
const TEXT = "#1E1E1E";                 // 본문
const GRAY_BG = "#F5F5F5";             // 코드/프롬프트 박스
const GRAY_BORDER = "#DDDDDD";         // 박스 테두리
```

### Step 6: 최종 검증

```bash
python /mnt/skills/public/docx/scripts/office/validate.py output.docx
```

**품질 검증 매트릭스 (8항목):**
| # | 항목 | 기준 | 자동 |
|---|------|------|------|
| 1 | 섹션 수 | 9개 섹션 완비 | ✅ |
| 2 | 총 페이지 | 15~20p 범위 | ✅ |
| 3 | 실습 워크시트 | 최소 3개 실습 | ✅ |
| 4 | FAQ 항목 | 최소 5개 | ✅ |
| 5 | 용어집 | 최소 15개 용어 | ✅ |
| 6 | 비교표 포함 | 최소 2개 | ✅ |
| 7 | 골든 샘플 패턴 | 8개 체크 충족 | — |
| 8 | DOCX 유효성 | validate.py PASS | ✅ |

### Step 7: 출력 + 전달

1. DOCX를 `/mnt/user-data/outputs/`에 저장
2. 파일명: `{FILENAME}` (기본: `{도구명}-workshop-guide.docx`)
3. `present_files` 도구로 전달
4. 간결한 완료 요약

**완료 요약 형식:**
```
✅ 워크숍 가이드 생성 완료

📄 파일: {FILENAME}
📊 구성: {총 페이지}p / {섹션 수}개 섹션
🎯 실습: {워크시트 수}개 / FAQ: {FAQ 수}개 / 용어집: {용어 수}개
🎨 테마: {BRAND_COLOR}
✓ DOCX 유효성 검증 통과
```
