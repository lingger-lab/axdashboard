# 워크숍 실습 가이드 — 문서 구조 상세 요구사항

> 이 파일은 9개 섹션 각각의 목적, 필수 포함 요소, 페이지 배분, 작성 규칙을 정의합니다.
> Phase 1 Step 3 (구조 설계)과 Phase 2 Step 4 (콘텐츠 생성) 시 참조하세요.

---

## 섹션 1: 표지

**분량**: 1페이지  
**목적**: 워크숍의 정체성을 한눈에 전달

### 필수 요소
- **제목**: "{TOOL_NAME} 워크숍 실습 가이드" (H1, 중앙 정렬)
- **부제**: "{TOOL_TAGLINE}" (H2, 중앙 정렬)
- **주최 정보 빈칸** (하단 3줄):
  ```
  주최: ___________________________
  일시: ___________________________
  장소: ___________________________
  ```

### 디자인 규칙
- 제목에 `BRAND_COLOR` 적용
- 상단 50%는 여백으로 시각적 무게감 부여
- 페이지 번호 표시하지 않음

## 섹션 2: 시작하기 전에

**분량**: 1페이지  
**목적**: 참가자의 심리적 안전감 확보 + 준비 상태 확인

### 필수 요소

**1) 가이드 목적 (3줄 이내)**
- 말하듯 친근한 톤으로 작성
- 패턴: "이 가이드는 ~을 위한 것입니다. ~를 처음 접하시는 분도 걱정 마세요. 오늘 워크숍이 끝나면 ~할 수 있게 됩니다."

**2) 학습 목표 3가지 (체크박스)**
```
☐ 목표 1: {도구의 기본 개념}을 이해한다
☐ 목표 2: {핵심 공식/프레임워크}를 활용하여 직접 {결과물}을 만든다
☐ 목표 3: {결과물}을 개선하는 반복 전략을 익힌다
```

**3) 준비물 체크리스트**
| ☐ | 준비물 | 비고 |
|---|--------|------|
| ☐ | {도구} 계정 | 무료 가입 가능 |
| ☐ | 크롬(Chrome) 브라우저 | 최신 버전 권장 |
| ☐ | 이어폰/헤드셋 | 결과물 확인용 (음악/영상 도구의 경우) |
| ☐ | 필기구 | 워크시트 작성용 |

> 도구 특성에 따라 항목 조정 (예: 이미지 도구는 이어폰 불필요)

**4) 격려 문구 (하단)**
- 이탤릭, 중앙 정렬, `BRAND_COLOR`
- 예: *"처음부터 완벽할 필요 없습니다. 오늘은 즐기면서 배우는 날입니다!"*

## 섹션 3: 핵심 개념 이해

**분량**: 1~2페이지  
**목적**: 도구의 기본 구조를 직관적으로 파악

### 필수 요소

**1) 섹션 도입부**
- 패턴: "🎯 이 섹션에서 배울 것: {도구}의 {핵심 구분}을 이해하고, 내 상황에 맞는 {선택}을 할 수 있습니다."
- 일상적 비유 1개 포함

**2) 비교표 (최소 5행)**
| 구분 | 모드/기능 A | 모드/기능 B |
|------|------------|------------|
| 난이도 | ⭐ 표시 | ⭐ 표시 |
| 입력 분량 | 구체적 수치 | 구체적 수치 |
| 제어 수준 | 설명 | 설명 |
| 추천 상황 | "~할 때" | "~할 때" |
| **비유** | 일상적 비유 | 일상적 비유 |

> **중요**: 마지막 행에 반드시 **비유**를 넣어 직관적 이해를 도우세요.

**3) "이럴 때는 이 모드!" 판단 기준**
- 2~3개의 시나리오 제시
- 패턴: "만약 ~하고 싶다면 → [모드 A]를 선택하세요"

## 섹션 4: 핵심 공식/프레임워크

**분량**: 2~3페이지  
**목적**: 도구 활용의 핵심 방법론을 단계별로 체득

### 필수 요소

**1) 섹션 도입부** (골든 샘플 패턴)

**2) 공식 개요**
- 전체 공식을 한 줄로 요약
- 회색 배경 박스로 시각적 구분
- 예: `[장르] + [BPM] + [무드] + [악기] + [보컬] + [시대상] + [방향]`

**3) 각 단계별 상세 설명**
각 단계는 다음 구조를 따릅니다:
```
**단계 N. {이름}({English}) {한 줄 설명}**

{1~2문장 설명 + 비유}

| 🚫 이렇게 쓰면 | ✅ 이렇게 쓰세요 |
|----------------|-----------------|
| 모호한 예 | 구체적 예 |
| 모호한 예 | 구체적 예 |

💡 TIP: {실전 팁 1~2줄}
```

**4) 카테고리별 완성 예시 (최소 3개)**
각 예시는 다음을 포함:
- 카테고리/장르명
- 완성된 프롬프트 (코드 박스)
- "왜 이렇게 썼는지" 한 줄 해설

## 섹션 5: 상세 기법/고급 옵션

**분량**: 2페이지  
**목적**: 결과물의 품질을 높이는 세부 기술 습득

### 필수 요소

**1) 기능/태그/파라미터 일람표**
| 기능명 | 역할 | 사용 예시 |
|--------|------|-----------|
| ... | ... | 코드/명령 포함 |

> 최소 8행 이상

**2) 좋은 예 vs. 나쁜 예 비교 (2열 비교표)**
| 🚫 나쁜 예 | ✅ 좋은 예 | 왜 다른가? |
|-----------|-----------|-----------|
| ... | ... | 1줄 설명 |

> 최소 3쌍 이상

**3) 활용 템플릿 (복사용)**
- 회색 배경 박스
- 모노스페이스 폰트
- 빈칸 슬롯 포함 (사용자가 채울 부분)

## 섹션 6: 실습 워크시트 ★핵심 섹션

**분량**: 2~3페이지  
**목적**: 배운 내용을 직접 손으로 해보며 체화

### 설계 원칙
- 인쇄 후 **필기 가능**하도록 빈칸/밑줄 충분히
- 각 실습 시작에 **"이 실습의 목표"** 한 문장 + ⏱ **소요시간**
- 빈칸 옆에 **(힌트: ...)** 제공
- 실습 난이도: 1 < 2 < 3 순서로 점진적

### 실습 구성

**[실습 1] 첫 번째 결과물 만들기** ⏱ 약 15분
- 형식: 빈칸 채우기형 표
- 하단에 "최종 조합 공간" (큰 빈칸 박스)
- 각 단계의 빈칸에 힌트 제공

**[실습 2] 구조화 연습** ⏱ 약 10분
- 형식: 빈 템플릿에 자신의 내용 채우기
- 도구의 구조/태그/파라미터를 직접 적용해보는 연습

**[실습 3] 반복 생성 & 비교** ⏱ 약 15분
- 형식: 비교 기록표

| 시도 | 변경 사항 | 결과 특징 | 만족도 (1~5) |
|------|----------|----------|-------------|
| 1차 | (기본 프롬프트) | | ⭐⭐⭐⭐⭐ |
| 2차 | | | ⭐⭐⭐⭐⭐ |
| 3차 | | | ⭐⭐⭐⭐⭐ |

- 하단에 "내가 발견한 패턴:" 서술형 빈칸

## 섹션 7: 문제 해결 가이드

**분량**: 1페이지  
**목적**: 흔한 실패를 빠르게 진단하고 해결

### 필수 요소

**1) FAQ 최소 5개**
각 항목 형식:
```
😣 증상: {사용자가 겪는 문제를 구체적으로}

🔍 원인: {왜 이런 일이 발생하는지 1~2줄}

✅ 해결법:
1. {구체적 행동 1}
2. {구체적 행동 2}
3. {필요 시 행동 3}
```

**2) 핵심 원칙 강조 박스**
- TIP 박스 스타일 (왼쪽 색상 바)
- 예: "🔑 황금 규칙: 한 번에 하나씩만 바꾸기! 여러 요소를 동시에 수정하면 뭐가 효과가 있었는지 알 수 없습니다."

**분량**: 1~2페이지  
**목적**: 참조 자료 + 향후 학습 경로

### 필수 요소

**1) 용어집 (최소 15개)**
| 용어 | 뜻 | 관련 섹션 |
|------|-----|----------|
| ... | 쉬운 말로 1~2줄 | §N |

**2) 추천 리소스 (빈칸 5줄)**
```
1. _________________________________
2. _________________________________
3. _________________________________
4. _________________________________
5. _________________________________
```
> 워크숍 진행자가 자신의 추천 링크를 기입할 수 있도록 빈칸으로 제공

**3) 자유 메모 공간**
- 가로선 10줄
- 상단에 "📝 오늘 워크숍에서 기억하고 싶은 것:" 라벨
