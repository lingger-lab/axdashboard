## 골든 샘플 1: 섹션 도입부 패턴

**적용 섹션**: 모든 섹션의 첫 문단  
**핵심 패턴**: 목표 명시 → 일상적 비유 → 안심 메시지

### 샘플 (수노 AI — 핵심 개념 이해)

```
🎯 이 섹션에서 배울 것: 수노 AI(Suno AI)의 두 가지 핵심 모드를 이해하고,
내 상황에 맞는 모드를 선택할 수 있습니다.

요리에 비유하면, 하나는 "밀키트"이고 다른 하나는 "레시피북"입니다.
밀키트(심플 모드)는 재료가 다 손질되어 있어 바로 만들 수 있고,
레시피북(커스텀 모드)은 내가 원하는 재료를 골라 나만의 요리를 할 수 있습니다.

어떤 모드를 선택하든 맛있는 요리가 나옵니다. 차이는 "얼마나 내 입맛에
맞추고 싶은가"의 문제일 뿐이에요.
```

### 품질 체크포인트
- [ ] "🎯 이 섹션에서 배울 것:" 으로 시작
- [ ] 일상적 비유 1개 포함 (요리, 카페, 쇼핑 등)
- [ ] 3~4문단 이내
- [ ] 마지막 문장이 안심/격려 메시지

### 비유 패턴 은행 (도구별 활용)
| 비유 소재 | 적용 가능한 개념 |
|----------|----------------|
| 밀키트 vs 레시피북 | 심플 모드 vs 커스텀 모드 |
| 자판기 커피 vs 핸드드립 | 자동 생성 vs 수동 제어 |
| 내비게이션 vs 지도 | AI 추천 vs 직접 선택 |
| 기성복 vs 맞춤정장 | 템플릿 vs 커스터마이징 |
| 물감 세트 vs 빈 캔버스 | 프리셋 vs 자유 창작 |

## 골든 샘플 2: 비교표 패턴

**적용 섹션**: §3 핵심 개념 이해  
**핵심 패턴**: 5행 이상 + 마지막 행에 비유 + 시각적 아이콘

### 샘플 (수노 AI — 심플 vs 커스텀)

```
| 구분 | 심플 모드(Simple) | 커스텀 모드(Custom) |
|------|-------------------|---------------------|
| 난이도 | ⭐ 매우 쉬움 | ⭐⭐⭐ 보통 |
| 입력 분량 | 최대 200자 | 최대 3,000자 |
| 제어 수준 | AI가 대부분 결정 | 사용자가 세밀하게 제어 |
| 추천 상황 | "일단 빨리 만들어보고 싶을 때" | "특정 장르와 분위기를 정확히 원할 때" |
| 소요 시간 | 30초 | 3~5분 |
| **비유** | **자판기 커피 ☕** | **핸드드립 커피 ☕** |
```

### 품질 체크포인트
- [ ] 최소 5행 이상
- [ ] 마지막 행은 반드시 **비유** (볼드 + 이모지)
- [ ] 수치가 있는 항목은 구체적 숫자 사용
- [ ] "추천 상황" 행은 인용부호로 감싸기
- [ ] 난이도는 ⭐ 아이콘으로 시각화

## 골든 샘플 3: 공식 단계 설명 패턴

**적용 섹션**: §4 핵심 공식/프레임워크  
**핵심 패턴**: 이름 + 비유 → 좋은/나쁜 예 대비표 → TIP 박스

### 샘플 (수노 AI — 7단계 프롬프트 공식 중 2단계)

```
**단계 2. BPM(Tempo) — 곡의 심장박동을 정하기**

BPM은 "곡의 심장박동"입니다. 숫자가 클수록 빠르고 에너지가 넘치며,
작을수록 느리고 감성적입니다. "빠르게"라는 단어보다 숫자를 쓰면
AI가 훨씬 정확하게 리듬을 잡아줍니다.

| 🚫 이렇게 쓰면 | ✅ 이렇게 쓰세요 | 느낌 |
|----------------|-----------------|------|
| fast tempo | 130 BPM | 신나는 댄스 |
| slow | 72 BPM | 잔잔한 발라드 |
| medium speed | 100 BPM | 편안한 산책 |

💡 TIP: BPM을 잘 모르겠다면, 좋아하는 곡을 들으며 1초에 몇 번
손뼉을 치는지 세어보세요. 그 숫자에 60을 곱하면 대략적인 BPM입니다!

📌 BPM 빠른 참조표:
- 60~80: 발라드, 명상 음악
- 80~100: R&B, 재즈
- 100~120: 팝, 인디
- 120~140: 댄스, EDM
- 140+: 드럼앤베이스, 하드코어
```

### 품질 체크포인트
- [ ] 단계 번호 + 영문명 + 한 줄 비유 제목
- [ ] 1~2문장으로 핵심 설명
- [ ] 좋은/나쁜 예 대비표 (최소 3행)
- [ ] 💡 TIP 박스 (실전에서 바로 쓸 수 있는 팁)
- [ ] 선택적: 📌 빠른 참조표 (수치가 있는 단계에 적합)

## 골든 샘플 4: 실습 워크시트 패턴

**적용 섹션**: §6 실습 워크시트  
**핵심 패턴**: 목표 + 소요시간 → 빈칸표 (힌트 포함) → 최종 조합 공간

### 샘플 (수노 AI — 실습 1)

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
실습 1: 나의 첫 번째 곡 만들기
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

🎯 이 실습의 목표: 7단계 프롬프트 공식을 활용하여
나만의 스타일 프롬프트를 완성합니다.

⏱ 예상 소요시간: 약 15분

아래 빈칸을 하나씩 채워 나만의 프롬프트를 완성해 보세요!

| 단계 | 질문 | 나의 선택 |
|------|------|-----------|
| 1. 장르 | 어떤 느낌의 음악을 만들고 싶나요? | |
| | (힌트: pop, rock, jazz, R&B, EDM...) | _________________ |

| 3. 무드 | 어떤 감정을 담고 싶나요? | |
| | (힌트: 신나는, 잔잔한, 슬픈, 웅장한, 몽환적인...) | _________________ |
| 4. 악기 | 중심이 될 악기는? | |
| | (힌트: piano, acoustic guitar, synth, orchestra...) | _________________ |
| 5. 보컬 | 어떤 목소리를 원하나요? | |

▼ 위 7개를 조합한 나의 최종 프롬프트를 아래에 적어보세요:

```

### 품질 체크포인트
- [ ] 🎯 목표 + ⏱ 소요시간 상단에 명시

- [ ] 각 빈칸에 (힌트: ...) 포함
- [ ] 하단에 최종 조합 공간 (큰 빈칸 박스)
- [ ] 마지막에 다음 행동 안내 + 격려 이모지
- [ ] 인쇄 시 실제로 필기 가능한 여백

## 골든 샘플 5: FAQ 문제 해결 패턴

**적용 섹션**: §7 문제 해결 가이드  
**핵심 패턴**: 😣 증상 → 🔍 원인 → ✅ 해결법 (번호 매기기)

### 샘플 (수노 AI — 보컬 속도 문제)

```
😣 증상: 보컬이 가사를 너무 빨리 읽어서 부자연스럽게 들려요

🔍 원인: 한 줄에 단어가 너무 많으면, AI가 정해진 마디 안에 모든 가사를
우겨넣으려고 빠르게 부릅니다. 사람도 숨 한번 안 쉬고 긴 문장을 말하면
서두르게 되는 것과 같습니다.

✅ 해결법:
1. 가사 한 줄을 8단어(한국어 기준 15글자) 이하로 줄이세요
2. 한 섹션(Verse/Chorus)을 8행 이내로 구성하세요
3. 줄 사이에 빈 행을 넣어 호흡 공간을 만들어 보세요

*가사를 줄이고 다시 생성해 보세요. 대부분 이것만으로 해결됩니다!* 🎵

────────────────────────────────────────────

😣 증상: 생성된 곡이 전체적으로 마음에 들지 않아요

🔍 원인: 스타일 프롬프트의 여러 요소가 서로 충돌할 수 있습니다.
예를 들어, "발라드 + 140 BPM"은 서로 맞지 않는 조합입니다.

✅ 해결법:
1. 한꺼번에 모든 것을 바꾸지 마세요 — "한 번에 하나씩만"이 핵심입니다
2. 먼저 장르만 바꿔서 재생성해 보세요
3. 장르가 괜찮으면 BPM을 조정하세요
4. 그래도 안 되면 무드를 바꿔보세요

*8~15개 변주를 만들어 비교하는 것이 프로의 방법입니다.
포기하지 마세요!* 💪
```

### 품질 체크포인트
- [ ] 😣🔍✅ 이모지 구조 일관 적용
- [ ] 증상은 사용자의 말투로 작성 ("~해요", "~인데요")
- [ ] 원인은 비유 포함 (일상에서의 유사 경험)
- [ ] 해결법은 번호 매기기 (구체적 행동)
- [ ] 마지막에 격려 문구 (이탤릭)
- [ ] 항목 사이 구분선
