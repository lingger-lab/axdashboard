# 미끼 진단 도구 — 엔터랩스 운영 대시보드 통합 사양서

> 대상: Claude Code (엔터랩스 AX 운영 대시보드에 진단 모듈 추가)
> 작성 목적: career-app-diagnoser 스킬을 대시보드의 '리드 진단' 모듈로 통합
> 연동 방식: **수동 입력** (메타 자동연동 없음 — 검증 단계)

---

## 1. 이 모듈이 하는 일 (한 줄)

메타 광고로 들어온 리드의 경력 2가지를 운영자가 붙여넣으면 →
career-app-diagnoser 스킬이 진단 카드를 생성 →
운영자가 검수·회신하고, 전 과정이 운영 데이터로 자동 기록된다.

## 2. 왜 수동 입력인가 (구현 시 명심)

- 검증 단계 리드 = 한 자릿수~수십. 웹훅 구축 비용 > 수동 처리 비용.
- 운영자가 리드를 한 건씩 보는 것 자체가 학습(20건 패턴 체득).
- 메타 자동연동(웹훅/Zapier)은 **검증 후 V2**. 지금 만들지 말 것.
- → 이 모듈에 메타 API 연동 코드를 넣지 않는다.

```
┌─────────────────────────────────────────────┐

├─────────────────────────────────────────────┤

│   이름(선택) [___________]                     │

│   Q2 강점    [_________________________]       │

├─────────────────────────────────────────────┤
│  [결과 영역] — 스킬이 생성한 진단 카드 표시       │
│   (분기 A/B/C 자동 판정 결과)                   │

│   날짜 | 경력 | 강점 | 분기 | 앱 | 회신 | 전환    │

```

## 4. 데이터 모델 (운영 DB에 테이블 1개 추가)

테이블명: `lead_diagnoses`

| 필드 | 타입 | 설명 |
|---|---|---|
| id | PK | 자동 |
| created_at | datetime | 입력 시각 |
| name | text | 이름(선택) |
| q1_career | text | Q1 경력 (예: "미용실 18년") |
| q2_strength | text | Q2 강점 |
| branch | enum(A/B/C) | 스킬이 판정한 분기 |
| is_regulated | bool | 규제영역 여부 (가드레일 발동) |
| diagnosis_card | text | 생성된 카드 전문 |
| replied | bool | 회신 완료 체크 |
| converted | bool | 오픈채팅 합류/구매 전환 |
| reaction_note | text | 반응 메모 (회신율 분석용) |

→ 이 테이블이 곧 **정밀화의 측정 인프라**. 검증 후 이 데이터로 스킬을 다듬는다.

## 5. 스킬 호출 방식

- career-app-diagnoser SKILL.md를 Claude API(claude-opus-4-8 또는 sonnet)에 시스템 프롬프트로 주입.
- 사용자 입력(Q1·Q2)을 메시지로 전달 → 카드 텍스트 응답 수신.
- 응답에서 분기(A/B/C)·규제경고 유무를 파싱해 DB 필드에 분리 저장.
- **스킬 본문은 SKILL.md를 단일 원본으로 참조** (대시보드에 로직 중복 구현 금지).

## 6. 핵심 동작 규칙

1. [진단 생성] 클릭 → 스킬 호출 → 카드 + 분기 + 규제플래그 반환 → 결과 영역 표시 + DB 저장.
2. 규제영역(is_regulated=true)이면 → 결과 상단에 빨강 경고 배지 ("⚠️ 규제영역 — 안내/보조로 회신").
3. 운영자가 카드 검수 → [복사]로 카톡 회신 → [회신완료] 체크 → replied=true.
4. 나중에 전환 확인되면 converted 체크 + reaction_note 기록.

## 7. 만들지 말 것 (범위 제한)

- ❌ 메타 API/웹훅 연동 (V2)
- ❌ 자동 발송 (반드시 운영자 수동 회신 — 학습 목적)
- ❌ 진단 로직을 대시보드에 따로 구현 (SKILL.md가 단일 원본)
- ❌ 입력 질문 늘리기/분기 세분화 (검증 데이터 나오기 전 정밀화 금지)

## 8. 통합 후 단일 원본 원칙

- 통합 완료 시 → 대시보드의 SKILL.md 참조본을 **단일 원본**으로 선언.
- 광고 검증용 독립 사본(~/.claude/skills/)은 **임시**. 검증 끝나면 폐기.
- 영구히 두 원본을 두지 않는다 (동기화 혼란 방지).
