# 엔터랩스 AX 전환 — 개요 (Claude Code 실행용)

> **문서 버전**: 1.0
> **대상**: Claude Code, 엔터랩스 팀(Brad·김동현·박이사)
> **목적**: 흩어진 자산(34레포·20스킬·RAG·로컬LLM)을 하나의 흐름으로 묶어, 업무를 체계화하고 콘텐츠 발행을 자동화하는 엔터랩스 자체 AX 전환의 전체 그림.

---

## 0. 한 줄 정의

엔터랩스 AX는 "도구 추가"가 아니라 **이미 가진 자산을 3층(데이터→지식→실행)으로 묶고, 업무 프로세스를 체계화하며 콘텐츠 발행을 자동화하는 것**이다.

> 영감: '똑똑한개발자' 서장원 인터뷰 — AX는 도구 도입이 아니라 일하는 리듬의 전환이며, DX(데이터 적재)가 선행되어야 한다. 다 넣는다고 좋은 게 아니라 핵심만, 사람이 검수·결정.

---

## 1. 핵심 원칙

1. **DX 우선**: 데이터·자산이 한곳에 모여야 AI가 작동. 자산 카탈로그화가 1번 작업.
2. **기존 자산 최대 활용**: 새로 만들지 않는다. 34레포·20스킬·RAG·로컬LLM(Qwen3)을 연결.
3. **확인을 자동화**: 34레포 확인을 사람이 손으로 하지 않고 Claude Code가 스캔(인증으로 비공개·rate limit 우회).
4. **빠른 ROI 먼저**: 콘텐츠 자동화(집출 뉴스레터·영상)로 즉시 성과 → 팀이 AX 효과 체감 → 확산.
5. **사람이 검수·결정**: 자동화는 정보 정리까지, 최종 판단은 사람(휴먼 인 더 루프).

---

## 2. 3층 구조

```
[3층 — 실행] 에이전트·자동화 (콘텐츠 파이프, cron 발행, 알림)
       ↑
[2층 — 지식] RAG·검색 (팀 자료를 AI가 찾아 활용) ← 김동현 RAG + 로컬LLM(Qwen3)
       ↑
[1층 — 데이터] 자산·데이터 적재 (DX) ← 가장 먼저, Claude Code 스캔으로 자동화
```

구축 순서는 아래에서 위로. 1층(데이터) 없이 3층(자동화) 만들면 모래 위 집.

## 3. 4기둥 (3층 위에 엔터랩스 업무 특화)

```
┌──────────┬──────────┬──────────┬─────────────────┐
│ ①자산허브 │ ②데이터  │ ③프로세스 │ ④콘텐츠         │
│ (카탈로그)│ 레이크   │ 체계화   │ 자동화 파이프    │
│ 34레포    │ (DX)     │ (워크플로)│ (뉴스레터·영상)  │
│ 20스킬    │ RAG연결  │ 5단계뼈대 │ 집출 발행자동화  │
└──────────┴──────────┴──────────┴─────────────────┘
        ↑ 공통 기반: RAG + 로컬LLM(Qwen3) + Claude
```

## 4. 엔터랩스 업무 4도메인 (스킬 분석 결과)

20개 스킬 분석 결과, 엔터랩스 업무는 4도메인으로 수렴한다. 자산 분류 기준이 된다.

| 도메인 | 대표 자산(스킬) |
|---|---|
| 콘텐츠 생산 | Zipchul-ax-content-system, AI숏폼 빌더, AI뉴스 큐레이터, AI뉴스레터 빌더, 프롬프트 생성, 강연대본·프레젠테이션 생성 |
| 전략·분석 | 제3의눈(경선전략·여론조사·본선 양자분석), 비즈니스전략·일론머스크 제1원칙 |
| 아이템·사업 발굴 | AMP 아이템 스카우트, 아이템 발굴 내외시장·페인포인트, 씨앗·뉴스기반 브랜딩 |
| 선거·여론(CampOne) | CampOne 여론조사 사업 진단, 경선전략 실행문서 생성기, 공천 AI 모델 고득점 플래너 |

> 콘텐츠 생산이 최다 → ④ 콘텐츠 자동화가 실전 1순위 모듈인 근거.

## 6. 기술 스택 (곁에와 통일)

| 레이어 | 기술 | 비고 |
|---|---|---|
| 프론트·앱 | Next.js + TypeScript + Tailwind | campone 대시보드 UI/컴포넌트 재사용 |
| DB·인증·저장 | **Supabase** | 곁에와 통일(Cloud Run/Prisma 아님) |
| 배포 | **Vercel** | 곁에와 통일 |
| 지식 | 김동현 RAG + 로컬LLM(Qwen3) | 기존 자산 |
| 자동화 | cron(Vercel Cron/Supabase) + 기존 스킬 | 콘텐츠 파이프 |

> campone-plane-dashboard는 Cloud Run+Prisma지만, 스택 일원화를 위해 **UI·컴포넌트·레이아웃만 재사용**하고 배포·DB는 Vercel+Supabase로 맞춘다. 팀이 한 가지 인프라만 관리.

---

## 7. 곁에와의 관계

- AX 대시보드 = 내부 도구, 곁에 = 제품. 별개지만 기반 공유.
- 둘 다 Next.js+TS+Vercel+Supabase → 코드·컴포넌트·배포 노하우 재사용.
- 곁에 운영 자동화(곁에 마스터 v3.4 §9.2)는 AX 대시보드 ②③에 흡수.
